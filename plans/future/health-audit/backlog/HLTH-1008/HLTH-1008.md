# HLTH-1008: Resolve missing @repo/upload-client module

## Context

`apps/web/app-instructions-gallery/src/hooks/usePresignedUpload.ts:25` imports from `@repo/upload-client` which does not exist as a workspace package. This causes 3 TS2307 errors and cascades into 10+ test failures. The package was likely renamed to `@repo/upload` during a refactor.

## Goal

Determine the canonical package name for upload client functionality, update all imports in `app-instructions-gallery`, and add the correct workspace dependency.

## Non-Goals

- Not creating a new package from scratch if it already exists under a different name
- Not changing the upload hook's logic, only its import paths

## Scope

1. Check whether `@repo/upload` exports the symbols imported from `@repo/upload-client`
2. If yes: update `package.json` to depend on `@repo/upload@workspace:*` and fix import paths
3. If no: either create `@repo/upload-client` as a thin re-export package, or add the missing symbols to `@repo/upload`

**Affected file:** `apps/web/app-instructions-gallery/src/hooks/usePresignedUpload.ts`

## Acceptance Criteria

### AC1: No TS2307 for upload-client
**When** `pnpm turbo check-types --filter=@repo/app-instructions-gallery` runs
**Then** zero `Cannot find module '@repo/upload-client'` errors

### AC2: Tests unblocked
**When** `pnpm turbo test --filter=@repo/app-instructions-gallery` runs
**Then** test failures caused by the missing module are resolved
