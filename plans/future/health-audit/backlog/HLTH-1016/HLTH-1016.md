# HLTH-1016: Update @repo/gallery tests after shadcn data-testid → data-slot migration

## Context

The shadcn/ui upgrade replaced `data-testid` attributes with `data-slot` on primitive components. `@repo/gallery` has 44 test failures across 9 test files (`GalleryDataTable.test.tsx`, `.sorting.test.tsx`, `.multiSort.test.tsx`, `.composable.test.tsx`, `.animations.test.tsx`, and `GalleryCard` tests) because queries like `getByTestId('gallery-card-actions')` and `getByTestId('status-*')` find nothing.

## Goal

Update all failing test selectors in `@repo/gallery` to use semantic queries (`getByRole`, `getByLabelText`, `getByText`) or `data-slot` where semantic queries are insufficient.

## Non-Goals

- Not re-adding `data-testid` attributes to shadcn primitives
- Not changing the test coverage scope

## Scope

Key selectors to fix:
- `data-testid="gallery-card-actions"` → `getByRole('button', { name: /actions/i })`
- `data-testid="delete-confirm-delete"` → `getByRole('button', { name: /delete/i })`
- `data-testid="favorite-button"` → `getByRole('button', { name: /favorite/i })` or `getByLabelText`
- `data-testid="edit-button"` → `getByRole('button', { name: /edit/i })`
- `data-testid="status-*"` → query by `data-slot="badge"` or by status text content

Also audit sort state assertions and CSS class assertions that may have changed with the GalleryDataTable refactor.

## Acceptance Criteria

### AC1: All 44 previously-failing gallery tests pass
**When** `pnpm turbo test --filter=@repo/gallery` runs
**Then** ≥44 previously-failing tests now pass

### AC2: No new test failures introduced
**When** `pnpm turbo test --filter=@repo/gallery` runs
**Then** the total failure count does not increase

## Test Plan

- `pnpm turbo test --filter=@repo/gallery`
- Check that all 9 affected test files show PASS
