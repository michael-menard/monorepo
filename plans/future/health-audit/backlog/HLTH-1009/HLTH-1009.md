# HLTH-1009: Add missing upload mutation exports to @repo/api-client

## Context

`@repo/app-instructions-gallery` imports `useCreateUploadSessionMutation`, `useCompleteUploadSessionMutation`, and `useUploadThumbnailMutation` from `@repo/api-client` but these hooks do not exist in the package. This causes 3 TS2305 errors and several test failures.

## Goal

Add the three missing RTK Query mutation endpoints to the appropriate API slice(s) in `@repo/api-client` and export them from the package index.

## Non-Goals

- Not implementing the Lambda endpoint handler logic (API side)
- Only adding the RTK client-side hook definitions

## Scope

Add to the relevant API slice in `packages/core/api-client/src/rtk/`:
```typescript
export const { useCreateUploadSessionMutation } = uploadApi
export const { useCompleteUploadSessionMutation } = uploadApi
export const { useUploadThumbnailMutation } = galleryApi  // or uploadApi
```
Also export `CompleteUploadSessionResponse` type if missing.

## Acceptance Criteria

### AC1: No TS2305 for missing hooks
**When** `pnpm turbo check-types --filter=@repo/app-instructions-gallery` runs
**Then** zero `has no exported member` errors for the three hooks

### AC2: Hooks available at runtime
**When** the upload flow is used in `app-instructions-gallery`
**Then** all three mutation hooks are callable

## Test Plan

- `pnpm turbo check-types --filter=@repo/app-instructions-gallery`
- `pnpm turbo build --filter=@repo/api-client`
