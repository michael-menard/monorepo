# HLTH-1015: Fix minor single-package test issues

## Context

Four packages each have a single isolated test failure from unrelated causes:
- `@repo/db` — flush timer interval assertion mismatch
- `@repo/kbar-sync` — path traversal security check no longer throws
- `@repo/main-app` — router test relies on jsdom `window.location` navigation
- `@repo/upload` — text selector fails on split text nodes

## Goal

Fix all four failures. Each is a small, self-contained fix.

## Non-Goals

- Not changing the production behaviour of any of these packages

## Scope

**`@repo/db` — `src/telemetry-sdk/__tests__/init.test.ts > INIT-002`:**
Read the current flush interval value and update the assertion to match.

**`@repo/kbar-sync` — `scripts/__tests__/sync-story.test.ts > rejects path traversal`:**
Investigate why `validateFilePath` no longer throws. Restore the security check or update the test input to one that is still rejected.

**`@repo/main-app` — `src/routes/__tests__/router.test.ts`:**
Use `createMemoryHistory()` from `@tanstack/react-router` instead of relying on `window.location`:
```typescript
import { createMemoryHistory, createRouter } from '@tanstack/react-router'
const router = createRouter({ history: createMemoryHistory(), routeTree })
```

**`@repo/upload` — `src/components/InstructionsUpload/__tests__/InstructionsUpload.test.tsx`:**
Replace `screen.getByText('test.pdf')` with a custom text matcher:
```typescript
screen.getByText((_, el) => el?.textContent?.includes('test.pdf') ?? false)
```

## Acceptance Criteria

### AC1: All four packages pass their previously-failing tests
**When** `pnpm turbo test --filter=@repo/db --filter=@repo/kbar-sync --filter=@repo/main-app --filter=@repo/upload` runs
**Then** the previously-failing test cases pass
