# HLTH-1005: Prefix unused RTK callback variables in @repo/api-client

## Context

Several RTK Query `onQueryStarted` and `onCacheEntryAdded` callbacks in `@repo/api-client` destructure `error`, `meta`, and `result` but never use them. TypeScript reports TS6133 for each unused variable. Because these type errors live in a shared package, they replicate into all 8 consuming app packages during `check-types`.

## Goal

Prefix all unused destructured variables in the affected RTK files with `_` so TypeScript ignores them. This is a mechanical one-line-per-error fix.

## Non-Goals

- No functional changes to RTK Query behaviour
- Not removing the callbacks themselves

## Scope

**`packages/core/api-client/src/rtk/gallery-api.ts`**
- Line 178: `error` → `_error`

**`packages/core/api-client/src/rtk/wishlist-api.ts`**
- Lines 219, 326, 461, 536: `error` → `_error`
- Lines 307, 445, 520: `meta` → `_meta`
- Lines 326, 461, 536: `result` → `_result`

## Acceptance Criteria

### AC1: No TS6133 errors in api-client RTK files
**When** `pnpm turbo check-types --filter=@repo/api-client` runs
**Then** zero TS6133 errors in `gallery-api.ts` and `wishlist-api.ts`

### AC2: Downstream apps no longer report these errors
**When** `pnpm turbo check-types` runs for any consuming app
**Then** no TS6133 errors originate from `packages/core/api-client/src/rtk/`

## Test Plan

- `pnpm turbo check-types --filter=@repo/api-client`
- `pnpm turbo test --filter=@repo/api-client` — no new failures
