# HLTH-1002: Deduplicate react-hook-form version across workspace

## Context

Two versions of `react-hook-form` are installed: `7.66.1` and `7.71.1`. When packages reference `Control` from different versions, the type `_updateIsValidating` is missing from the older version, causing ~12 TS2741 errors in `@repo/app-instructions-gallery` and `@repo/main-app`.

## Goal

Pin the entire workspace to `react-hook-form@7.71.1` by adding a pnpm override, then run `pnpm install` to deduplicate.

## Non-Goals

- No API changes to form components
- Not a full react-hook-form upgrade review

## Scope

**Root `package.json`** — add pnpm overrides block:
```json
"pnpm": {
  "overrides": {
    "react-hook-form": "7.71.1"
  }
}
```
Then run `pnpm install`.

## Acceptance Criteria

### AC1: Single Version Installed
**Given** the override is applied and `pnpm install` has run
**When** `pnpm list react-hook-form` is run
**Then** only `7.71.1` appears

### AC2: Type Errors Cleared
**Given** the deduplication is complete
**When** `pnpm turbo check-types --filter=@repo/app-instructions-gallery --filter=@repo/main-app` is run
**Then** all `_updateIsValidating`-related TS2741 errors are gone

## Test Plan

- `pnpm turbo check-types --filter=@repo/main-app --filter=@repo/app-instructions-gallery`
- `pnpm turbo test --filter=@repo/main-app` — no new failures
