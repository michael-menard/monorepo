# HLTH-1001: Fix @repo/knowledge-base build failures

## Context

`@repo/knowledge-base` is the only package currently failing to build (57/58 pass). It has 7 TypeScript errors across 4 files, all introduced by recent script additions. This is a P1 blocker — the knowledge-base MCP server cannot be deployed until the build is clean.

## Goal

Resolve all 7 TypeScript build errors in `@repo/knowledge-base` so that `pnpm turbo build` exits 0 for all 58 packages.

## Non-Goals

- No functional changes to MCP tool behaviour
- No refactoring beyond the minimum needed to fix the errors

## Scope

### Files to Change

| File | Error | Fix |
|------|-------|-----|
| `apps/api/knowledge-base/src/mcp-server/tool-handlers.ts:3719–3720` | `Property 'story' does not exist` — result exposes `.stories[]` not `.story` | Replace `.story.id` / `.story.title` with `.stories[0]?.id` / `.stories[0]?.title` |
| `apps/api/knowledge-base/src/scripts/db-init.ts:79` | `Cannot find name 'readFileSync'` | Add `import { readFileSync } from 'fs'` |
| `apps/api/knowledge-base/src/scripts/migrate-artifacts-to-kb.ts:102` | `'STAGES' declared but never read` | Prefix with `_STAGES` or delete |
| `apps/api/knowledge-base/src/scripts/seed-kb-first-stories.ts:196` | `'result' declared but never read` | Remove the `const result =` assignment; call the expression directly |
| `apps/api/knowledge-base/src/scripts/seed-kb-stories.ts:160` | Property `stage`/`dir` does not exist on type `never` | Add type guard before access; check return type of function on ~line 155 |

## Acceptance Criteria

### AC1: Build Passes
**Given** the fixes are applied
**When** `pnpm turbo build --filter=@repo/knowledge-base` is run
**Then** the command exits 0 with no TypeScript errors

### AC2: Full Build Still Passes
**Given** HLTH-1001 is merged
**When** `pnpm turbo build --filter='!@repo/playwright'` is run
**Then** all 58 packages build successfully (0 failures)

### AC3: No Functional Regression
**Given** the tool-handlers change
**When** an MCP tool that returns story results is called
**Then** the correct story data is returned (`.stories[0]` instead of `.story`)

## Test Plan

- Run `pnpm turbo build --filter=@repo/knowledge-base` — must exit 0
- Run `pnpm turbo test --filter=@repo/knowledge-base` — must not introduce new failures
