# HLTH-1004: Restore Zod exports in QuotaIndicator.tsx

## Context

`packages/core/app-component-library/src/indicators/QuotaIndicator.tsx` imports `z` from `zod` but does not use it (TS6133 lint error). Meanwhile `src/indicators/index.ts` re-exports `QuotaTypeSchema`, `QuotaInfoSchema`, `QuotaIndicatorPropsSchema`, `QuotaBarPropsSchema`, `QuotaCardPropsSchema`, and `QUOTA_DISPLAY_NAMES` — none of which are exported from `QuotaIndicator.tsx`. This causes 8 TS errors and 1 lint error that fan out to all 7 consuming app packages.

Per `CLAUDE.md`, all types must be Zod-first. The Zod schemas were likely removed during a recent refactor.

## Goal

Restore the Zod schema exports in `QuotaIndicator.tsx` and align `src/indicators/index.ts` re-exports so everything compiles cleanly.

## Non-Goals

- No changes to the visual behaviour of `QuotaIndicator`
- Not adding new quota types beyond the existing set

## Scope

In `packages/core/app-component-library/src/indicators/QuotaIndicator.tsx`:
```typescript
import { z } from 'zod'

export const QuotaTypeSchema = z.enum(['upload', 'storage', 'api'])
export type QuotaType = z.infer<typeof QuotaTypeSchema>

export const QuotaInfoSchema = z.object({

  limit: z.number(),
  type: QuotaTypeSchema,

export type QuotaInfo = z.infer<typeof QuotaInfoSchema>

export const QUOTA_DISPLAY_NAMES: Record<QuotaType, string> = {
  upload: 'Uploads',
  storage: 'Storage',
  api: 'API Calls',
}

export const QuotaIndicatorPropsSchema = z.object({ ... })
```
