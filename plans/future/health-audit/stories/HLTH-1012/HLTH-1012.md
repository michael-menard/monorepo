# HLTH-1012: Fix vi.mock factories missing useUpdateItemPurchaseMutation

## Context

`@repo/app-wishlist-gallery` has ~9 test failures in `main-page.datatable.test.tsx` and related files because `vi.mock('@repo/api-client/rtk/wishlist-gallery-api', ...)` factories do not include `useUpdateItemPurchaseMutation`. When tests render components that call this hook, they receive `undefined` instead of `[vi.fn(), { isLoading: false }]`.

## Goal

Update all `vi.mock` factories in `@repo/app-wishlist-gallery` tests to include the missing mutation hook.

## Non-Goals

- Not changing the actual hook implementation
- Not adding new tests — only fixing existing mock setup

## Scope

In affected test files, update the mock factory:
```typescript
vi.mock('@repo/api-client/rtk/wishlist-gallery-api', () => ({
  useGetWishlistItemsQuery: vi.fn().mockReturnValue({ data: [], isLoading: false }),
  useUpdateItemPurchaseMutation: vi.fn().mockReturnValue([vi.fn(), { isLoading: false }]),
  useDeleteWishlistItemMutation: vi.fn().mockReturnValue([vi.fn(), { isLoading: false }]),
  // ... all other hooks used in the component tree
}))
```

**Files to update:** Search `apps/web/app-wishlist-gallery/src/**/__tests__/*.test.tsx` for `vi.mock` blocks missing `useUpdateItemPurchaseMutation`.

## Acceptance Criteria

### AC1: No undefined hook errors in wishlist tests
**When** `pnpm turbo test --filter=@repo/app-wishlist-gallery` runs
**Then** test failures caused by missing `useUpdateItemPurchaseMutation` mock are resolved
