# HLTH-1018: Fix @repo/knowledge-base test DB schema issues

## Context

`@repo/knowledge-base` has 11 test failures caused by test DB schema gaps:
1. pgvector IVFFlat index is missing from the test DB setup, causing 2 vector index tests to fail
2. `task_audit_log.task_id` FK uses `SET NULL` instead of `CASCADE`, causing a cascade constraint failure in 1 test
3. `?|` PostgreSQL jsonb/array operator unavailable, causing 6 KB list tests to fail
4. Embedding cache tests fail because test DB is not seeded

## Goal

Fix the test database setup so all 11 knowledge-base tests pass.

## Non-Goals

- Not changing production DB migration files (only test setup)
- Not changing the test logic itself

## Scope

1. **pgvector index**: Ensure the test DB migration runner includes the `CREATE INDEX ... USING ivfflat` statement
2. **FK cascade**: Update `task_audit_log` FK from `ON DELETE SET NULL` to `ON DELETE CASCADE` in the migration files
3. **`?|` operator**: Ensure `pg_trgm` and `pgvector` extensions are created in test DB setup
4. **Embedding cache seeding**: Add seed data for embedding cache in the test setup script

## Acceptance Criteria

### AC1: Vector index tests pass
**When** `pnpm turbo test --filter=@repo/knowledge-base` runs
**Then** `smoke.test.ts > Vector Index` tests pass

### AC2: Tasks table cascade test passes
**When** `pnpm turbo test --filter=@repo/knowledge-base` runs
**Then** `smoke.test.ts > Tasks Table` test passes

### AC3: KB list tests pass
**When** `pnpm turbo test --filter=@repo/knowledge-base` runs
**Then** all 6 `crud-operations/__tests__/kb-list.test.ts` tests pass
