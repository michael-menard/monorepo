# HLTH-1010: Add VITE_SERVERLESS_API_BASE_URL to Vitest env config

## Context

`@repo/api-client`'s `environments.ts` throws at module import time when `VITE_SERVERLESS_API_BASE_URL` is not defined. This causes test suite crashes in `@repo/app-component-library` (3 test files) and `@repo/reset-password` (1 test file) with `Error: VITE_SERVERLESS_API_BASE_URL is required`.

## Goal

Provide a fallback env var value in the Vitest configuration for all affected packages so that tests can import `@repo/api-client` without crashing.

## Non-Goals

- Not changing the runtime validation logic in `environments.ts`

## Scope

In `packages/core/app-component-library/vitest.config.ts`:
```typescript
test: {
  env: {
    VITE_SERVERLESS_API_BASE_URL: 'http://localhost:3000',
  },
}
```

In `apps/web/reset-password/vitest.config.ts`:
```typescript
test: {
  env: {
    VITE_SERVERLESS_API_BASE_URL: 'http://localhost:3000',
  },
}
```

## Acceptance Criteria

### AC1: No import-time crash in app-component-library tests
**When** `pnpm turbo test --filter=@repo/app-component-library` runs
**Then** no test file crashes with `VITE_SERVERLESS_API_BASE_URL is required`

### AC2: reset-password tests no longer crash on import
**When** `pnpm turbo test --filter=@repo/reset-password` runs
**Then** the crash is gone

## Test Plan

- `pnpm turbo test --filter=@repo/app-component-library`
- `pnpm turbo test --filter=@repo/reset-password`
