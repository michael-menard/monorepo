# HLTH-1017: Update wishlist/sets-gallery/app-dashboard tests for data-testid changes

## Context

Following the shadcn upgrade, ~20 tests across three packages fail due to broken `data-testid` selectors:

- `@repo/app-wishlist-gallery`: `delete-confirm-delete`, `wishlist-drag-preview`, screen-reader instruction elements
- `@repo/app-sets-gallery`: `set-card-action-view`
- `@repo/app-dashboard`: `QuickActions` button count (expects 3, gets 1) and button text ("Browse Gallery" not found)

## Goal

Update all broken selectors and assertions across the three packages to match current component output.

## Non-Goals

- Not restoring removed buttons or changing component behaviour

## Scope

**`@repo/app-wishlist-gallery`:**
- `delete-confirm-delete` → `getByRole('button', { name: /delete/i })`
- `wishlist-drag-preview` → re-add `data-testid` to `WishlistDragPreview` component, or query by `aria-label`
- Screen-reader instruction elements → query by accessible text

**`@repo/app-sets-gallery`:**
- `set-card-action-view` → `getByRole('button', { name: /view/i })` or re-add `data-testid`

**`@repo/app-dashboard`:**
- Read current `QuickActions` component to see how many buttons render and what their text is
- Update assertions accordingly

## Acceptance Criteria

### AC1: All ~20 selector-related failures resolved
**When** `pnpm turbo test --filter=@repo/app-wishlist-gallery --filter=@repo/app-sets-gallery --filter=@repo/app-dashboard` runs
**Then** all previously-broken `data-testid` assertion failures pass

## Test Plan

- `pnpm turbo test --filter=@repo/app-wishlist-gallery`
- `pnpm turbo test --filter=@repo/app-sets-gallery`
- `pnpm turbo test --filter=@repo/app-dashboard`
