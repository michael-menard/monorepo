# HLTH-1014: Fix @repo/app-instructions-gallery test infrastructure

## Context

`@repo/app-instructions-gallery` has ~15 test failures from three distinct infrastructure problems:
1. MSW is missing handlers for `/mocs/:mocId` (unhandled request errors)
2. `dataTransfer` is not implemented in jsdom (drag event tests crash with `Cannot set properties of undefined`)
3. `Worker is not defined` in jsdom when HEIC library initialises at import time

## Goal

Fix all three test infrastructure issues so that integration and unit tests run cleanly.

## Non-Goals

- Not testing actual drag-and-drop UX in jsdom (only fixing the crash)
- Not fixing HEIC conversion logic

## Scope

**1. Add MSW handlers** in `apps/web/app-instructions-gallery/src/test/setup.ts`:
```typescript
server.use(
  http.get('/mocs/:mocId', ({ params }) => HttpResponse.json(mockMocData)),
)
```

**2. Mock `dataTransfer`** in test setup or individual tests:
```typescript
Object.defineProperty(window, 'DataTransfer', {
  value: class { effectAllowed = ''; dropEffect = '' },
})
```

**3. Mock HEIC `Worker`** in test setup:
```typescript
global.Worker = class { postMessage() {} addEventListener() {} } as any
```

## Acceptance Criteria

### AC1: No unhandled MSW request errors
**When** `pnpm turbo test --filter=@repo/app-instructions-gallery` runs
**Then** no `MSW: unhandled request` warnings cause test failures

### AC2: No dataTransfer crash
**When** drag event tests run
**Then** no `Cannot set properties of undefined (setting 'effectAllowed')` errors

### AC3: No Worker crash
**When** HEIC-related test files run
**Then** no `Worker is not defined` ReferenceError
