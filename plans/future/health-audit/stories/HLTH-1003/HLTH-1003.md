# HLTH-1003: Fix import.meta.env type errors in @repo/api-client

## Context

`@repo/api-client` uses `import.meta.env.VITE_*` variables in `src/config/environments.ts` and several RTK files. The package's `tsconfig.json` does not include `"vite/client"` in its types, so TypeScript reports `Property 'env' does not exist on type 'ImportMeta'` (TS2339). This error fans out to 9+ consuming app packages.

## Goal

Add the `vite/client` type reference to `@repo/api-client`'s tsconfig so that `import.meta.env` is correctly typed. This is a one-line fix that clears errors across the entire consuming tree.

## Non-Goals

- No changes to how env vars are read at runtime
- Not replacing `import.meta.env` with `process.env`

## Scope

**Option A (recommended):** Add to `packages/core/api-client/tsconfig.json`:
```json
{
  "compilerOptions": {
    "types": ["vite/client"]
  }
}
```

**Option B:** Add `/// <reference types="vite/client" />` to `packages/core/api-client/src/env.d.ts`.

**Affected files with errors:**
- `src/config/environments.ts:24,25,26,27,28,29`
- `src/rtk/gallery-api.ts:123`
- `src/rtk/instructions-api.ts:82`
- `src/rtk/wishlist-api.ts:151`

## Acceptance Criteria

### AC1: No import.meta.env errors in api-client
**When** `pnpm turbo check-types --filter=@repo/api-client` runs
**Then** zero TS2339 errors on `import.meta.env`

### AC2: Downstream packages cleared
**When** `pnpm turbo check-types --filter=@repo/app-component-library` runs
**Then** all `import.meta.env` TS2339 errors are gone (was 9 errors)
