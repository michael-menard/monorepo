# HLTH-1020: Auto-fix remaining lint errors across 8 packages

## Context

After HLTH-1019, 8 packages still have lint errors totalling ~89 issues. Most are auto-fixable by `eslint --fix` (import ordering, prettier formatting). The rest are manual: unused variables, `no-console` violations, and `\Z` regex escapes.

## Goal

Bring all 8 packages to zero lint errors using a combination of `eslint --fix` and targeted manual fixes.

## Non-Goals

- Not changing logic or behaviour

## Scope

**Auto-fixable (run `pnpm lint --fix`):**
- `@repo/api-core`: 5 `prettier/prettier` errors
- `@repo/api-client`: 1 `prettier/prettier` error
- `@repo/main-app`: some `import/order` errors
- `@repo/knowledge-base`: `import/order` and `import/no-duplicates`

**Manual fixes:**
| Package | Error | Fix |
|---------|-------|-----|
| `@repo/lego-api` | 18 `no-unused-vars` (type imports) | Remove unused type imports |
| `@repo/app-instructions-gallery` | 26 `no-unused-vars` in `upload-page-fix.tsx` | Delete file or complete it on a branch |
| `@repo/main-app` | `no-console` | Replace `console.log` with `logger.info` from `@repo/logger` |
| `@repo/upload-types` | `no-console` | Replace `console.log` with `logger.info` |
| `@repo/knowledge-base` | `\Z` regex escape | Replace `\Z` with `$` (JS doesn't support `\Z`) |

## Acceptance Criteria

### AC1: All 8 packages have 0 lint errors
**When** `pnpm turbo lint --filter=@repo/api-core --filter=@repo/api-client --filter=@repo/main-app --filter=@repo/knowledge-base --filter=@repo/lego-api --filter=@repo/app-instructions-gallery --filter=@repo/upload-types` runs
**Then** exit code is 0 for every package

## Test Plan

- `pnpm turbo lint --filter='!@repo/playwright'`
- All packages report 0 errors
