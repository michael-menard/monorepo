# HLTH-1007: Register missing TanStack Router routes in @repo/main-app

## Context

`@repo/main-app` uses TanStack Router with type-safe `Link` components. Sub-app components reference routes like `/activity`, `/parts`, `/mocs/$mocId`, `/add`, and `/collection` that are not registered in the root router tree, causing ~8 TS2322 type errors.

## Goal

Register all missing routes in the main-app TanStack Router route tree (or update sub-app `Link` components to use their own scoped router) so all TS2322 route-type errors are resolved.

## Non-Goals

- Not implementing the page content for new routes if they are stubs
- Not changing the navigation UX

## Scope

Investigate each missing route:
- `/activity` — used in `app-dashboard` sub-app
- `/parts` — parts list route
- `/mocs/$mocId` — MOC detail
- `/add` — used in `app-wishlist-gallery`
- `/collection` — collection view

For each: either add the route to `apps/web/main-app/src/routes/` and register it in the route tree, or update the `Link` component to not depend on root-app route types.

## Acceptance Criteria

### AC1: No route TS2322 errors
**When** `pnpm turbo check-types --filter=@repo/main-app` runs
**Then** zero TS2322 errors related to route path strings

## Test Plan

- `pnpm turbo check-types --filter=@repo/main-app`
- Manual smoke test: navigation to each newly registered route renders without crash
