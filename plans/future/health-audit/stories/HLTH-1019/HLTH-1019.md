# HLTH-1019: Fix @repo/accessibility-testing ESLint env configuration

## Context

`@repo/accessibility-testing` has 55 lint errors, all from the `no-undef` rule. Symbols like `Element`, `MutationObserver`, `KeyboardEvent`, `document`, `window`, and `require` are reported as undefined because the ESLint config for this package does not declare `env: { browser: true, node: true }`.

## Goal

Add the browser and node environment declarations to the ESLint config for `@repo/accessibility-testing`. This is a one-line config change.

## Non-Goals

- No source code changes

## Scope

In `packages/core/accessibility-testing/` — update `eslint.config.js` (or `.eslintrc.json`):
```json
{
  "env": {
    "browser": true,
    "node": true
  }
}
```

## Acceptance Criteria

### AC1: All 55 no-undef errors gone
**When** `pnpm turbo lint --filter=@repo/accessibility-testing` runs
**Then** zero errors reported
