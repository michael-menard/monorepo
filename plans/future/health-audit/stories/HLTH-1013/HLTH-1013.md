# HLTH-1013: Fix @repo/app-dashboard logger mock and QuickActions test assertions

## Context

`@repo/app-dashboard` has 6 test failures:
1. `App.test.tsx` (3 failures) — `vi.mock('@repo/logger')` does not export `createLogger`, causing `TypeError: createLogger is not a function`
2. `QuickActions.test.tsx` (3 failures) — assertions expect 3 buttons and "Browse Gallery" text, but the component now renders 1 button with different text

## Goal

Fix both issues: update the logger mock to include `createLogger`, and update the QuickActions assertions to match the current component output.

## Non-Goals

- Not adding new QuickActions buttons
- Not changing the logger package itself

## Scope

**`apps/web/app-dashboard/src/App.test.tsx`:**
```typescript
vi.mock('@repo/logger', () => ({
  logger: { info: vi.fn(), error: vi.fn(), warn: vi.fn(), debug: vi.fn() },
  createLogger: vi.fn().mockReturnValue({ info: vi.fn(), error: vi.fn(), warn: vi.fn(), debug: vi.fn() }),
}))
```

**`apps/web/app-dashboard/src/components/__tests__/QuickActions.test.tsx`:**
- Read the current `QuickActions` component to determine actual button count and labels
- Update test assertions to match

## Acceptance Criteria

### AC1: App.test.tsx passes
**When** `pnpm turbo test --filter=@repo/app-dashboard` runs
**Then** all 3 `App.test.tsx` failures are resolved

### AC2: QuickActions assertions match component
**When** `pnpm turbo test --filter=@repo/app-dashboard` runs
**Then** QuickActions tests pass with no assertion mismatches
