# HSKP-2008: Consolidate Wishlist Schemas to Single Source

## Context

Wishlist Zod schemas exist in two locations that have drifted out of sync:
- `packages/core/api-client/src/schemas/wishlist.ts` — used by the frontend
- `apps/api/endpoints/wishlist/schemas/index.ts` — used by the Lambda handlers

Because they evolved independently, there are field-level inconsistencies: `userId` is typed as `z.string().uuid()` in one location but the Cognito sub is not guaranteed to be a UUID; `createdAt`/`updatedAt` use different representations; `store` and `currency` have no enum constraints in either location.

This duplication means frontend and backend can silently diverge — a field renamed in one location causes a type error in neither until runtime. The canonical source should be `@repo/api-client`, which is already shared across the monorepo.

This story depends on HSKP-2007 being complete, so handlers are functioning before we change their schema imports.

---

## Goal

Consolidate all wishlist Zod schemas to `@repo/api-client` as the single canonical source, resolve all field-level inconsistencies, and update all handler imports to reference the shared package.

---

## Non-Goals

- Changing the database schema or Drizzle models
- Changing any API response shapes (the fix must be transparent to API consumers)
- Adding new schemas for domains other than wishlist
- Migrating other duplicate schemas in the codebase (this story is wishlist-only)

---

## Scope

### Schema Consolidation Target
`packages/core/api-client/src/schemas/wishlist.ts` becomes the single source of truth.

### Field Alignment

| Field | Current (frontend) | Current (backend) | Resolution |
|-------|-------------------|-------------------|------------|
| `userId` | `z.string().uuid()` | `z.string()` | Use `z.string()` — Cognito sub is not always a UUID |
| `createdAt` | `z.date()` | `z.string()` | Use `z.string().datetime()` — JSON serialises as ISO string |
| `updatedAt` | `z.date()` | `z.string()` | Use `z.string().datetime()` — same as above |
| `store` | `z.string()` | `z.string()` | Add `z.enum([...storeValues])` schema |
| `currency` | `z.string()` | `z.string()` | Add `z.enum([...currencyValues])` schema |

### Changes Required
1. **Update `@repo/api-client` schemas**: Apply field alignment table above. Add `StoreSchema` and `CurrencySchema` enum schemas.
2. **Delete `apps/api/endpoints/wishlist/schemas/index.ts`**: Remove the duplicate schema file.
3. **Update all 8 wishlist handler imports**: Replace `../schemas` with `@repo/api-client`.
4. **Update frontend imports**: Verify frontend components and RTK Query hooks reference `@repo/api-client` (should already be the case, but confirm).
5. **Type-check the entire monorepo**: Run `pnpm check-types:all` to confirm no regressions.
