# HSKP-2003: Drizzle MCP Server

## Context

When Claude Code generates database queries, handler code, or Zod schemas, it needs accurate knowledge of the database schema — table names, column names, types, constraints, and relationships. Without this, the AI makes assumptions that lead to incorrect field names, missing required columns, and type mismatches.

The Drizzle MCP Server exposes the monorepo's Drizzle ORM schema files as structured, queryable tools that Claude Code can call in real time. This gives the AI ground-truth schema information, dramatically improving the accuracy of generated code.

This story depends on HSKP-2002 (MCP Server Infrastructure) for the server factory, caching, and error handling foundation.

---

## Goal

Build a Drizzle MCP Server that exposes four tools for querying database schema information, with sub-200ms response times for cached queries and file-watching cache invalidation when schema files change.

---

## Non-Goals

- Executing SQL queries against the database
- Generating migration files
- Modifying the Drizzle schema
- Supporting non-Drizzle ORM schema formats
- Live database introspection — source of truth is the Drizzle schema TypeScript files, not the running DB

---

## Scope

### MCP Tools

**`list_tables`**
- Returns all table names defined in the Drizzle schema
- Includes column count per table
- Response shape: `{ tables: Array<{ name: string, columnCount: number }> }`

**`get_table_schema`**
- Accepts a table name
- Returns complete column definitions: name, Drizzle type, TypeScript type, nullable, default value, primary key flag, unique flag
- Response shape: `{ table: string, columns: Array<ColumnDef> }`

**`get_relations`**
- Accepts a table name
- Returns all foreign key relationships for that table: source column, target table, target column, relation type (one-to-one, one-to-many, many-to-many)
- Response shape: `{ table: string, relations: Array<RelationDef> }`

**`get_zod_schema`**
- Accepts a table name
- Generates and returns Zod schema TypeScript code for that table (insert schema and select schema)
- Useful for scaffolding — the generated code can be pasted directly into a handler
- Response shape: `{ table: string, zodSchema: string }`

### Server Location
`tools/mcp-servers/drizzle/`

### Caching
- All four tools are cached using the `@repo/mcp-shared` cache
- Cache is invalidated by file-watching on the Drizzle schema directory (`apps/api/*/schema.ts` or equivalent glob)
- Target: sub-200ms for all cached responses

## Acceptance Criteria

- [ ] **AC1**: `list_tables` returns an array of all tables defined in the Drizzle schema files
- [ ] **AC2**: Each entry includes the table name and a correct column count
- [ ] **AC3**: Response is returned in under 200ms when cached

### Tool: get_table_schema
- [ ] **AC4**: `get_table_schema({ table: "wishlist_items" })` returns all columns for that table
- [ ] **AC5**: Each column entry includes: `name`, `drizzleType`, `tsType`, `nullable`, `hasDefault`, `isPrimaryKey`, `isUnique`
- [ ] **AC6**: Returns a `McpToolError` with a clear message when the table name does not exist
- [ ] **AC7**: Response is returned in under 200ms when cached

### Tool: get_relations
- [ ] **AC8**: `get_relations({ table: "wishlist_items" })` returns all foreign key relationships for that table
- [ ] **AC9**: Each relation includes: `sourceColumn`, `targetTable`, `targetColumn`, `relationType`
- [ ] **AC10**: Returns an empty relations array (not an error) for tables with no relations
- [ ] **AC11**: Response is returned in under 200ms when cached

### Tool: get_zod_schema
- [ ] **AC12**: `get_zod_schema({ table: "wishlist_items" })` returns valid TypeScript code defining Zod insert and select schemas
- [ ] **AC13**: The generated Zod schema compiles without TypeScript errors when pasted into a `.ts` file
- [ ] **AC14**: Column types are mapped correctly: `text` → `z.string()`, `integer` → `z.number().int()`, `boolean` → `z.boolean()`, `timestamp` → `z.string().datetime()`, etc.
- [ ] **AC15**: Response is returned in under 200ms when cached

### Cache Invalidation
- [ ] **AC16**: Modifying a Drizzle schema file invalidates all cached responses within 1 second
- [ ] **AC17**: The next request after invalidation re-parses the schema files and returns up-to-date data

### Claude Code Integration

- [ ] **AC19**: A sample Claude Code prompt ("What columns does the wishlist_items table have?") returns accurate data from `get_table_schema`

## Implementation Notes

- Parse Drizzle schema using TypeScript AST (via `ts-morph` or `typescript` compiler API) rather than importing/executing the schema files — this avoids needing a DB connection at parse time
- Alternatively, use Drizzle's own introspection utilities if they expose programmatic schema access
- The Zod schema generator for `get_zod_schema` should follow the same patterns as the existing Zod schemas in the codebase (see CLAUDE.md: always use `z.infer<>`)
- Column type mapping table should be exhaustive — log a warning for unmapped types rather than silently dropping columns
