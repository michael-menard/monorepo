# HSKP-2007: Fix Redis Dependency in Wishlist Handlers

## Context

During the WKFL-020 QA review, a runtime crash was identified in all wishlist handlers. Every wishlist handler imports and calls `invalidateWishlistCaches()`, which internally calls `getRedisClient()`. Redis was disabled in the current deployment — the `REDIS_URL` environment variable is not set — which causes `getRedisClient()` to throw an error at runtime the moment any wishlist endpoint is invoked.

This means every wishlist API call crashes with an unhandled error rather than executing the actual business logic. All 8 wishlist handlers are affected.

---

## Goal

Remove the Redis cache invalidation dependency from all 8 wishlist handlers so that wishlist endpoints function correctly without Redis being available, eliminating the runtime crash.

---

## Non-Goals

- Implementing a working Redis cache for wishlist data (Redis is intentionally disabled)
- Any changes to wishlist business logic, database queries, or response shapes
- Adding new caching mechanisms to replace Redis
- Changes to other domains (MOC, Gallery, Parts List)

---

## Scope

### Affected Files
All 8 wishlist handler files that call `invalidateWishlistCaches()`:
1. `apps/api/endpoints/wishlist/create-item/handler.ts`
2. `apps/api/endpoints/wishlist/update-item/handler.ts`
3. `apps/api/endpoints/wishlist/delete-item/handler.ts`
4. `apps/api/endpoints/wishlist/reorder/handler.ts`
5. `apps/api/endpoints/wishlist/upload-image/handler.ts`
6. `apps/api/endpoints/wishlist/search/handler.ts`
7. `apps/api/endpoints/wishlist/list/handler.ts`
8. `apps/api/endpoints/wishlist/get-item/handler.ts`

### Solution (Option A — Recommended)
Remove `invalidateWishlistCaches()` calls entirely from all 8 handlers. Remove the import of the Redis client from these files. If `invalidateWishlistCaches()` is not used anywhere else after this change, remove the function definition as well.

### Solution (Option B — Fallback)
If complete removal is not safe (e.g., the function has other callers), replace with a no-op stub that logs a `debug` message:
```typescript
async function invalidateWishlistCaches(): Promise<void> {
  logger.debug('Cache invalidation skipped: Redis not configured')
}
```
