# LNGG-0080: Workflow Command Integration - Connect Adapters to LangGraph Commands

## Context

**Background:**
LNGG-001 through LNGG-006 implemented six core adapters for LangGraph workflows:
1. **Story File Adapter** (YAML read/write/parse) - `packages/backend/orchestrator/src/adapters/story-file-adapter.ts`
2. **Index Management Adapter** (stories.index.md updates) - `packages/backend/orchestrator/src/adapters/index-adapter.ts`
3. **Decision Callback System** (interactive user prompts) - `packages/backend/orchestrator/src/adapters/decision-callbacks/`
4. **Stage Movement Adapter** (directory-based stage transitions) - `packages/backend/orchestrator/src/adapters/stage-movement-adapter.ts`
5. **KB Writing Adapter** (knowledge base integration) - `packages/backend/orchestrator/src/adapters/kb-writer/`
6. **Checkpoint Adapter** (resume support) - `packages/backend/orchestrator/src/adapters/checkpoint-adapter.ts`

LNGG-0070 validated these adapters work correctly through comprehensive integration tests:
- ✅ 48/48 tests passing
- ✅ 96.5% test coverage
- ✅ All atomic operations verified
- ✅ Backward compatibility validated

**Current Problem:**
The actual workflow commands in `.claude/commands/` do NOT yet use these adapters:
- `/dev-implement-story` - Main implementation workflow orchestrator
- `/qa-verify-story` - Post-implementation verification workflow
- `/story-move` - Story stage transition utility
- `/checkpoint` - Checkpoint management
- `/index-update` - Index management
- `/pm-story` - Story generation workflow
- `/elab-story` - Story elaboration workflow

These commands duplicate adapter logic or use manual file operations, missing:
- ❌ Standardized interface for story file operations
- ❌ Validation, atomic writes, and error handling
- ❌ Checkpoint resume capability
- ❌ Decision callbacks for autonomous mode
- ❌ KB writing for deferred insights

**Impact:**
- Increased maintenance burden (duplicate file handling logic)
- Risk of data corruption (no atomic operations in commands)
- Cannot leverage tested adapter functionality
- Blocks full LangGraph workflow automation
- Integration gaps prevent workflow reliability

## Goal

Create LangGraph workflow nodes that integrate adapters with workflow commands, establishing a bridge layer that:
1. Connects workflow commands to `StoryFileAdapter` for story YAML operations
2. Integrates `IndexAdapter` for stories.index.md updates during stage transitions
3. Wires `StageMovementAdapter` to `/story-move` command
4. Connects `CheckpointAdapter` to checkpoint read/write operations
5. Integrates `DecisionCallback` system for user prompts in autonomous modes
6. Optionally connects `KBWriterAdapter` for deferred KB writes

This allows commands to leverage validated, tested, atomic adapter operations instead of manual file handling.

## Architecture Notes

### Node Wrapper Pattern
Each adapter gets wrapped in a LangGraph node following this pattern:

```typescript
// Example: story-file-node.ts
import { Annotation } from '@langchain/langgraph'
import { StoryFileAdapter } from '../adapters/story-file-adapter.js'
import { logger } from '@repo/logger'

// Define state annotation
const StoryFileState = Annotation.Root({
  story_path: Annotation<string>(),
  story_data: Annotation<any>(),
  error: Annotation<Error | null>(),
})

// Create node function
export async function storyFileNode(state: typeof StoryFileState.State) {
  try {
    const adapter = new StoryFileAdapter()
    const storyData = await adapter.read(state.story_path)

    logger.info('Story file read successfully', { story_path: state.story_path })

    return { story_data: storyData, error: null }
  } catch (error) {
    logger.error('Story file read failed', { error, story_path: state.story_path })
    return { error: error as Error }
  }
}

// Export state annotation for graph composition
export { StoryFileState }
```

### State Flow Design
Workflow commands compose multiple adapter nodes:

```typescript
// Example: dev-implement-story graph composition
const graph = new StateGraph(WorkflowState)
  .addNode('load_story', storyFileNode)
  .addNode('load_checkpoint', checkpointNode)
  .addNode('update_index', indexNode)
  .addNode('move_stage', stageMovementNode)
  .addEdge(START, 'load_story')
  .addEdge('load_story', 'load_checkpoint')
  // ... conditional edges based on checkpoint data
```

### Error Handling Strategy
1. **Adapter-level errors** - Typed errors (StoryNotFoundError, InvalidYAMLError, etc.) thrown by adapters
2. **Node-level errors** - Caught and returned in state.error field
3. **Graph-level errors** - Conditional edges route to error handling nodes
4. **Command-level errors** - Final error reporting to user via logger

### Testing Strategy
1. **Unit tests** - Test each node wrapper in isolation (mock adapter)
2. **Integration tests** - Test nodes with real adapters in temp directories
3. **E2E tests** - Test full workflow command execution with adapter nodes
4. **Backward compatibility tests** - Test with existing story files (legacy + v2 formats)
