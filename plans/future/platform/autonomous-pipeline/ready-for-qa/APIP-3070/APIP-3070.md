# APIP-3070 — Cold Start Bootstrapping and Exploration Budget

## Context

The autonomous pipeline's Learning-Aware Model Router (APIP-3040) delivers empirically-guided model routing once `wint.model_affinity` accumulates high-confidence profiles (`sample_size >= 20`). However, on day one before any telemetry flows through the system, every profile has `confidence_level = 'none'` and `sample_size = 0`.

Without an explicit cold-start strategy, the pipeline would fall back to the static Ollama → OpenRouter → Anthropic escalation chain for 100% of dispatches — potentially routing to Ollama (cheapest but least reliable) for all code generation tasks where Ollama has no empirical track record.

APIP-3070 is the twelfth and final story on the critical path. It adds two complementary mechanisms on top of APIP-3040's three-tier routing:

1. **Conservative OpenRouter defaults** — When `wint.model_affinity` has zero qualifying profiles, the router short-circuits to a pre-configured conservative OpenRouter model, bypassing the static escalation chain and ensuring reliable generation from day one.
2. **10% random Ollama exploration budget** — A configurable fraction of dispatches are routed to Ollama to accumulate telemetry data, accelerating the ramp-up to high-confidence affinity profiles.
3. **Optional manual profile seeding** — Operators may pre-populate `wint.model_affinity` from external benchmark data to bootstrap the affinity system faster. Always optional.
4. **Profile confidence gating** — The routing entry point to tier 2 (affinity-guided) gates on `confidence_level IN ('medium', 'high')`, clarifying the transition from exploration mode to affinity mode.

**No cold-start mechanism, exploration budget, or seeding capability exists anywhere in the pipeline today.**

## Acceptance Criteria

- [ ] **AC-1**: `PipelineModelRouterConfig` (Zod schema, post-APIP-3040) is extended with the following new named-constant fields: `conservativeOpenRouterModel: z.string().min(1).default('openrouter/anthropic/claude-3-haiku')`, `explorationBudgetFraction: z.number().min(0).max(1).default(0.1)`, `explorationMinSuccessRateFloor: z.number().min(0).max(1).default(0.3)`, `manualSeedEnabled: z.boolean().default(false)`, and optional `randomFn: z.function().optional()` for PRNG injection. No TypeScript interfaces. All values are injectable via config for tests.

- [ ] **AC-2**: `PipelineModelRouter.dispatch()` implements cold-start detection before affinity lookup: if `wint.model_affinity` has zero rows with `confidence_level IN ('medium', 'high')` for all `(change_type, file_type)` combinations (the affinity cache is empty or all profiles are `'none'`/`'low'`), the router short-circuits directly to the conservative OpenRouter default (tier 4) without executing tier 2 or tier 3. A lazy-evaluated, cache-invalidation-aware flag `_hasAnyHighConfidenceProfile` is used to avoid a full-table scan on every dispatch during cold-start.

- [ ] **AC-3**: The Profile Confidence Gate is implemented as the routing entry point to tier 2: a profile is considered qualifying only when both `confidence_level IN ('medium', 'high')` AND `sample_size >= affinityMinSampleSize` (from APIP-3040 config). If no qualifying profile exists for the dispatched `(change_type, file_type)`, the router proceeds to tier 3 (exploration) rather than falling through to tier 4 directly.

- [ ] **AC-4**: The 10% exploration budget is implemented as a per-ChangeSpec-attempt probabilistic gate in tier 3: at each `dispatch()` call that reaches tier 3, the router samples `randomFn()` (default `Math.random`, injectable for tests) and if the value is less than `explorationBudgetFraction`, routes the attempt to Ollama. The exploration assignment is skipped (falls to tier 4) if: (a) any `wint.model_affinity` entry for the `(change_type, file_type)` with `success_rate < explorationMinSuccessRateFloor` AND `sample_size > 0`, or (b) the Ollama provider is unavailable. When `sample_size = 0`, exploration is always allowed regardless of floor (no evidence of failure yet).

- [ ] **AC-5**: Optional manual profile seeding is implemented as a standalone utility function `importAffinitySeeds(seeds: ManualAffinitySeedEntry[], db: Db): Promise<SeedImportResult>` exported from `packages/backend/orchestrator/src/pipeline/affinity-seeder.ts`. The function accepts a Zod-validated array of `ManualAffinitySeedEntrySchema` objects and upserts them into `wint.model_affinity` using `ON CONFLICT (model, change_type, file_type) DO UPDATE SET` pattern from APIP-3020. Returns `{ seeded: number, skipped: number, errors: SeedImportError[] }`.

- [ ] **AC-6**: `importAffinitySeeds()` applies the same `CONFIDENCE_THRESHOLDS` constants (imported from APIP-3020's `pattern-miner.ts`) to any seed entries that provide `sample_size` but not an explicit `confidence_level`, ensuring consistency between mined and seeded profiles.

## Architecture Notes

**Four-Tier Routing Precedence (after APIP-3070):**
```
Tier 1: wint.model_assignments DB override (always wins, operator control)
Tier 2: Affinity-guided (confidence ≥ medium, success_rate ≥ threshold — from APIP-3040)
Tier 3: Exploration slot (randomFn() < explorationBudgetFraction, floor not violated, Ollama available)
Tier 4: Conservative OpenRouter default (cold-start and all other fallbacks)
```

**Cold-Start Detection Strategy:**
Use a lazy-evaluated cached flag `_hasAnyHighConfidenceProfile: boolean | null` (null = uninitialized). On first dispatch after construction or after `invalidateAffinityCache()`, query `wint.model_affinity` once for any row with `confidence_level IN ('medium', 'high')`. Cache the result. If none found, skip tiers 2 and 3 entirely, avoiding unnecessary DB queries on every dispatch during cold-start. Reset flag on `invalidateAffinityCache()`.

**PRNG Injection:**
`Math.random()` is not injectable for deterministic testing. Accept optional `randomFn: () => number = Math.random` in `PipelineModelRouterConfig`. In production, uses `Math.random`. In unit tests, inject a deterministic function (e.g., `() => 0.05` for exploration, `() => 0.5` for no-exploration).

**Single-Process Limitation (APIP ADR-001 Decision 4):**
`_hasAnyHighConfidenceProfile` and per-dispatch exploration counters are in-memory state. This is valid for single-process Phase 0–3 deployment. If APIP-3080 (parallel worktrees) or Phase 4+ introduces multiple processes, exploration state will not be shared across instances. This limitation must be documented in code comments but requires no action for Phase 0–3.

**Exploration Budget Semantics:**
The 10% budget is randomized at the individual `ChangeSpec` dispatch level — each attempt independently has a 10% chance of being routed to Ollama. It is NOT a 10%-of-all-stories allocation. Setting `explorationBudgetFraction = 0.0` disables exploration entirely. Setting `explorationBudgetFraction = 1.0` forces all eligible attempts to Ollama (testing scenario only).

**Exploration Floor:**
`explorationMinSuccessRateFloor = 0.3` prevents routing to Ollama if early telemetry shows `success_rate < 0.3` with `sample_size > 0`. When `sample_size = 0`, exploration is always allowed (no evidence of failure). This prevents runaway failures from a demonstrably poor-performing Ollama model from consuming exploration budget indefinitely.

**Tier Renaming Note (from APIP-3040 → APIP-3070):**
_Added by autonomous elaboration — GAP-001._ APIP-3040 defines a three-tier routing structure where "Tier 3" is the static Ollama → OpenRouter → Anthropic escalation chain (fallback). APIP-3070 inserts exploration as new Tier 3, pushing the static fallback to Tier 4. When extending APIP-3040 code in ST-2 and ST-4, any references to the static escalation chain as "Tier 3" (in variable names, constants, comments, or `__types__/index.ts`) must be updated to "Tier 4" to match the four-tier architecture.

**Cold-Start Flag Query Clarification:**
_Added by autonomous elaboration — GAP-002._ The `_hasAnyHighConfidenceProfile` flag uses a single `COUNT(*) > 0` query for any qualifying row in `wint.model_affinity` — it does not reuse the per-key affinity cache from APIP-3040. This is intentional: the global cold-start check is a cheap aggregate query that avoids iterating all `(change_type, file_type)` combinations on every dispatch. After `invalidateAffinityCache()`, both the global cold-start flag and the per-key affinity cache reset independently — the global flag re-queries first (fast), and per-key cache entries lazy-load on demand (correct behavior, no race condition).

## Seed Requirements

**Optional manual profile seeding** is available via `importAffinitySeeds()` (AC-5). Operators may provide a Zod-validated array of `ManualAffinitySeedEntry` objects. Each entry maps `(model, change_type, file_type)` → `(success_rate, sample_size, confidence_level)`. If `confidence_level` is omitted, it is derived from `sample_size` using `CONFIDENCE_THRESHOLDS` (AC-6).

Seeding data source is left to operator judgment (published benchmark results, internal spike results, manually curated estimates). The system operates safely without any seeding via conservative OpenRouter defaults.
