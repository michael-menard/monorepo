# APIP-5000: Test Infrastructure Setup for Autonomous Pipeline Unit Testing (Phase 0)

## Context

The autonomous pipeline (APIP series) is a planned system for fully autonomous story implementation. Phase 0 establishes the foundation: a BullMQ work queue (APIP-0010), a plain TypeScript supervisor loop (APIP-0020), LangGraph deployment (APIP-0030), and a model router (APIP-0040). Before these components can be safely unit tested, the package needs a Vitest configuration, Zod-validated fixture factories for BullMQ jobs and supervisor state, and vi.fn()-based mock utilities for the BullMQ worker and queue interfaces.

APIP-5000 was originally scoped as "LangGraph Graph Unit Testing" and included Phase 1 graph mock templates for structurer, implementation, review, and QA graphs. This was incorrect: APIP-0020 is a plain TypeScript async loop (not a LangGraph graph), and Phase 1 graph interfaces do not yet exist. The story has been rescoped to Phase 0 deliverables only.

**Key sequencing note**: The `apps/api/autonomous-pipeline` package does not yet exist in the codebase. This story includes package scaffolding (ST-1) unless APIP-0010 delivers it first.

## Goal

Establish Vitest test infrastructure for the `apps/api/autonomous-pipeline` package: a vitest.config.ts with coverage thresholds, Zod-validated fixture factories for BullMQ jobs and supervisor state, vi.fn()-based mock utilities for BullMQ Worker and Queue interfaces, and one integration smoke test that validates the real supervisor loop against mocked infrastructure.

## Non-Goals

- LangGraph graph node mock utilities (node stubs, state snapshot fixtures for structurer, implementation, review, QA graphs) — deferred to a new Phase 1 story
- Test templates for Phase 1 graphs — premature until graph interfaces are elaborated
- Mocking the LangGraph SDK itself (StateGraph, compile(), channel reducers) — per seed risk mitigation
- Integration tests against a real Redis instance — covered by APIP-5001
- E2E test infrastructure — covered by APIP-5002
- BullMQ Bull Board dashboard testing — covered by APIP-0010
- Coverage enforcement for Phase 1 graphs — deferred to Phase 1 test infrastructure story

## Architecture Notes

- APIP-0020 is a **plain TypeScript async function** (not a LangGraph graph). The supervisor state fixture and integration smoke test are TypeScript-only — no LangGraph SDK imports.
- Fixture factories must import their Zod schemas from APIP-0010/APIP-0020 production exports. Locally-defined copies of production schemas are a forbidden pattern: they cause silent fixture drift when production schemas evolve.
- The `vi.fn()` mock pattern is sufficient for BullMQ Worker and Queue. No dedicated mock library (e.g., jest-mock-extended) is needed.
- `fileParallelism: false` in vitest.config.ts is appropriate if integration tests share process state. For pure unit tests, parallelism can be enabled per the knowledge-base config pattern.
