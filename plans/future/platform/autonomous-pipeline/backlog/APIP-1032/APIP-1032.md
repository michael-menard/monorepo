## Architecture Notes

### Change Loop Conditional Edge Structure
```
change-loop â†’ pass  â†’ commit â†’ (more specs?) â†’ change-loop | evidence-production
           â†’ retry  â†’ (attempts < maxRetries?) â†’ change-loop | abort
           â†’ abort  â†’ evidence-production
```

The change-loop node does NOT use `createToolNode` wrapping (per GAP-3 resolution in APIP-1031). Its retry logic is driven entirely by application-level `maxRetries` state, not the node-factory retry layer.

### Atomic Commit Pattern
Each ChangeSpec results in exactly one git commit before the loop advances:

```typescript
execSync(`git -C ${worktreePath} add -A`, { stdio: 'pipe' })
const sha = execSync(
  `git -C ${worktreePath} commit -m "feat(${storyId}): ${changeSpec.id} â€” ${changeSpec.summary}"`,
  { stdio: 'pipe' }
).toString().trim()
// Update state.completedChanges AFTER successful commit only
state.completedChanges.push({ id: changeSpec.id, sha })
```

### Idempotent Resume Check
Before dispatching each ChangeSpec, check `completedChanges`:

```typescript
const alreadyCommitted = state.completedChanges.some(r => r.id === changeSpec.id)
if (alreadyCommitted) {
  logger.info('change_spec_skipped_resume', { storyId, changeSpecId: changeSpec.id })
  continue
}
```

### BudgetExhaustedError Propagation
The change loop does NOT swallow `BudgetExhaustedError`. It marks evidence `status: aborted`, stops dispatching remaining ChangeSpecs, and re-throws for the BullMQ job processor (APIP-0020 supervisor) to handle as permanent failure.

### OPP-2 Consideration (Micro-Verify Timeout)
The ELAB identified an opportunity to add a configurable `microVerifyTimeoutMs` (default: 120000ms) to prevent runaway test hangs. This should be considered during elaboration of APIP-1032 or added as an explicit AC. If not added as an AC, document the absence as an accepted risk.

### Zod-First Schema Requirement
All new types in the change-loop node (micro-verify result, abort reason record) MUST be `z.object()` schemas with `z.infer<>` type aliases. Zero TypeScript interfaces (per WINT-9090 lesson).
