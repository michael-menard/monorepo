# APIP-1010: Structurer Node in Elaboration Graph

## Context

The autonomous pipeline's diff planner (APIP-1020+) requires structured, machine-readable decomposition of a story's acceptance criteria into an ordered list of expected file changes before implementation can proceed. Currently, the elaboration graph (`packages/backend/orchestrator/src/graphs/elaboration.ts`) produces human-readable aggregate findings but no structured change outline that a downstream diff planner can consume.

**Baseline Gap (2026-02-13)**: No Structurer node exists in any graph. No `change_outline` schema exists. No complexity estimation or split-flagging logic exists anywhere in the codebase. The elaboration graph currently terminates at `aggregate_findings` → `update_readiness` → `save_to_db` → `complete` with no downstream structured output suitable for diff planning.

**Critical path position**: APIP-1010 is story #3 on the 12-story critical path. It unblocks APIP-1020 (Diff Planner) and ultimately APIP-1030 (Implementation Graph).

**Active in-progress context**:
- APIP-0020 (Supervisor Loop) is in elaboration. Its implementation must treat `changeOutline` as an optional nullable field it does not yet consume — no rework required.
- APIP-5006 (LangGraph Server Infrastructure) is in elaboration. The Structurer node's LLM calls (if used) require the dedicated server for integration testing.
- APIP-5007 (DB Schema Versioning) is in elaboration. No direct conflict — Structurer does not write to Aurora.

## Goal

Add a **Structurer node** to the existing elaboration graph that reads the elaborated story's acceptance criteria, produces a `change_outline` (ordered list of `ChangeOutlineItem` records with complexity estimates), and flags stories exceeding the configurable split threshold (`splitThreshold`, default 15 atomic changes) for human review before proceeding to implementation.

## Non-Goals

- Defining the ChangeSpec schema (diff planner input contract) — that is APIP-1020's research spike
- Integrating `change_outline` into any downstream graph — deferred to APIP-1020, APIP-1030
- Automated story splitting (creating two stories) — APIP-1010 only flags; splitting is a human operator action
- LLM-based complexity estimation as the primary path — heuristic-first is acceptable for Phase 1
- Writing `changeOutline` to Aurora — output lives in LangGraph state and `ElaborationResult` only
- Modifying `FinalAcceptanceCriterionSchema` or `SynthesizedStory` schemas — Structurer reads, does not modify
- Operator CLI visibility for split-flagged stories — APIP-5005
- Any UI/dashboard changes — APIP-2020
- Modifying `packages/backend/database-schema/` — protected

## Architecture Notes

**Node placement** in elaboration graph:

```
aggregate_findings → [afterAggregate conditional edge]
  → structurer (if enabled:true) → update_readiness
  → update_readiness (if enabled:false, bypass)
update_readiness → save_to_db → complete
```

**Heuristic complexity estimation (v1)**:
- For each AC, parse description for distinct system/module references
- 1 system reference → `low` (1-3 changes), midpoint: 2
- 2 system references → `medium` (3-8 changes), midpoint: 5
- 3+ system references or cross-cutting (from `escapeHatchResult`) → `high` (8-15 changes), midpoint: 11
- `estimatedAtomicChanges` = midpoint of range
- `totalEstimatedAtomicChanges` = sum across all items
- Cross-cutting flag from `escapeHatchResult` bumps estimates up one level

**Backward compatibility**:
- `runElaboration(currentStory, previousStory, config)` signature unchanged
- New `ElaborationResult` fields are all nullable with defaults
- New `ElaborationConfig` field (`structurerConfig`) has full defaults, no callers need to change

**splitReason format** (human-readable, actionable):
```
"Story exceeds split threshold: 22 estimated atomic changes (threshold: 15). Contributing changes: [CO-3: modify AuthService (8), CO-7: create UserQuotaAdapter (9)]."
```

**ChangeOutlineItem schema design** — intentionally minimal per ADR-001 Decision 3:
- Fields: `id`, `filePath`, `changeType`, `description`, `complexity`, `estimatedAtomicChanges`, `relatedAcIds`
- Include `extensions: z.record(z.unknown()).optional()` escape hatch for APIP-1020 spike additions

## Infrastructure Notes

Per APIP ADR-001 Decision 4: Structurer runs on the dedicated local LangGraph server as part of the elaboration graph. No Lambda. No new infrastructure beyond what APIP-5006 provisions.

Server restart required after elaboration graph update to pick up new Structurer node wiring.

## Conflict Resolution Notes

**Conflict 1 (APIP-0020 dependency)**: `changeOutline` added as nullable field to `ElaborationResult`. APIP-0020 supervisor implementation should not assert on `ElaborationResult` shape beyond what it needs — the field will be present post-APIP-1010 but null or populated. No rework required for APIP-0020.

**Conflict 2 (APIP-1020 ChangeSpec schema not yet defined)**: `ChangeOutlineItem` is designed minimally with an `extensions` escape hatch. APIP-1020 spike may add fields via `extensions` without requiring a schema-breaking migration. APIP-1010 does not anticipate ChangeSpec internals.
