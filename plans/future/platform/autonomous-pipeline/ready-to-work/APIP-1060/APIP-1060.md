# APIP-1060: QA Graph with Autonomous Verdict

## Context

The autonomous pipeline produces implementations (APIP-1030) that pass through multi-dimensional code review (APIP-1050). Before the Merge Graph (APIP-1070) commits and merges changes, a quality gate must autonomously validate two things: (1) all tests pass, and (2) each acceptance criterion is demonstrably fulfilled. This story implements that gate — the QA Graph.

The existing `QaVerifySchema` at `packages/backend/orchestrator/src/artifacts/qa-verify.ts` defines the data contract for QA results. The `EvidenceSchema` at `packages/backend/orchestrator/src/artifacts/evidence.ts` defines the implementation evidence produced by APIP-1030. No LangGraph graph currently exists to orchestrate the test execution, evidence verification, and gate decision process autonomously.

The QA graph runs on the dedicated local server (APIP ADR-001 Decision 4 — no Lambda). It is a LangGraph worker graph dispatched from the plain TypeScript supervisor (APIP ADR-001 Decision 2) via a BullMQ job with `stage: 'qa'`. On PASS verdict, the supervisor advances the story to the Merge Graph (APIP-1070). On FAIL or BLOCKED, the supervisor routes back to the Implementation Graph (APIP-1030) with the QA artifact as failure context.

**Implementation dependency**: APIP-1060 requires APIP-1050 (Review Graph) to be complete before integration testing. Elaboration and unit test development can proceed independently using mocked inputs.

## Goal

Implement a LangGraph `StateGraph` that autonomously validates AC fulfillment and test passage before allowing merge, producing a `QaVerify` artifact with a structured `PASS`, `FAIL`, or `BLOCKED` verdict. No human gate. No manual approval step.

## Non-Goals

- Implementing the Merge Graph (APIP-1070) — it consumes the QA artifact this graph produces
- Implementing the Implementation Graph retry loop on QA FAIL (APIP-1030) — routing back is the supervisor's responsibility
- Implementing a human-approval gate — this story explicitly targets autonomous verdict
- Building operator CLI or dashboard visibility for QA results (APIP-5005, APIP-2020)
- Modifying `packages/backend/database-schema/` or `@repo/db` client — protected
- Implementing the model router (APIP-0040) — QA graph consumes an injected `ModelClient` abstraction
- Implementing secrets engine (APIP-5004) — QA graph reads credentials from environment variables
- Full E2E pipeline testing (APIP-5002)
- Change telemetry instrumentation (APIP-3010)
- Any UI component changes

## Architecture Notes

**Node sequence**:
1. `check-preconditions` — Gate: review must be PASS, evidence must be valid. BLOCKED otherwise.
2. `run-unit-tests` — `pnpm test` in worktree via async spawn. Retry on timeout.
3. `run-e2e-tests` (conditional on `enableE2e: true`) — Playwright via async spawn. Retry up to `playwrightMaxRetries`. Only FAIL if all attempts fail.
4. `verify-acs` — Per-AC model call (Claude only). Returns structured `{ status, cited_evidence, reasoning }`. Model failure per AC → BLOCKED (not FAIL), continue.
5. `gate-decision` — Aggregates all verifications and test results. Claude call. BLOCKED on model failure.
6. `write-qa-artifact` — Persists `QA-VERIFY.yaml`. Writes lessons to KB if enabled. Sets `qaComplete: true`.

**ModelClient abstraction**:
```typescript
const ModelClientSchema = z.object({
  callModel: z.function()
    .args(z.string(), z.object({ model: z.string() }))
    .returns(z.promise(z.string()))
})
type ModelClient = z.infer<typeof ModelClientSchema>
```
Injected at `createQAGraph(config, { modelClient })`. Enables testability without APIP-0040 dependency.

**Subprocess timeout pattern**:
```typescript
const proc = spawn('pnpm', ['test'], { cwd: worktreeDir, env: process.env })
const exitPromise = new Promise<number>((resolve, reject) => {
  proc.on('exit', code => resolve(code ?? 1))
  proc.on('error', reject)
})
const timeoutPromise = new Promise<never>((_, reject) =>
  setTimeout(() => { proc.kill(); reject(new Error('timeout')) }, testTimeoutMs)
)
const exitCode = await Promise.race([exitPromise, timeoutPromise])
```

**AC verification prompt — anti-hallucination constraint**:
The `AC_VERIFICATION_PROMPT_V1` constant must include:
- AC text verbatim (no paraphrasing)
- All evidence items verbatim (no summarizing)
- Instruction: "Do not infer, assume, or extrapolate. Only respond PASS if the provided evidence explicitly demonstrates the criterion."
- Structured JSON response requirement with `cited_evidence` field

**Gate decision — Claude hard constraint**: The `gateModel` field is `z.literal('claude')` — not overridable to cheap models. Gate decision is safety-critical.

**Artifact output path**: `plans/future/platform/autonomous-pipeline/in-progress/{storyId}/QA-VERIFY.yaml`

**FAIL routing**: The QA graph does NOT implement the retry routing back to APIP-1030. It only produces the `QaVerify` artifact with verdict. The supervisor reads the verdict and dispatches accordingly.
