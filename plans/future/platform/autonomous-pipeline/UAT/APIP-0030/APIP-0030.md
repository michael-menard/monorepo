# APIP-0030: LangGraph Platform Docker Deployment

## Context

APIP-5006 established the server infrastructure baseline: a provisioned dedicated server with Docker Compose v2, base compose structure, and documented network configuration. This story builds on that foundation.

The autonomous pipeline (APIP) requires LangGraph worker graphs (elaboration, implementation, review, QA, merge) for all structured AI work. These graphs require the LangGraph Platform — a self-hosted Docker service that provides durable graph execution, PostgreSQL-backed checkpointing, state recovery, and Studio UI for operator inspection.

This story delivers the Docker Compose configuration and operational documentation to deploy LangGraph Platform on the dedicated server. It is a pure infrastructure story — no TypeScript code, no Lambda handlers, no UI components.

The monorepo already runs two isolated PostgreSQL instances (port 5432 main, port 5433 Knowledge Base). APIP-0030 adds a third: `langgraph-checkpoint-postgres` connecting to Aurora `langgraph_checkpoint` database with a dedicated connection pool — following the same isolation pattern established for the KB pgvector instance (`apps/api/knowledge-base/`).

**Predecessor**: APIP-5006 (must complete before implementation begins)
**Successor**: APIP-5003 (security hardening), APIP-0020 (supervisor connects to port 8123)

## Goal

Deploy the LangGraph Platform Docker configuration on the dedicated server, providing a durable, restart-safe execution environment for APIP worker graphs with PostgreSQL-backed state checkpointing.

**Note on supervisor scope**: The TypeScript supervisor (APIP-0020) is a separate plain TypeScript process — NOT deployed via LangGraph Platform per ADR-001 Decision 2. The supervisor connects to LangGraph Platform HTTP API at port 8123 as a client. APIP-0030 deploys the platform the worker graphs run on, not the supervisor itself.

## Non-Goals

- Deploying or implementing any LangGraph worker graphs (elaboration, implementation, review, QA, merge) — those are APIP-1010 through APIP-1070
- Implementing the TypeScript supervisor loop — that is APIP-0020

- Implementing BullMQ work queue — that is APIP-0010

## Architecture Notes

### LangGraph Platform Services

The compose file will include these services:

1. **`langgraph-platform`** — LangGraph Platform server
   - Image: pinned to specific version (e.g., `langchain/langgraph-api:<version>`) — do NOT use `latest`
   - Port: `8123:8123` (API) — accessible from host and Docker network
   - Port: `127.0.0.1:8124:8124` (Studio UI) — localhost-only binding per SEC-001
   - Healthcheck: HTTP check on `/ok` or `/health`
   - `restart: unless-stopped`

2. **`langgraph-checkpoint-postgres`** — Checkpoint database configuration
   - Connects to Aurora PostgreSQL `langgraph_checkpoint` database
   - Uses environment variable for connection string: `LANGGRAPH_CHECKPOINT_DATABASE_URL`
   - Connection pool: max 3, dedicated `langgraph_checkpoint_user` credentials

3. **`langgraph-checkpoint-init`** — One-shot schema initializer
   - Runs `langgraph-checkpoint-postgres` schema migrations
   - `restart: "no"`
   - `depends_on: langgraph-checkpoint-postgres: condition: service_healthy`

### Connection Pool Isolation

```
Aurora PostgreSQL
  ├── monorepo (port 5432) ← @repo/db (main application pool)
  ├── knowledgebase (port 5433) ← KB pg.Pool (pgvector, max 10)
  └── langgraph_checkpoint ← dedicated pg.Pool (max 3, langgraph_checkpoint_user)
```

The `langgraph-checkpoint-postgres` connection MUST use a separate `pg.Pool` instance — never import from `@repo/db`. Connection string: `postgresql://langgraph_checkpoint_user:<password>@<aurora_host>:5432/langgraph_checkpoint`.

### Supervisor Integration (APIP-0020 Reference)

The TypeScript supervisor (implemented in APIP-0020) connects to LangGraph Platform as an HTTP client:
- **Within Docker network**: `http://langgraph-platform:8123`
- **From host (dev/debugging)**: `http://localhost:8123`

The supervisor does NOT run inside LangGraph Platform. It runs as a separate TypeScript process on the same server, connecting to the LangGraph Platform API over HTTP.

**Deployment procedure (Phase 0 — manual):**
```bash

scp infra/langgraph-server/compose.langgraph-server.yaml user@server:/opt/langgraph-server/

# On server
docker compose -f /opt/langgraph-server/compose.langgraph-server.yaml pull
docker compose -f /opt/langgraph-server/compose.langgraph-server.yaml up -d
```
