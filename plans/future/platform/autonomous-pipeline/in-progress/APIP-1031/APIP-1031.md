Worktree cleanup never blocks graph result propagation.

- [ ] **AC-4**: A `load-story` node reads the story file (STORY-SEED.md or elaborated story) and the ChangeSpec plan (produced by APIP-1020's Diff Planner) into graph state. Missing story or ChangeSpec plan produces a typed `LoadError` that transitions to `abort` edge.

- [ ] **AC-9**: An `evidence-production` node is implemented after the change loop position in the graph. It writes an `EVIDENCE.yaml` file to the story's feature directory using `EvidenceSchema` + `createEvidence()` / `addTouchedFile()` / `addCommandRun()` helpers. In APIP-1031, the evidence node is wired but the change loop data fields (commit SHAs, micro-verify results) will be populated by APIP-1032. The node must handle empty/partial state gracefully.

- [ ] **AC-10**: The implementation graph logs structured events with `@repo/logger` for skeleton lifecycle transitions: `graph_started`, `worktree_created`, `story_loaded`, `evidence_written`, `graph_completed` — with fields: `storyId`, `attemptNumber`, `durationMs` on each event. (Change-loop-specific events — `change_spec_dispatched`, `micro_verify_passed`, `micro_verify_failed`, `change_committed`, `change_aborted` — are logged by APIP-1032's change-loop node.)

## Architecture Notes

### Thread ID Convention
LangGraph checkpoint thread ID: `{storyId}:implementation:{attemptNumber}`

Consistent with the convention established in APIP-0020 (`{storyId}:{stage}:{attempt}`). This must be documented and respected. Changing it post-implementation requires checkpoint data migration.

### GAP-1 Resolution: CommitRecord Schema for Graph State
The `ImplementationGraph` state Annotation MUST include a `completedChanges` field typed via Zod:

```typescript
// In graphs/implementation.ts
const CommitRecordSchema = z.object({
  id: z.string(),
  sha: z.string(),
})
type CommitRecord = z.infer<typeof CommitRecordSchema>

// In ImplementationGraphStateAnnotation:
completedChanges: Annotation<CommitRecord[]>({
  default: () => [],
  reducer: (existing, update) => [...existing, ...update],
})
```

This schema is the input type for the idempotent skip check in APIP-1032's ST-6. Define it in APIP-1031 (ST-1) so APIP-1032 can import it.

### GAP-2 Resolution: Non-Blocking Worktree Cleanup Pattern
`create-worktree` node cleanup must use the fire-and-forget async IIFE pattern:

```typescript
// Non-blocking cleanup — do NOT await
void (async () => {
  try {
    execSync(`git worktree remove --force ${worktreePath}`, { stdio: 'pipe' })
    logger.info('worktree_cleaned', { storyId, worktreePath })
  } catch (e) {
    logger.warn('worktree_cleanup_failed', { storyId, worktreePath, error: e })
  }
})()
```

This ensures cleanup is attempted but never blocks the calling flow. Cleanup errors are logged but not propagated.

### GAP-3 Resolution: createToolNode Wrapping Scope
Internal graph nodes (`load-story`, `create-worktree`, `evidence-production`, and the stub change-loop position) are registered as **plain async functions** in `StateGraph.addNode()`, matching `elaboration.ts` exactly. `createToolNode` is used **only** for the `runImplementation()` workflow adapter export.

The change-loop node (APIP-1032) will manage its own `maxRetries` counter at the application level. Wrapping it in `createToolNode` would add a conflicting retry layer (maxAttempts: 2, timeout: 10s from node-factory) that must be avoided.

### IModelDispatch Injection Pattern
The change-loop stub (and eventually the real change-loop in APIP-1032) receives `IModelDispatch` via config/constructor injection:

```typescript
// packages/backend/orchestrator/src/pipeline/i-model-dispatch.ts
const IModelDispatchSchema = z.object({
  dispatch: z.function()
    .args(z.object({
      storyId: z.string(),
      agentId: z.string(),
      context: ChangeSpecSchema,
    }))
    .returns(z.promise(ModelResponseSchema))
})
type IModelDispatch = z.infer<typeof IModelDispatchSchema>
```

### ChangeSpec Placeholder Schema
A minimal placeholder schema enables the graph skeleton to compile and enables unit tests before APIP-1020 closes:

```typescript
// packages/backend/orchestrator/src/graphs/change-spec-schema.ts
// PLACEHOLDER — replace with APIP-1020 ADR schema when available
const ChangeSpecSchema = z.object({
  id: z.string(),
  summary: z.string(),
  touchedPackages: z.array(z.string()),
  instructions: z.string(),
})
type ChangeSpec = z.infer<typeof ChangeSpecSchema>
```

### Zod-First Schema Requirement
All new types (graph state, ChangeSpec placeholder, micro-verify result stub, CommitRecord, LoadError) MUST be `z.object()` schemas with `z.infer<>` type aliases. Zero TypeScript interfaces in graph or node files (per WINT-9090 lesson).
