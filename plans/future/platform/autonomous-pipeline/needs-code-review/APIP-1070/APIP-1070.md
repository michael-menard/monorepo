# APIP-1070: Merge Graph with Learnings Extraction

## Context

The autonomous pipeline completes the review (APIP-1050) and QA (APIP-1060) stages for a story's implementation. When the QA Graph emits a `PASS` verdict, the pipeline needs to take the implementation from a git worktree branch, create a pull request, wait for CI to pass, squash-merge it onto main, clean up the worktree, and capture the implementation's learnings into the Knowledge Base.

There is no automated merge pipeline in the codebase today. The `QaVerifySchema` at `packages/backend/orchestrator/src/artifacts/qa-verify.ts` defines the data contract for the QA gate input, and the `persist-learnings` node at `packages/backend/orchestrator/src/nodes/completion/persist-learnings.ts` implements the KB write-back pattern — but no LangGraph graph exists to orchestrate the PR creation, CI polling, squash-merge, worktree cleanup, and learnings extraction steps autonomously.

This story builds the Merge Graph: the final graph in the autonomous pipeline's Phase 1 critical path. It is a headless LangGraph `StateGraph` running on the dedicated local server with no UI components and no Aurora database writes.

### Reality Baseline

- **Baseline date**: 2026-02-13
- **Key gaps confirmed**: No Merge Graph exists. No `gh` CLI or GitHub API integration exists in the orchestrator package. No PR creation, CI polling, or squash-merge automation exists anywhere in `packages/backend/orchestrator/`.
- **Stable contracts**: `QaVerifySchema` at `packages/backend/orchestrator/src/artifacts/qa-verify.ts` is fully defined and stable. `persist-learnings` node is implemented and reusable. The LangGraph worker graph pattern is established in `elaboration.ts`.
- **Integration gate**: Unit test development can proceed independently. Full integration testing is blocked on APIP-1060 (QA Graph) completing and producing a real `QA-VERIFY.yaml` artifact.

## Goal

Implement a LangGraph `StateGraph` that: (1) gates on QA PASS verdict, (2) rebases the story branch on main, (3) creates or updates a GitHub PR, (4) polls CI to completion with exponential back-off, (5) squash-merges onto main, (6) cleans up the git worktree on all terminal paths, (7) extracts learnings from the QA artifact and persists them to the Knowledge Base, and (8) writes a `MERGE.yaml` artifact recording the full outcome.

## Non-Goals

- Implementing the QA Graph (APIP-1060) — the Merge Graph reads the QA artifact as input.
- Implementing the supervisor loop dispatch that calls the Merge Graph — that is APIP-0020.
- Implementing the Documentation Graph (APIP-1040) — that story runs after merge and is a separate graph.
- Implementing the operator CLI or dashboard visibility for merge results — APIP-5005, APIP-2020.
- Modifying `packages/backend/database-schema/` or `@repo/db` client — protected areas.
- Building the secrets engine (APIP-5004) — Merge Graph reads GitHub token from environment variable.
- Full E2E pipeline testing — APIP-5002.
- Change telemetry instrumentation — APIP-3010.
- Implementing the model router (APIP-0040) — Merge Graph makes no LLM calls.
- Any UI component changes.
- Regular merge strategy — squash-merge only; do not implement regular merge or rebase-merge.

## Architecture Notes

### Node Sequence

```
START
  → check-preconditions      (QA gate + gh auth preflight)
  → rebase-branch            (git fetch + rebase + push)
  → create-or-update-pr      (gh pr create/edit)
  → poll-ci                  (gh pr checks with exponential back-off)
  → squash-merge             (gh pr merge --squash)
  → cleanup-worktree         (git worktree remove --force + branch delete)
  → extract-learnings        (QA lessons → KB via createPersistLearningsNode)
  → write-merge-artifact     (MERGE.yaml atomic write)
  → END
```

All failure paths (MERGE_BLOCKED, MERGE_FAIL) skip to `cleanup-worktree` and continue through `extract-learnings → write-merge-artifact → END`.

### Subprocess Pattern

All `gh` CLI and `git` invocations use `child_process.spawn` (never `execSync`):

```typescript
type SubprocessRunner = (
  args: string[],
  opts: { cwd: string; env?: Record<string, string> }
) => Promise<{ exitCode: number; stdout: string; stderr: string }>
```

- Injectable at node construction time for testability
- `GH_TOKEN` injected via `env: { ...process.env, GH_TOKEN: config.ghToken ?? process.env.GH_TOKEN }`
- Timeout via `Promise.race([exitPromise, timeoutPromise])`

### CI Polling Design

The `poll-ci` node implements an **internal async loop** (not LangGraph edges per poll iteration). This avoids excessive checkpoint writes. LangGraph checkpoints after the node completes.

```typescript
// Back-off schedule
let interval = config.ciPollIntervalMs  // 30s default
const maxInterval = config.ciPollMaxIntervalMs  // 5min default
const startTime = Date.now()

while (Date.now() - startTime < config.ciTimeoutMs) {
  const result = await ghRunner(['pr', 'checks', ...])
  // evaluate: pass / fail / still running
  const jitter = 0.9 + Math.random() * 0.2  // ±10%
  interval = Math.min(interval * 2, maxInterval) * jitter
  await sleepFn(interval)
  state.ciPollCount++
}
// If here: timeout
```


### Thread ID Convention

LangGraph checkpoint thread ID: `` `${storyId}:merge:${attempt}` ``

This follows the established convention from APIP-1030 (`{storyId}:{stage}:{attempt}`). If the merge is interrupted, the supervisor can re-dispatch with the same thread ID and LangGraph will resume from the last checkpoint.

## Infrastructure Notes

- **Runs on dedicated local Docker server** (APIP ADR-001 Decision 4). No Lambda. `gh` CLI and git are local-process invocations.
- **`gh` CLI must be installed and authenticated** on the server. The `check-preconditions` node includes a `gh auth status` preflight. If not authenticated, returns `MERGE_BLOCKED` with operator instructions.
- **GitHub token** read from `GH_TOKEN` environment variable (or `config.ghToken`). Managed by APIP-5004 (Secrets Engine). Never hardcoded.
- **Git worktree** must exist at `config.worktreeDir` before the graph runs. Created by the supervisor (APIP-0020) prior to dispatch.
