# APIP-0040: Model Router v1 with Rate Limiting and Token Budgets

## Context

The autonomous pipeline dispatches AI model calls for every phase of story development: elaboration, implementation, review, QA, and merge. Without cost controls, a single runaway story could exhaust monthly model budgets.

The existing `ModelRouter` in `packages/backend/orchestrator/src/models/` provides tier-based model selection and escalation logic via `ModelRouterFactory` (singleton), but has three critical gaps for autonomous pipeline use:

1. **No per-provider rate limiting** — nothing prevents a burst of LLM calls from hitting provider rate limits simultaneously.
2. **No per-story token budget tracking** — there is no cross-phase accumulation or hard cap. The existing `checkTokenBudget()` in `token-budget.ts` enforces phase-level limits but does not accumulate across a story's lifetime.
3. **No DB-backed configuration** — model assignments are YAML-file-driven, requiring code deploys to change which model handles which agent type.

The epic review (COST-CONTROL-001, severity: high) mandates that APIP-0040 delivers a hard budget cap circuit breaker before APIP-1030 (Implementation Graph) begins executing real stories.

This story is blocked on APIP-0010 (BullMQ work queue) because the `PipelineModelRouter` must expose a callable interface usable by the BullMQ job processor spawned in APIP-0020.

## Goal

Build a `PipelineModelRouter` wrapper in a new `packages/backend/orchestrator/src/pipeline/` directory that:

1. Wraps the existing `ModelRouter` with a **token-bucket rate limiter** per provider (Ollama, OpenRouter, Anthropic).
2. Tracks **per-story cumulative token spend** and enforces a configurable hard budget cap.
3. Implements the **Ollama → OpenRouter → Claude escalation chain** with explicit error logging at each step.
4. Reads **model assignment overrides** from a new `wint.model_assignments` DB table.
5. Surfaces **budget exhaustion** as a typed `BudgetExhaustedError` so BullMQ job processors can mark jobs as permanently failed (no retry).

## Non-Goals

- Do not implement AI-driven model affinity or learning-based routing (deferred to APIP-3040).
- Do not implement concurrency limits for parallel story execution (deferred to APIP-3080).
- Do not implement secrets vault integration — use environment variables only. APIP-5004 handles vault.
- Do not add a UI or CLI surface for model router configuration (deferred to APIP-5005).
- **Do not modify any file in `packages/backend/orchestrator/src/models/`** (ModelRouter, task-selector, strategy-loader). These are protected by active WINT/MODL UAT coverage.
- Do not implement cron-based budget reset (deferred; Phase 0 budget is per-story lifetime).

### Typed Error Classes

Three distinct error classes must be defined using Zod schemas for field validation, with explicit `this.name` assignment in each constructor to preserve prototype chains across TypeScript transpilation:

```
BudgetExhaustedError extends Error
  fields: storyId: string, tokensUsed: number, budgetCap: number

RateLimitExceededError extends Error
  fields: provider: string, waitedMs: number

ProviderChainExhaustedError extends Error
  fields: providers: string[], lastError: string
```

### Singleton Pattern

`PipelineModelRouterFactory` follows the exact same pattern as `ModelRouterFactory`:
- Private static `_instance: PipelineModelRouter | null`
- `getInstance()` initializes once (including DB query for model_assignments) and returns the cached instance
- Rate limiter state (token buckets) and budget accumulator state are held on the singleton instance

### Token Bucket Implementation

In-memory implementation only for Phase 0 (ADR-001 Decision 4: single dedicated server). Token bucket state resets on process restart — acceptable because BullMQ job persistence handles restart recovery separately. Bucket `wait()` must use `setTimeout` internally for `vi.useFakeTimers()` compatibility.

### DB-Backed Config Strategy

On initialization, `PipelineModelRouter` queries `wint.model_assignments` and caches results in memory. Cache is invalidated (and re-fetched on next dispatch) when `invalidateAssignmentsCache()` is called. No polling loop. Any code that writes to `wint.model_assignments` must call `invalidateAssignmentsCache()`. For Phase 0 (single process), this is sufficient without distributed coordination.

### Escalation Chain

The escalation chain is a simple ordered array `['ollama', 'openrouter', 'anthropic']`. For each provider attempt:
1. Call `ModelRouterFactory.getInstance().getProvider(modelString)` to get the `ILLMProvider`
2. Call `provider.complete(prompt)`
3. On success: record tokens, return result
4. On failure: log `{ domain: 'pipeline_model_router', provider, reason, nextProvider }` at warn level and try next

Silent error swallowing is prohibited. Each failure must be logged with explicit provider name and error reason.

### Budget Exhaustion as BullMQ Signal

`BudgetExhaustedError` must be distinguishable as a permanent failure (do not retry). BullMQ job processors check `error instanceof BudgetExhaustedError` to mark the job as permanently failed vs. a transient error eligible for retry.
