# ST-1020: Story Template — Goal, Examples, Edge Cases Sections

## Context

The pm-spawn-patterns.md story file structure currently lists 17 required sections but does not include `## Examples` or `## Edge Cases`. The `## Goal` section exists as item 4. The pm-story-generation-leader Phase 4 synthesis references canonical references and subtasks but does not enforce generating Examples or Edge Cases sections. Adding these sections enables small LLMs and human reviewers to instantly understand scope through concrete examples.

## Goal

Every generated story file must include `## Examples` (2+ input/output examples) and `## Edge Cases` (2+ edge cases) sections, enabling both human reviewers and small LLMs to instantly understand scope.

## Non-Goals

- Changing the YAML frontmatter schema
- Modifying worker agents (test-plan, dev-feasibility, uiux)
- Retroactively updating existing generated stories

## Scope

Files touched:
- `.claude/agents/_reference/patterns/pm-spawn-patterns.md` — Add Examples and Edge Cases to the story file structure section list
- `.claude/agents/pm-story-generation-leader.agent.md` — Update Phase 4 synthesis to generate these sections

## Acceptance Criteria

- AC-1: `pm-spawn-patterns.md` Story File Structure includes `## Examples` section (after Goal, before Non-goals)
- AC-2: `pm-spawn-patterns.md` Story File Structure includes `## Edge Cases` section (after Examples, before Non-goals)
- AC-3: `pm-story-generation-leader.agent.md` Phase 4 synthesis instructions include directives to generate `## Examples` with 2+ concrete input/output examples
- AC-4: `pm-story-generation-leader.agent.md` Phase 4 synthesis instructions include directives to generate `## Edge Cases` with 2+ edge cases
- AC-5: Quality Gates table in pm-story-generation-leader includes gates for Examples and Edge Cases presence

## Examples

**Input**: A story seed about "Add dark mode toggle"
**Expected Output in generated story file**:
```markdown
## Examples

1. User clicks dark mode toggle → all UI surfaces switch to dark palette within 100ms
2. User refreshes page after enabling dark mode → dark mode persists (stored in localStorage)

## Edge Cases

1. System preference is dark but user explicitly set light mode → user preference wins
2. Component uses hardcoded colors instead of theme tokens → renders incorrectly in dark mode (should be caught by lint rule)
```

## Subtasks

1. Update pm-spawn-patterns.md section numbering to include Examples and Edge Cases
2. Add Examples and Edge Cases generation directives to Phase 4 synthesis in pm-story-generation-leader
3. Add quality gate rows for Examples and Edge Cases presence
