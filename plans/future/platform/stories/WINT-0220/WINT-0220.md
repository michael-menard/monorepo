# WINT-0220: Define Model-per-Task Strategy

## Context

The WINT (Workflow Intelligence) epic aims to build a sophisticated AI-powered workflow system capable of autonomous story execution, intelligent model selection, and adaptive learning. A critical foundation for this system is defining **which AI model should handle each type of task**.

### Current State

We currently have:
- **100+ agent files** with informal `model:` frontmatter (haiku/sonnet/opus)
- **Ollama local infrastructure** already set up for fast, cost-effective models
- **MODEL_STRATEGY.md** with informal cost optimization guidelines (50% reduction target)
- **Provider adapters** (MODL-0010) being implemented for OpenRouter/Ollama/Anthropic
- **Existing model assignment system** (`model-assignments.ts`) with TypeScript loader

### Problem

Without a formal, documented model-per-task strategy, we risk:
- **Inefficient cost allocation**: Using expensive Claude models for simple tasks (lint, formatting)
- **Inconsistent quality**: Using weak models for complex reasoning tasks (gap analysis, decisions)
- **Migration complexity**: Scaling to 200+ agents without clear assignment criteria
- **No escalation path**: No defined triggers for quality vs. cost trade-offs
- **Portability issues**: Strategy not documented, relies on tribal knowledge

### Opportunity

A comprehensive model-per-task strategy will:
1. **Optimize costs**: Route simple tasks to free local Ollama models (40-60% cost reduction target)
2. **Preserve quality**: Keep complex reasoning on Claude Sonnet/Opus where needed
3. **Enable scaling**: Provide clear criteria for assigning models to new agents
4. **Support experimentation**: Define escalation triggers for quality/cost trade-offs
5. **Unblock downstream work**: WINT-0230 (unified interface), WINT-0240 (Ollama fleet), WINT-0250 (escalation triggers)

---

## Goal

Define and document a comprehensive model-per-task strategy that maps each workflow task type to an optimal model tier, establishes decision criteria, documents escalation triggers, and provides a migration path from the current informal system to a formal, version-controlled strategy.

## Architecture Notes

### Design Principles

1. **Agent-First Routing**: Use agent name as primary routing key (more granular than task type)
2. **Tier-Based Selection**: Define clear tiers (0-3) with objective criteria
3. **Cost Optimization**: Default to free local models where quality sufficient
4. **Quality Gates**: Preserve Claude Sonnet/Opus for reasoning-critical tasks
5. **Graceful Degradation**: Always have fallback when primary model unavailable
6. **Versioned Configuration**: Strategy is version-controlled, reviewable, auditable

### Configuration Structure

**Two-file approach** for human + machine readability:

```
packages/backend/orchestrator/docs/
  WINT-0220-STRATEGY.md        # Human-readable (rationale, examples, migration)
  WINT-0220-STRATEGY.yaml      # Machine-readable (tier definitions, mappings)
```

**YAML Schema** (validated with Zod):
```yaml
strategy_version: "1.0.0"
effective_date: "2026-02-15"
review_date: "2026-03-15"

tiers: [...]           # 4 tier definitions
task_types: [...]      # Task type → tier mappings
escalation_triggers:   # Quality/cost/failure triggers
  quality: [...]
  cost: [...]
  failure: [...]
cost_analysis:         # Baseline vs. projected
  current_estimate: "..."
  proposed_estimate: "..."
  savings_target: "..."
  assumptions: [...]
```

### Backward Compatibility Strategy

**Approach**: Additive changes only, no breaking changes

1. **Existing agents continue working**: `model:` frontmatter still honored
2. **Tier field optional**: Agents without tier use fallback mapping
3. **Loader extension**: `model-assignments.ts` loader checks for tier, falls back to model name

```typescript
// Backward compatibility layer
const MODEL_TO_TIER: Record<string, number> = {
  'opus': 0,
  'sonnet': 1,
  'haiku': 2,
  'ollama': 3,
}

export function getModelTier(agentName: string): number {
  const assignment = modelAssignments[agentName]

  // New tier-based assignment
  if (assignment?.tier !== undefined) return assignment.tier

  // Fallback to model name mapping
  const modelType = assignment?.model.toLowerCase()
  return MODEL_TO_TIER[modelType] ?? 1 // default to Tier 1 (Sonnet)
}
```
