# WINT-0150: Create doc-sync Skill

## Context

The doc-sync command and agent already exist in the codebase at `.claude/commands/doc-sync.md` and `.claude/agents/doc-sync.agent.md`, providing automated synchronization of workflow documentation when agent/command files change. However, there is currently no formal Skill definition that wraps this functionality and provides the standardized user-facing interface.

**Problem:**
Skills provide a consistent, well-documented interface for reusable workflow capabilities. Without a Skill wrapper, the doc-sync functionality lacks:
- Standardized YAML-based metadata for tooling integration
- Comprehensive execution phase documentation in the expected format
- Clear parameter and flag specifications in the Skill format
- Integration patterns with other workflows documented in one place
- Discoverability through the standard Skills directory structure

**Existing Assets:**
- `/doc-sync` command definition (v1.0.0) with full usage documentation
- `doc-sync.agent.md` worker agent (v1.0.0) with complete implementation
- Established Skill patterns from `/review` and `/qa-gate`
- Section mapping documentation in the agent file

## Goal

Create a Skill definition file at `.claude/skills/doc-sync/SKILL.md` that provides a standardized, complete interface to the doc-sync functionality, enabling users to understand and invoke documentation synchronization capabilities through the established Skills pattern.

The Skill will serve as the canonical user-facing documentation and integration point for the doc-sync capability.

## Non-Goals

- **Do not modify the existing `doc-sync.agent.md`** - this story is only about creating the Skill wrapper
- **Do not change the existing `/doc-sync` command implementation** - only document it in Skill format
- **Do not implement new features** - this is documentation and standardization only
- **Do not modify other Skills** - this is a standalone Skill creation
- **Do not create unit tests** - Skills are documentation artifacts, not executable code
- **Do not change the doc-sync workflow** - preserve existing behavior exactly
