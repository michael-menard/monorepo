# AUDT-0010: Polish Code Audit Graph & Schemas

## Context

The Code Audit Engine (AUDT epic) provides a repeatable, multi-lens codebase analysis system built on LangGraph. The infrastructure has been scaffolded in a prior implementation session:

- **LangGraph graph**: `code-audit.ts` with pipeline and roundtable routing modes

## Architecture Notes

### LangGraph Graph Structure

**Pipeline mode** (fast, skip debate):
```
START → scan_scope → [9 lens nodes parallel] → synthesize → deduplicate → persist_findings → persist_trends → END
```

**Roundtable mode** (thorough, include debate):
```
START → scan_scope → [9 lens nodes parallel] → devils_advocate → roundtable → synthesize → deduplicate → persist_findings → persist_trends → END
```

Conditional routing implemented via:
```typescript
const shouldUseRoundtable = (state: CodeAuditState) => state.mode === 'roundtable'
```

### Zod Schema Hierarchy

```
AuditFindingsSchema (top-level)
├── LensResultSchema[] (findings from each lens)
│   ├── finding_id
│   ├── title
│   ├── severity
│   ├── confidence
│   └── ...
├── ChallengeResultSchema[] (devil's advocate output)
├── RoundtableResultSchema (roundtable consensus)
├── DedupResultSchema (deduplication mapping)
└── TrendSnapshotSchema (aggregated trends)
```

All schemas use Zod-first pattern:
```typescript
const SchemaName = z.object({ ... })
type SchemaName = z.infer<typeof SchemaName>
```

### File Discovery Algorithm

scan-scope uses recursive directory walk:
1. Start at project root (cwd)
2. Recursively walk all directories
3. Exclude: `.git`, `node_modules`, `dist`, `.next`, `.turbo`, `coverage`
4. Include: `*.ts`, `*.tsx`, `*.js`, `*.jsx` (source files)
5. Categorize by path patterns:
   - `apps/web/**` → frontend
   - `packages/backend/**` → backend
   - `**/__tests__/**` → tests
   - `*.config.ts` → config
   - `packages/core/**` → shared
