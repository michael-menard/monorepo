# INFR-0020: Artifact Writer/Reader Service

## Context

The orchestrator package (`@repo/orchestrator`) contains low-level YAML persistence utilities (`YamlArtifactReader`, `YamlArtifactWriter`, `YamlArtifactBridge`) that directly manipulate workflow artifact files. These utilities are currently embedded in LangGraph nodes like `save-to-db` and `load-from-db`, requiring direct instantiation and manual configuration of dependencies like PathResolver and SurfaceNormalizer.

**Problem:**
There is no unified, high-level service API for reading and writing workflow artifacts. Current usage requires:
1. Direct instantiation of low-level classes (`YamlArtifactReader`, `YamlArtifactWriter`)
2. Manual configuration of PathResolver, SurfaceNormalizer
3. Repetitive error handling and logging
4. No centralized validation or caching strategy

This makes it difficult to:
- Consume artifacts from MCP tools (KBAR-0110, KBAR-0120, KBAR-0130)
- Build CLI commands that manipulate artifacts
- Ensure consistent artifact handling across the system
- Add cross-cutting concerns (caching, validation, audit logging)

**Reality Baseline:**
- Core artifact schemas exist: Story, Checkpoint, Scope, Plan, Evidence, Review, QA-Verify, Audit-Findings (completed in INFR-0110)
- Low-level persistence utilities deployed: YamlArtifactReader, YamlArtifactWriter, YamlArtifactBridge, PathResolver, SurfaceNormalizer
- INFR-0120 (Review/QA artifact schemas) is ready-to-work and may add additional artifact types
- INFR-0041 (Workflow Event SDK) completed, sets event schema patterns to follow

## Goal

Create a high-level `ArtifactService` that wraps existing low-level persistence utilities with a clean, typed API for reading and writing all artifact types. The service should:

1. Provide methods for reading/writing all artifact types with automatic validation
2. Handle configuration, path resolution, and error handling internally
3. Support both YAML-only and YAML+DB persistence modes
4. Auto-detect story stage when not explicitly provided
5. Expose a simple, consumable API for MCP tools, CLI commands, and LangGraph nodes

## Non-Goals

## Architecture Notes

### Service Design

**ArtifactServiceConfigSchema** (Zod):
```typescript
const ArtifactServiceConfigSchema = z.object({
  workspaceRoot: z.string(),
  mode: z.enum(['yaml', 'yaml+db']).default('yaml'),
  storyRepo: z.any().optional(), // Required if mode is 'yaml+db'
  workflowRepo: z.any().optional(), // Required if mode is 'yaml+db'
  feature: z.string().optional(), // Can be inferred from storyId
  stage: z.string().optional() // Will auto-detect if not provided
})

type ArtifactServiceConfig = z.infer<typeof ArtifactServiceConfigSchema>
```

**ArtifactService API:**
```typescript
class ArtifactService {
  // Read operations
  readStory(storyId: string, stage?: string): Promise<ArtifactReadResult<StoryArtifact>>
  readCheckpoint(storyId: string, stage?: string): Promise<ArtifactReadResult<Checkpoint>>
  readScope(storyId: string, stage?: string): Promise<ArtifactReadResult<Scope>>
  readPlan(storyId: string, stage?: string): Promise<ArtifactReadResult<Plan>>
  readEvidence(storyId: string, stage?: string): Promise<ArtifactReadResult<Evidence>>
  readReview(storyId: string, stage?: string): Promise<ArtifactReadResult<Review>>
  readQaVerify(storyId: string, stage?: string): Promise<ArtifactReadResult<QaVerify>>
  readAuditFindings(storyId: string, stage?: string): Promise<ArtifactReadResult<AuditFindings>>
  readAllArtifacts(storyId: string, stage?: string): Promise<ArtifactReadResult<AllArtifacts>>

  // Write operations
  writeStory(storyId: string, data: StoryArtifact, stage?: string): Promise<ArtifactWriteResult>
  writeCheckpoint(storyId: string, data: Checkpoint, stage?: string): Promise<ArtifactWriteResult>
  writeScope(storyId: string, data: Scope, stage?: string): Promise<ArtifactWriteResult>
  writePlan(storyId: string, data: Plan, stage?: string): Promise<ArtifactWriteResult>
  writeEvidence(storyId: string, data: Evidence, stage?: string): Promise<ArtifactWriteResult>
  writeReview(storyId: string, data: Review, stage?: string): Promise<ArtifactWriteResult>
  writeQaVerify(storyId: string, data: QaVerify, stage?: string): Promise<ArtifactWriteResult>
  writeAuditFindings(storyId: string, data: AuditFindings, stage?: string): Promise<ArtifactWriteResult>

  // Utility operations
  moveStoryToStage(storyId: string, fromStage: string, toStage: string): Promise<ArtifactWriteResult>

```

**Stage Auto-Detection Algorithm:**
```typescript
async detectStage(storyId: string): Promise<string | null> {
  const stages = ['backlog', 'elaboration', 'ready-to-work', 'in-progress', 'ready-for-qa', 'UAT']

  for (const stage of stages) {
    const storyPath = this.pathResolver.resolveStoryPath(storyId, stage)
    if (await fileExists(storyPath)) {
      return stage
    }
  }

  return null // Story not found in any stage
}
```
