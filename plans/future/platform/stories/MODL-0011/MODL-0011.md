# MODL-0011: Refactor Provider Base Class - DRY up Duplicated Methods

**Epic:** MODL (Model Experimentation)
**Type:** Tech Debt / Refactoring
**Priority:** P2
**Points:** 2
**Status:** uat
**Dependencies:** MODL-0010 (in UAT status)

---

## Problem Statement

MODL-0010 introduced a working multi-provider abstraction for LLMs but contains ~150 lines of duplicated code across the three provider implementations (OllamaProvider, OpenRouterProvider, AnthropicProvider). Methods like `getCachedInstance()`, `clearCaches()`, and the cache pattern in `getModel()` are identical across all providers, violating the DRY principle and increasing maintenance burden.

**Technical Debt Origin:** Accepted during MODL-0010 code review iteration 4 to expedite MVP delivery.

---

## Goal

Refactor the provider architecture to eliminate method-level duplication by extracting common caching and lifecycle logic into an abstract `BaseProvider` class, reducing code by ~40% while maintaining full backward compatibility.

## Acceptance Criteria

### AC-1: Abstract BaseProvider Class
**Given** a provider needs common caching and lifecycle methods
**When** I create a new BaseProvider abstract class
**Then** it should:
- Implement `ILLMProvider` interface
- Provide `getCachedInstance(configHash: string): BaseChatModel | null`
- Provide `clearCaches(): void` for test cleanup
- Provide template method pattern for `getModel()` with hooks for provider-specific logic
- Maintain static caches: `configCache`, `instanceCache`, `availabilityCache`

**Verification:**
```typescript
// base.ts should export:
export abstract class BaseProvider implements ILLMProvider {
  protected static configCache: ProviderConfig | null = null
  protected static instanceCache: Map<string, BaseChatModel> = new Map()
  protected static availabilityCache: AvailabilityCache | null = null

  public static getCachedInstance(configHash: string): BaseChatModel | null { ... }
  public static clearCaches(): void { ... }
  public async getModel(modelName: string): Promise<BaseChatModel> { ... }

  // Abstract methods for provider-specific logic
  protected abstract loadConfig(): Promise<ProviderConfig>
  protected abstract createModel(model: ParsedModel, config: ProviderConfig): Promise<BaseChatModel>
  public abstract checkAvailability(forceCheck?: boolean): Promise<boolean>
}
```
