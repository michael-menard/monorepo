## Context

The WINT-0220 strategy defines a 4-tier model selection system (Tier 0: Critical/Opus, Tier 1: Complex/Sonnet, Tier 2: Routine/Ollama, Tier 3: Simple/Ollama) to optimize costs while preserving quality. The strategy is documented and validated in `/packages/backend/orchestrator/docs/WINT-0220-STRATEGY.md`, but there is currently no implementation of the tier-based routing logic.

**Current System State**:
- **Agent-level assignments** via `model-assignments.ts` loads agent→model mappings from YAML
- **Provider adapters** via MODL-0010 (completed) provide `ILLMProvider` interface with Ollama/Anthropic/OpenRouter implementations
- **Static model selection** where agents specify model in frontmatter with no dynamic tier selection
- **No escalation logic** for quality/cost/failure scenarios
- **No automatic fallback** when Ollama unavailable

**Dependencies Satisfied**:
- ✅ WINT-0220 strategy definition complete (UAT with qa_verdict: PASS)
- ✅ MODL-0010 provider adapter interfaces finalized (completed)
- ✅ `WINT-0220-STRATEGY.yaml` machine-readable spec available

## Problem

Without a unified model interface that implements the WINT-0220 strategy:
- Agents are statically assigned to models with no tier-based routing
- No escalation logic for quality/cost/failure scenarios
- Strategy configuration exists but isn't operationalized
- No automatic fallback when Ollama unavailable
- Cost optimization targets (60% reduction) cannot be achieved
- Agent invocations manually manage provider selection and fallback chains

## Goal

Create a unified model interface (`ModelRouter` or `UnifiedModelInterface`) that operationalizes the WINT-0220 strategy by:

1. Loading strategy configuration from `WINT-0220-STRATEGY.yaml`
2. Routing agent invocations to appropriate model tier based on agent name + task characteristics
3. Implementing escalation logic for quality thresholds, cost budgets, failure retries, and human-in-loop triggers
4. Providing fallback chains when primary model unavailable (Ollama down → Claude Haiku)
5. Maintaining backward compatibility with existing `model:` frontmatter assignments
6. Exposing clean API for orchestrator to request models without knowing tier logic

**Expected Outcome**: Orchestrator agents can request models via unified interface without managing provider selection, tier logic, or fallback chains. The interface automatically routes to optimal model tier based on strategy configuration, enabling 60% cost reduction target.

## Non-Goals

This story focuses on **interface implementation**, not orchestrator integration or ML-based optimization:

- ❌ **Integrating into Orchestrator Workflows**: Workflow integration is follow-up work
- ❌ **Model Cost Tracking**: Telemetry is WINT-0260 scope (depends on this story)
- ❌ **Quality Leaderboards**: Model performance evaluation is MODL-0030/0040 scope
- ❌ **Ollama Fleet Configuration**: Ollama model setup is WINT-0240 scope
- ❌ **Auto-Escalation Based on Complexity**: Complexity scoring is future ML work (requires WINT-5xxx ML pipeline)
- ❌ **Migrating Agent Assignments**: Strategy defines plan, migration is follow-up work
- ❌ **UI for Strategy Management**: Strategy editing is manual YAML updates (no UI in scope)

### AC-3: Escalation Logic Implementation

- [ ] **Quality escalation**: Retry with next higher tier on gate failure (max 3 retries → Tier 0 or human)
- [ ] **Cost de-escalation**: Downgrade non-critical tasks when budget threshold exceeded (80% warning, 95% critical)
- [ ] **Failure escalation**: Escalate to Tier 0 or human after 3 failures at current tier
- [ ] **Human-in-loop**: Pause workflow on low confidence (<50%), scope violations, security concerns
- [ ] All escalation paths terminate at Tier 0 or human review (no circular dependencies)

**Validation**:
```typescript
// Quality escalation
const result1 = await router.escalate({
  trigger: 'quality',
  currentTier: 2,
  retries: 1,
})
expect(result1.newTier).toBe(1)

// Max retries exhausted
const result2 = await router.escalate({
  trigger: 'quality',
  currentTier: 1,
  retries: 3,
})
expect(result2.requiresHuman).toBe(true)

// Cost de-escalation
const result3 = await router.escalate({
  trigger: 'cost',
  budgetUsed: 0.82,
  currentTier: 1,
})
expect(result3.newTier).toBe(2) // Downgrade to cheaper tier
```

```typescript
private static providerCache = new Map<string, ILLMProvider>()

getProvider(modelString: string): ILLMProvider {
  const cacheKey = generateConfigHash(modelString)
  if (this.providerCache.has(cacheKey)) {
    return this.providerCache.get(cacheKey)!
  }
  const provider = this.createProvider(modelString)
  this.providerCache.set(cacheKey, provider)
  return provider
}
```

**Availability Checking** (from `checkEndpointAvailability()`):
```typescript
async checkProviderAvailability(providerName: string): Promise<boolean> {
  const cacheKey = `availability:${providerName}`
  const cached = this.cache.get(cacheKey)
  if (cached && Date.now() - cached.timestamp < 30000) {
    return cached.available
  }

  const available = await this.performHealthCheck(providerName)
  this.cache.set(cacheKey, { available, timestamp: Date.now() })
  return available
}
```
