# WINT-0070: Create Workflow Tracking Tables

## Context

WINT-0010 (Create Core Database Schemas) established 6 schema groups in the `wint` PostgreSQL namespace, including the **Workflow Tracking schema group** with three tables:

1. **workflowExecutions** - Tracks workflow execution instances
2. **workflowCheckpoints** - Records checkpoint state during execution
3. **workflowAuditLog** - Comprehensive audit trail for workflow state changes

**Current Implementation Status**: These tables are **already implemented** in WINT-0010 (UAT status) at `/packages/backend/database-schema/src/schema/wint.ts` (lines 890-1012).

**Problem**: The story WINT-0070 ("Create Workflow Tracking Tables") appears to duplicate work already completed in WINT-0010. The index lists WINT-0070 as depending on WINT-0010 and blocking WINT-0080 (Seed Initial Workflow Data), which suggests the tables should already exist before seeding data.

**Discovery**: During seed generation, analysis revealed that the Workflow Tracking tables from WINT-0010 are comprehensive, well-tested (80%+ coverage), and include:
- UUID primary keys with defaultRandom()
- Timestamps with timezone support
- Foreign key relations with cascade delete
- Comprehensive indexing for query optimization
- Drizzle relations for lazy loading
- Auto-generated Zod schemas for validation

**Scope Clarification Needed**: Before implementation, stakeholders must clarify whether:
1. WINT-0070 is a duplicate and should be marked complete
2. WINT-0070 validates existing tables (validation-only approach)
3. WINT-0070 extends tables with additional capabilities not in WINT-0010

## Architecture Notes

### Database Schema Architecture

**Namespace**: `wint` (PostgreSQL schema)

**Isolation Pattern**:
```typescript
import { pgSchema } from 'drizzle-orm/pg-core'

export const wintSchema = pgSchema('wint')
```

**Table Definition Pattern** (from WINT-0010):
```typescript
export const workflowExecutions = wintSchema.table('workflow_executions', {
  id: uuid('id').primaryKey().defaultRandom(),
  executionId: varchar('execution_id', { length: 255 }).notNull().unique(),
  workflowName: varchar('workflow_name', { length: 255 }).notNull(),
  status: workflowStatusEnum('status').notNull(),
  startedAt: timestamp('started_at', { withTimezone: true }).notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  // ... additional fields
}, (table) => ({
  executionIdIdx: index('workflow_executions_execution_id_idx').on(table.executionId),
  workflowNameIdx: index('workflow_executions_workflow_name_idx').on(table.workflowName),
  compositeIdx: index('workflow_executions_name_status_idx').on(table.workflowName, table.status),
}))
```

**Relations Pattern**:
```typescript
export const workflowExecutionsRelations = relations(workflowExecutions, ({ many }) => ({
  checkpoints: many(workflowCheckpoints),
  auditLogs: many(workflowAuditLog),
}))
```

**Zod Schema Pattern**:
```typescript
import { createInsertSchema, createSelectSchema } from 'drizzle-zod'

export const insertWorkflowExecutionSchema = createInsertSchema(workflowExecutions)
export const selectWorkflowExecutionSchema = createSelectSchema(workflowExecutions)
```
