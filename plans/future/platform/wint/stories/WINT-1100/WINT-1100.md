# WINT-1100: Create Shared TypeScript Types for Unified WINT Schema

## Context

**Current State**:
WINT-1080 (Schema Reconciliation) has created a unified database schema in `packages/backend/database-schema/src/schema/wint.ts` with 30+ tables across 6 schema groups (core, context_cache, telemetry, ml, graph, workflow). The schema uses Drizzle ORM v0.44.3 with drizzle-zod integration for auto-generated Zod schemas.

**Problem**:
Currently, LangGraph repositories (`story-repository.ts`, `workflow-repository.ts`) and MCP tools define their own local Zod schemas inline, creating several issues:
- **Duplication**: Same schemas defined in multiple places (e.g., `StoryRowSchema` in story-repository, session schemas in MCP tools)
- **Drift Risk**: Schema changes require coordinated updates across multiple files
- **Type Safety Gaps**: No shared type contracts at system boundaries where LangGraph and MCP tools interact
- **Missed Optimization**: drizzle-zod auto-generates schemas from Drizzle tables, but these aren't exported in a shared package

**Dependencies**:
- **Depends On**: WINT-1080 (Schema Reconciliation) - creates the unified database schema foundation
- **Blocks**: WINT-1090 (LangGraph Repository Updates) - repositories will consume these shared types
- **Blocks**: WINT-1110 (Migrate Existing LangGraph Data) - migration needs shared types for validation

## Goal

Create a shared types package in `packages/backend/orchestrator/src/__types__/` that:
1. Re-exports auto-generated Zod schemas from `packages/backend/database-schema` (drizzle-zod generated)
2. Adds custom Zod schemas for MCP tool inputs/outputs that aren't direct database mappings
3. Provides a single source of truth for all WINT-related types
4. Enables both LangGraph nodes and MCP tools to import from the same location

This establishes type-safe communication between LangGraph workflows and MCP tools using the unified schema from WINT-1080.

## Non-Goals

- **Creating New Database Tables**: This story only creates TypeScript/Zod type wrappers for existing tables from WINT-1080
- **Migrating MCP Tools**: MCP tools will migrate to shared types in separate stories (deferred to WINT-0090+)
- **Creating New Schemas**: Only re-export existing auto-generated schemas from drizzle-zod
- **Database Migrations**: No database schema changes - only TypeScript type definitions
- **LangGraph Graph Updates**: Do not modify LangGraph graphs/nodes (deferred to WINT-9000+ series)

### Negative Tests

1. **Invalid Schema Inputs**:
   - Wrong type for required field → schema.parse() throws
   - Missing required field → schema.parse() throws
   - Extra unknown fields → schema.parse() strips or throws (depending on strict mode)

2. **Type Mismatches**:
   - Insert schema with select-only fields → validation error
   - Select schema missing insert-required fields → compilation error

3. **Import Failures**:
   - Incorrect import path → TypeScript compilation error
   - Circular dependency → build failure
