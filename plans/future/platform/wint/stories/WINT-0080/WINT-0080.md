# WINT-0080: Seed Initial Workflow Data

## Context

**Background:** WINT-0070 (Workflow Tracking Tables) and WINT-0060 (Graph Relational Tables) have created the schema structure for workflow tracking and graph capabilities. These tables are currently empty and need foundational reference data to support workflow operations.

**Problem:** Workflow operations require reference data for:

1. **Phases (0-7)**: The 8 workflow phases defined in the WINT epic (Phase 0: Bootstrap, Phase 1: Foundation, Phase 2: Context Cache, Phase 3: Telemetry, Phase 4: Graph & Cohesion, Phase 5: ML Pipeline, Phase 6: Batch Mode, Phase 7: Migration)

2. **CRUD Capabilities**: Standard capabilities (create, view, edit, delete, upload, replace, download) referenced in user-flows.schema.json (WINT-0200)

3. **Agent Metadata**: Current agent inventory (143 agents based on file count) with metadata extracted from .agent.md frontmatter

4. **Command Metadata**: Current command inventory (33 commands based on file count)

5. **Skill Metadata**: Current skill inventory (14 skills based on directory count)

Without this seed data, workflow tracking and graph cohesion features cannot function.

**Current State (from Reality Baseline 2026-02-13):**
- `workflow.phases` table exists but is empty (WINT-0070)
- `graph.capabilities` table exists but is empty (WINT-0060)
- Agent reference table exists but is empty
- Command reference table exists but is empty
- Skill reference table exists but is empty
- 143 `.agent.md` files in `.claude/agents/` directory
- 33 command files in `.claude/commands/` directory
- 14 skill directories in `.claude/skills/` directory
- WINT stories index contains 140 stories across 8 phases

---

## Goal

Create idempotent seed data scripts that populate foundational reference tables with current workflow inventory:

1. **Workflow Phases**: Extract 8 phases from WINT stories index → `workflow.phases` table
2. **CRUD Capabilities**: Seed 7 standard operations → `graph.capabilities` table
3. **Agent Metadata**: Parse all `.agent.md` files → agent reference table
4. **Command Metadata**: Parse all command files → command reference table
5. **Skill Metadata**: Parse all skill directories → skill reference table

All seed scripts will be transaction-safe, idempotent (safe to rerun), and validate data with Zod schemas before insertion.

### Parser Strategy

**Frontmatter Parser (Generic):**
- Input: File path
- Output: Parsed object or null
- Error handling: Try/catch per file, log warnings, continue
- Fallbacks: Use filename if name missing, null for optional fields

**Index Parser (Phases):**
- Input: `stories.index.md` path
- Output: Array of 8 phase objects
- Logic: Regex match on `## Phase N:` headers, extract description
- Validation: Expect exactly 8 phases, warn if different

**Metadata Extractor (Agents/Commands/Skills):**
- Input: File/directory path
- Output: Metadata object
- Logic: Read frontmatter, extract known fields, ignore unknowns
- Validation: Required fields present, types correct

### Idempotency Strategy

Use `ON CONFLICT DO NOTHING` with unique constraints:

| Table | Unique Constraint | Conflict Resolution |
|-------|-------------------|---------------------|
| `workflow.phases` | `id` (PK) | DO NOTHING |
| `graph.capabilities` | `name` (UNIQUE) | DO NOTHING |
| `wint.agents` | `name` (UNIQUE) | DO NOTHING |
| `wint.commands` | `name` (UNIQUE) | DO NOTHING |
| `wint.skills` | `name` (UNIQUE) | DO NOTHING |

**Verification:** Must confirm unique constraints exist in schema files before implementation.

### Transaction Strategy

**Single Transaction for Full Seed:**
```typescript
await db.transaction(async (tx) => {
  await seedPhases(tx)
  await seedCapabilities(tx)
  await seedAgents(tx)
  await seedCommands(tx)
  await seedSkills(tx)
})
```

**Benefits:**
- Atomic: All or nothing
- Rollback on any error
- No partial state

**Risk:** 205 inserts may timeout (unlikely with modern Postgres)
**Mitigation:** Monitor duration, add batching only if needed

**Pre-flight Check:**
```typescript
async function verifyTablesExist(db: Database) {
  const requiredTables = [
    'workflow.phases',
    'graph.capabilities',
    'wint.agents',
    'wint.commands',
    'wint.skills'
  ]

  for (const table of requiredTables) {
    const [schema, name] = table.split('.')
    const exists = await db.execute(sql`
      SELECT EXISTS (
        SELECT 1 FROM information_schema.tables
        WHERE table_schema = ${schema}
        AND table_name = ${name}
      )
    `)

    if (!exists) {
      throw new Error(`Required table ${table} does not exist`)
    }
  }
}
```
