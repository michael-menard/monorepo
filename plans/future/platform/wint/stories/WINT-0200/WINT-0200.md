# WINT-0200: Create User Flows Schema with State/Capability Enums — PM Artifacts and Example

## Context

The `user-flows.schema.json` was created during the WINT Phase 0 bootstrap process. The file exists at `packages/backend/orchestrator/src/schemas/user-flows.schema.json` and contains the complete schema: `UserFlowState` enum (`loading`, `empty`, `validation_error`, `server_error`, `permission_denied`), `UserFlowCapability` enum (`create`, `view`, `edit`, `delete`, `upload`, `replace`, `download`), `maxItems: 5` on flows, and `maxItems: 7` on steps and capabilities.

The schema is referenced by WINT-0190 (`patch-plan.schema.json`) as the canonical JSON Schema pattern for this project, and by WINT-0210 (`Populate Role Pack Templates`) as a gate dependency. WINT-0210 cannot complete its PO role pack (AC-7) without WINT-0200 being formally complete.

The formal story PM artifacts have not been generated: no STORY-SEED (until now), no test plan, no dev-feasibility review, no story document, and no canonical `user-flows.example.json` file. The schema implementation predates the story generation workflow. This story retroactively produces all missing artifacts.

**Status discrepancy resolution**: The per-epic index marked WINT-0200 as `done` (Phase 0 bootstrap task completed — the schema was written). The platform index marked it `ready-to-work` (formal story artifacts not generated). This story resolves the discrepancy by generating the PM artifacts. The implementation scope is verification + example creation, not schema re-implementation.

Verify the existing `user-flows.schema.json` against the spec, produce a canonical `user-flows.example.json` with AJV validation evidence, and generate the complete story PM artifacts. Upon completion, WINT-0210 is unblocked.

## Architecture Notes

- JSON Schema draft 2020-12 is the established standard (`$schema: https://json-schema.org/draft/2020-12/schema`)
- `$id` follows canonical URL pattern: `https://lego-moc-platform.com/schemas/user-flows.schema.json`
- `$defs` used for all reusable sub-types (`UserFlowState`, `UserFlowCapability`, `UserFlowStep`, `UserFlow`)
- `maxItems` enforced at schema level (not just documentation) — this is the hard enforcement mechanism per WINT-0190 lesson
- JSON Schema draft 2020-12 cannot enforce sequential ordering of enum values within arrays — ordering is a documented convention; runtime enforcement is separate scope
