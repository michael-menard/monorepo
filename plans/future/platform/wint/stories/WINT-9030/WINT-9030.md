# WINT-9030: Create cohesion-prosecutor LangGraph Node (`nodes/story/cohesion-check.ts`)

## Context

The cohesion-prosecutor is a Product Owner agent that enforces feature completeness by ensuring every feature in the capability graph has all required CRUD lifecycle stages before being marked done. In the LangGraph parity phase (Phase 9), the core business logic must be ported to a native TypeScript LangGraph node so that feature cohesion checks operate identically in both the Claude Code agent workflow and the LangGraph orchestration workflow.

This story creates `packages/backend/orchestrator/src/nodes/story/cohesion-check.ts`, wrapping the cohesion sidecar's compute logic (`computeCheck` for per-feature checks, `computeAudit` for full graph audits) as an injectable, testable LangGraph node. The established pattern from `nodes/sync/doc-sync.ts` (native 7-phase sequential architecture, `createToolNode`, `updateState`, injectable config) is the direct implementation template.

**Dependency gate**: WINT-4010 (Create Cohesion Sidecar) is currently in `failed-code-review`. Implementation of this story should be gated on WINT-4010 reaching at least `ready-for-qa` to ensure a stable import API. Two options are available pending stabilization: (a) import `computeCheck`/`computeAudit` directly once WINT-4010 stabilizes, or (b) self-contain the compute logic by copying the Drizzle query pattern with a `// TODO: deduplicate with WINT-4010 once stable` comment.

---

## Goal

Port the cohesion-prosecutor's feature completeness check logic to a native LangGraph node so that feature cohesion audits run identically in both the Claude Code agent workflow and the LangGraph orchestration workflow.

---

## Examples

**Example 1 — Per-feature check (happy path):**
Given a `featureId: 'wish-list'` with capabilities rows for `create`, `read`, `update`, and `delete` in the graph DB, calling `cohesionCheckImpl({ featureId: 'wish-list', db })` produces:
```json
{ "status": "complete", "violations": [], "capabilityCoverage": { "create": true, "read": true, "update": true, "delete": true } }
```

**Example 2 — Full audit (happy path):**
Given `packageName: 'wint'` with 3 features, two fully covered and one missing `delete`, calling `cohesionAuditImpl({ packageName: 'wint', db })` produces a `CohesionAuditResult` containing all 3 features in the franken-features list with a coverage summary marking the third feature as incomplete.

---

## Edge Cases

**Edge Case 1 — Unknown feature ID:**
Given `featureId: 'does-not-exist'` with no rows in the `capabilities` table, the node returns `{ status: 'unknown', violations: [], capabilityCoverage: {} }` gracefully without throwing — matching the sidecar's contract for unknown features (AC-7).

**Edge Case 2 — Partial capability coverage:**
Given `featureId: 'partial-feat'` with only `create` and `read` capabilities present, the node returns `{ status: 'incomplete', violations: ['missing update capability', 'missing delete capability'], capabilityCoverage: { create: true, read: true, update: false, delete: false } }` — identical to the sidecar's response for the same input (parity requirement, AC-14).

**Edge Case 3 — Empty graph:**
Given `packageName: 'empty-pkg'` with no features or capabilities rows, `cohesionAuditImpl` returns a `CohesionAuditResult` with an empty franken-features array and zeroed coverage summary — no panic or null-pointer errors.

## Architecture Notes

### Node Structure (from doc-sync pattern)

```typescript
// cohesion-check.ts — following doc-sync.ts gold standard

// 1. Zod state extension schema (NEVER TypeScript interface)
const GraphStateWithCohesionCheckSchema = GraphStateSchema.extend({
  cohesionCheck: CohesionCheckResultSchema.optional(),
  cohesionAudit: CohesionAuditResultSchema.optional(),
})
type GraphStateWithCohesionCheck = z.infer<typeof GraphStateWithCohesionCheckSchema>

// 2. Config schema
const CohesionCheckConfigSchema = z.object({
  db: z.custom<DrizzleDb>(),
  featureId: z.string().optional(),
  packageName: z.string().optional(),
})

// 3. Inner compute function (injectable db, unit-testable)
async function cohesionCheckImpl(config: z.infer<typeof CohesionCheckConfigSchema>): Promise<...>

// 4. Node export via createToolNode
export const cohesionCheckNode = createToolNode(cohesionCheckImpl, defaultConfig)
export function createCohesionCheckNode(config) { return createToolNode(cohesionCheckImpl, config) }
```
