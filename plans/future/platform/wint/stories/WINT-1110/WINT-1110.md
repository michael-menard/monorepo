# WINT-1110: Migrate Existing LangGraph Data

## Summary

Create a migration script to move any existing data from LangGraph tables (PostgreSQL at port 5433, `public` schema) to the unified WINT schema (PostgreSQL at port 5432, `wint` schema).

## Context

WINT-1080 established the unified schema specification (WINT is source of truth).
WINT-1090 updated the LangGraph TypeScript repositories to use the unified wint schema.

This story (WINT-1110) creates the actual data migration script to move existing data from the LangGraph `public` schema to the unified `wint` schema, with rollback support.

## Source: LangGraph Database (port 5433, public schema)

Tables to migrate:
- `public.stories` → `wint.stories`
- `public.features` → `wint.features`
- `public.workflow_events` → `wint.state_transitions` + `wint.workflow_audit_log`

Tables to skip (stay in LangGraph DB):
- `public.elaborations` (LangGraph workflow specific)
- `public.gaps` (LangGraph workflow specific)
- `public.implementation_plans` (LangGraph workflow specific)
- `public.verifications` (LangGraph workflow specific)
- `public.proofs` (LangGraph workflow specific)
- `public.adrs` (LangGraph KB specific)

## Target: WINT Database (port 5432, wint schema)

The unified schema is already in place. Migration populates existing tables.

## Key Transformations Required

### Enum Normalization (Stories)
- `ready-to-work` → `ready_to_work`
- `in-progress` → `in_progress`
- `ready-for-qa` → `ready_for_qa`
- `uat` → `in_qa`
- `backlog`, `done` → unchanged

### Column Mapping (Stories)
- `public.stories.story_id` → `wint.stories.story_id` (same)
- `public.stories.goal` → `wint.stories.goal`
- `public.stories.non_goals` → `wint.stories.non_goals`
- `public.stories.packages` → `wint.stories.packages`
- `public.stories.surfaces` → `wint.stories.surfaces`
- `public.stories.blocked_by` → `wint.stories.blocked_by`
- `public.stories.depends_on` → `wint.stories.depends_on`
- `public.stories.follow_up_from` → `wint.stories.follow_up_from`

### Column Mapping (Features)
- `public.features.name` → `wint.features.feature_name`
- `public.features.description` → `wint.features.description`

### Workflow Events → WINT
- `public.workflow_events` → `wint.workflow_audit_log` (entity audit)
- State transition events → also go to `wint.state_transitions`

## Acceptance Criteria

1. **AC-001**: Migration script connects to both databases (port 5432 and 5433) successfully
2. **AC-002**: All stories from LangGraph `public.stories` are migrated to `wint.stories` with correct enum normalization
3. **AC-003**: All features from LangGraph `public.features` are migrated to `wint.features`
4. **AC-004**: Duplicate handling: existing records in `wint.stories` are not overwritten (UPSERT with ON CONFLICT DO NOTHING)
5. **AC-005**: Migration script logs progress and row counts (migrated, skipped, errors)
6. **AC-006**: Rollback script provided that can undo the migration
7. **AC-007**: Migration is idempotent - running twice produces same result
8. **AC-008**: All migrated stories have valid story_state enum values in `wint` schema

## Infrastructure

- Migration TypeScript script: `packages/backend/orchestrator/src/scripts/migrate-langgraph-data.ts`
- Rollback script: `packages/backend/orchestrator/src/scripts/migrate-langgraph-data-rollback.ts`

## Risk Notes

- Must test thoroughly before production run
- Must handle case where LangGraph DB is empty or unreachable
- Must not overwrite existing WINT data
- pgvector embeddings from LangGraph can be migrated as-is (nullable column in wint schema)
