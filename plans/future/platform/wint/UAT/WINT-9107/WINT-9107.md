# WINT-9107: Implement Node-Level Retry & Circuit Breaker Middleware for LangGraph Nodes

## Context

**Phase 9 — LangGraph Parity.** The WINT workflow is being ported to LangGraph, and individual LangGraph nodes require resilient execution: retry on transient failures and circuit-break when a downstream service is consistently failing.

**Existing Reality.** The orchestrator package already contains a complete runner infrastructure in `packages/backend/orchestrator/src/runner/` that implements:
- `NodeCircuitBreaker` class with CLOSED/OPEN/HALF_OPEN state machine
- `withNodeRetry()` with exponential backoff, jitter, and error classification integration
- `createNode()` factory that composes circuit breaker + retry + timeout + logging
- `createLLMNode()`, `createToolNode()`, `createSimpleNode()` convenience factories
- `RETRY_PRESETS` per node type (llm, tool, validation — defined inline in node-factory.ts)
- Full error hierarchy: `NodeRetryExhaustedError`, `NodeCircuitOpenError`, `NodeTimeoutError`, `NodeExecutionError`
- `classifyError()` and `isRetryableNodeError()` for retry decision routing
- `createBlockedUpdate()` and `createErrorUpdate()` state helpers that produce CIRCUIT_OPEN / RETRY_EXHAUSTED codes in GraphState updates
- Clean named-export surface via `packages/backend/orchestrator/src/runner/index.ts`

**Problem.** It is unclear whether this existing runner infrastructure fully satisfies the requirements defined in WINT-9105 (LangGraph Error Handling & Retry Strategy ADR), and whether it is properly discoverable and documented for node authors building new LangGraph nodes. The middleware may exist but not be validated against the specification.

**Dependency notes.** WINT-9010 (`packages/backend/workflow-logic/`) is in-qa. Any re-export of middleware types from that package is conditional on WINT-9010 QA passing. AC-7 (import path documentation) is satisfiable via `runner/index.ts` alone without waiting for WINT-9010.

Validate and surface the existing node-level retry + circuit breaker middleware as a documented, tested, and isolated API that WINT-9110 (Full Workflow LangGraph Graphs) can rely on with confidence. Close any gaps found between WINT-9105 requirements and the current runner implementation.

## Architecture Notes

**Circuit breaker isolation (AC-2):** Verified by code inspection — `createNode()` instantiates `new NodeCircuitBreaker(nodeConfig.circuitBreaker)` locally on line 125 of `node-factory.ts`. Each `createNode()` call produces a new closure over a distinct `NodeCircuitBreaker` instance. This matches the correct design: per-node, in-memory, not shared across node registrations. A test must explicitly assert this property.

**Retry preset design:** `createLLMNode()`, `createToolNode()`, and `createSimpleNode()` are convenience wrappers that inline preset configurations (maxAttempts, backoffMs, backoffMultiplier, maxBackoffMs, timeoutMs, jitterFactor) directly. These are the effective presets. `RETRY_PRESETS` as a named export is referenced in types.ts but the actual preset values live in node-factory.ts's convenience factories. The node author guide must clarify which factory to use for each node type.

**GraphState routing:** `createBlockedUpdate()` from `state-helpers.ts` sets the routing flags and error code. The existing factory already calls this for both `NodeRetryExhaustedError` (code: `RETRY_EXHAUSTED`) and `NodeCircuitOpenError` (code: `CIRCUIT_OPEN`). Integration tests must verify the actual state output structure, not just the thrown error types.

**HALF_OPEN recovery (critical test gap):** The HALF_OPEN state in `NodeCircuitBreaker` is exercised by waiting past `recoveryTimeoutMs`. Unit tests must use `vi.useFakeTimers()` for determinism. UAT integration tests must use real timing per ADR-005.
