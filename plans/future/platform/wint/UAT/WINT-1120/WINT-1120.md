# WINT-1120: Validate Foundation Phase

Validate Foundation Phase — Story CRUD, Shim, Commands, Unified Schema, and Worktree Integration

---

## Context

Phase 1 of the WINT workflow automation initiative has assembled a database-first story lifecycle management system. Across stories WINT-1011 through WINT-1160, the following five infrastructure areas have been built:

1. **Story CRUD via DB** (`core.stories` table + four MCP tools via WINT-0090): `storyGetStatus`, `storyUpdateStatus`, `storyGetByStatus`, `storyGetByFeature` — live and in UAT.
2. **Compatibility Shim** (WINT-1011, WINT-1012): Four shim functions provide DB-first reads with directory fallback and DB-only writes. `ShimDiagnostics` opt-in field (`source: db | directory | not_found`) enables observability. Designed for deletion in WINT-7100.
3. **Three Updated Commands** — `/story-status` (WINT-1040), `/story-update` (WINT-1050, v3.0.0), `/story-move` (WINT-1060) — each augmented to write/read via DB. All include `db_updated: true` in result YAML.
4. **Unified Schema** — LangGraph repositories (`story-repository.ts`, `workflow-repository.ts`) and Claude Code MCP tools operate on the same physical `wint.stories` table, with shared Zod types from WINT-1100.
5. **Worktree Integration** — `core.worktrees` table (WINT-1130), auto-registration on story start (WINT-1140), conflict detection for parallel work (WINT-1160), and cleanup on completion (WINT-1150).

This validation story exercises all five areas end-to-end against the live `postgres-knowledgebase` (port 5433) to confirm the foundation is solid before Phase 2 (Context Cache) work begins. The goal is **proof, not repair** — if failures are found, fix stories are filed and WINT-1120 receives a CONDITIONAL PASS verdict.

**Baseline:** `plans/baselines/BASELINE-REALITY-2026-02-13.md` — note the baseline predates Phase 1 UAT completions (WINT-1011, WINT-1012, WINT-1030, WINT-1130, WINT-1140, WINT-1150 all moved to UAT after baseline was cut).

## Architecture Notes

### DB-First Pattern Under Verification

The shim (`mcp-tools/src/story-compatibility/index.ts`) implements:
- **DB-hit path**: queries `core.stories` first; if found, returns immediately with `ShimDiagnostics.source = 'db'`. No directory scan.
- **DB-miss path**: if story absent from DB, falls back to directory scan using WINT-1030 swim-lane mapping. Returns with `ShimDiagnostics.source = 'directory'`.
- **DB-only write**: `shimUpdateStoryStatus` writes to DB only. If story not in DB, returns `null` + WARNING. No FS write on failure.

### Unified Schema Verification

Both LangGraph (`StoryRepository`) and Claude Code MCP tools (`storyGetStatus`) must point to the same physical table. Pre-check:
```sql
SELECT table_schema, table_name
FROM information_schema.tables
WHERE table_name = 'stories'
```
Expected: single row with schema `wint` (or `core` — confirm per WINT-1080 outcome).

### LangGraph Integration Point

`packages/backend/orchestrator/src/db/story-repository.ts` uses DI pattern — inject `@repo/db` client with real connection to postgres-knowledgebase port 5433 for integration testing. Do not use test/mock DB.
