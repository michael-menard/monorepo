# WINT-9110: Create Full Workflow LangGraph Graphs

> Phase 9 ظ¤ Full workflow LangGraph parity for the six major orchestration workflows.

---

## Context

The orchestrator currently has several specialized LangGraph graphs: `story-creation`, `elaboration`, `qa`, `review`, `merge`, `code-audit`, `metrics`, and `doc-graph`. These were built incrementally as individual APIP/WINT stories were implemented.

Phase 9 (WINT-9xxx) is porting the remaining Claude Code agent-based workflows to native LangGraph `StateGraph` instances to achieve full parity. The six major orchestration workflows ظ¤ **bootstrap**, **elab-epic**, **elab-story**, **dev-implement**, **qa-verify**, and **backlog-review** ظ¤ currently execute via Claude Code agent scripts or YAML-configured orchestration. They are not yet expressed as first-class LangGraph graphs, which means they cannot benefit from:

- The PostgreSQL checkpointer (WINT-9106) for resumable state
- Retry middleware (WINT-9107) for fault tolerance
- Parallel execution via Send API
- Unified thread-based state management across the full pipeline

**Conflict ظ¤ missing dependencies (warning, non-blocking):** WINT-9060 (batch-coordinator), WINT-9070 (backlog-curator node), WINT-9080 (ML pipeline nodes), and WINT-9100 (telemetry nodes) are all pending. WINT-9110's full workflow graphs depend on these nodes. Resolution: design all graphs to accept node dependencies as `z.unknown().optional()` injectable parameters, wire with stub implementations initially. Real nodes are dropped in as those stories land.

**Conflict ظ¤ middleware/checkpointer not yet available (warning, non-blocking):** WINT-9105 ADR, WINT-9106 checkpointer, and WINT-9107 middleware are all pending. Resolution: design graphs with explicit injection points typed as `z.unknown().optional()`. Apply existing try/catch pattern from `story-creation.ts` as interim error handling.

## Architecture Notes

### Graph Pattern (follow for all 6 graphs)

```typescript
// 1. Config Schema ظ¤ Zod with defaults + optional injected deps
export const XGraphConfigSchema = z.object({
  storyId: z.string().regex(/^[A-Z]{2,10}-\d{3,5}$/),
  attempt: z.number().int().positive().default(1),
  nodeTimeoutMs: z.number().positive().default(30000),
  retryMiddleware: z.unknown().optional(),   // WINT-9107 injection point
  checkpointer: z.unknown().optional(),       // WINT-9106 injection point
  telemetryNode: z.unknown().optional(),      // WINT-9100 injection point
  // graph-specific injectable deps below...
})

// 2. State Annotation ظ¤ overwrite reducer for most fields, append for arrays
const overwrite = <T>(_: T, b: T): T => b
export const XGraphStateAnnotation = Annotation.Root({
  phase: Annotation<string>({ reducer: overwrite, default: () => 'initializing' }),
  errors: Annotation<string[]>({ reducer: (a, b) => [...a, ...b], default: () => [] }),
  warnings: Annotation<string[]>({ reducer: (a, b) => [...a, ...b], default: () => [] }),
  // ...
})

// 3. Conditional edge functions ظ¤ always named, never inline, EXPORTED for routing tests (AC-9)
export function afterInitialize(state: XGraphState): string {
  if (state.errors.length > 0) return END
  return 'next_node'
}

// 4. Graph factory
export function createXGraph(config: XGraphConfig) {
  const graph = new StateGraph(XGraphStateAnnotation)
  graph.addNode('initialize', createInitializeNode(config))
  // ...
  graph.addConditionalEdges('initialize', afterInitialize)
  graph.setEntryPoint(START)
  return graph.compile({ checkpointer: config.checkpointer })
}

// 5. Entry function
export async function runX(inputs, config): Promise<XGraphResult> {
  const parsedConfig = XGraphConfigSchema.parse(config)
  const threadId = `${parsedConfig.storyId}:x-graph:${parsedConfig.attempt}`
  const startedAt = Date.now()
  try {
    const result = await createXGraph(parsedConfig).invoke(inputs, { configurable: { thread_id: threadId } })
    return { storyId: parsedConfig.storyId, success: true, durationMs: Date.now() - startedAt, ... }
  } catch (error) {
    return { storyId: parsedConfig.storyId, success: false, errors: [String(error)], ... }
  }
}

// 6. Node adapter
export const createXNode = () => createToolNode({ name: 'x-graph', run: runX })
```

## Test Plan

Testing strategy: **routing validation + injectable node integration tests**. No live LLM calls, no subprocess execution.

### Per-graph test file structure

Each `__tests__/X.test.ts` must include:

1. **Routing tests** ظ¤ call each `afterX(state)` conditional edge function with mock state objects, assert correct next node name returned. Cover at least: success path, error path, and any branching conditions.

3. **Happy-path integration test** ظ¤ mock `graph.invoke`, call `runX()`, assert result envelope shape matches `XGraphResultSchema`.

4. **Error/exception test** ظ¤ mock `graph.invoke` to throw, call `runX()`, assert `success: false` with non-empty `errors` array.

### elab-epic.ts additional tests

- Fan-out count: assert dispatch node emits exactly N `Send` objects for N input storyIds

- Telemetry injection: assert telemetry node called at entry/exit when `config.telemetryNode` provided
