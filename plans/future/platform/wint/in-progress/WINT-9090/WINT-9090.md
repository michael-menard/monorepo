# WINT-9090: Create Context Cache LangGraph Nodes

Port context-warmer and session-manager agent logic to LangGraph TypeScript nodes in `packages/backend/orchestrator/src/nodes/context/`.

---

## Context

Phase 9 of the WINT epic converts Claude Code agent logic into native LangGraph TypeScript nodes so that context caching and session management work identically in both the Claude Code agent execution path and the LangGraph execution path.

Currently, context warming (pre-populating `context_cache.contextPacks` with project context, agent missions, domain KB, and library cache) and session lifecycle management (create/update/complete/cleanup of `context_cache.contextSessions`) are implemented as Claude Code agent skills and MCP tools.

The MCP tools in `packages/backend/mcp-tools/src/context-cache/` (WINT-0100, QA: PASS) and `packages/backend/mcp-tools/src/session-management/` (WINT-2100 planned) are the behavioral source of truth. This story ports those behaviors into two LangGraph nodes within a new `nodes/context/` directory, following the established Phase 9 porting pattern from WINT-9020 (`doc-sync.ts`).

**Implementation gate**: Do not start implementation until WINT-9010 (`@repo/workflow-logic`) reaches `uat` or `completed` status. Story generation and elaboration proceed now.

**Source ambiguity note**: WINT-2100 (session-manager agent) is `ready-to-work` (agent not yet written). For session-manager.ts, the behavioral source is the MCP tools in `packages/backend/mcp-tools/src/session-management/`. This must be formally confirmed during elaboration.

---

## Goal

Produce two LangGraph-compatible TypeScript nodes:

1. **`context-warmer.ts`** â€” wraps context cache read/write/invalidate operations (portying `contextCacheGet`, `contextCachePut`, `contextCacheInvalidate` business logic)
2. **`session-manager.ts`** â€” wraps session lifecycle operations (porting `sessionCreate`, `sessionUpdate`, `sessionComplete`, `sessionCleanup` business logic)

Both nodes live in `packages/backend/orchestrator/src/nodes/context/`, follow the `createToolNode` factory pattern, extend GraphState with injectable DB dependencies, and degrade gracefully on DB failure.

## Architecture Notes

### Node Structure Pattern (from WINT-9020/doc-sync.ts)

```typescript
// Config schema with Zod defaults
export const ContextWarmerConfigSchema = z.object({
  db: z.any().optional(),  // injectable DB client
  ttlSeconds: z.number().default(3600),
  // ... other config fields
})
export type ContextWarmerConfig = z.infer<typeof ContextWarmerConfigSchema>

// Extended GraphState interface
export interface GraphStateWithContextCache extends GraphStateWithContext {
  contextCacheHit?: boolean | null
  contextPackContent?: string | null
  contextCacheError?: string | null
}

// Factory function (enables DI for testing)
export function createContextWarmerNode(config: Partial<ContextWarmerConfig>) {
  const resolvedConfig = ContextWarmerConfigSchema.parse(config)
  return createToolNode('context-warmer', async (state: GraphStateWithContextCache) => {
    // ... implementation
    // Graceful degradation on DB error:
    try {
      // ... DB operations
    } catch (err) {
      logger.error('context-warmer: DB failure', { err })
      return { contextCacheHit: null, contextPackContent: null }
    }
  })
}

// Default zero-config export
export const contextWarmerNode = createContextWarmerNode({})
```

### Graceful Degradation Contract

Both nodes MUST:
1. Wrap all DB operations in try/catch
2. Log errors via `@repo/logger` at `error` level
3. Return `null` or `false` for state fields when DB unavailable
4. Never throw â€” always return a `Partial<ExtendedState>` object

This mirrors the pattern in `context-cache-get.ts`:
```typescript
} catch (error) {
  logger.error('contextCacheGet: failed', { error })
  return null  // caller handles null = cache miss
}
```
