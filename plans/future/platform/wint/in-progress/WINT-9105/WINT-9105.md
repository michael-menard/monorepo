# WINT-9105: Define LangGraph Error Handling & Retry Strategy

## Context

The LangGraph parity phase (Phase 9) is assembling production-grade workflow graphs to mirror and eventually replace Claude Code agent-based orchestration. Before any graph assembly story (WINT-9060, WINT-9110) begins, the error handling, retry, and fault-tolerance contracts must be defined in a single authoritative document.

The codebase contains two partially overlapping error handling foundations:

1. **Node runner module** (`packages/backend/orchestrator/src/runner/`) — implements `NodeCircuitBreaker` (failureThreshold=5, recoveryTimeoutMs=60000), `withNodeRetry` (exponential backoff + jitter), `classifyError` (retryable vs terminal), and `RETRY_PRESETS` (llm, tool, validation categories).

2. **Workflow error schema module** (`packages/backend/orchestrator/src/errors/workflow-errors.ts`) — implements `WorkflowErrorTypeSchema` (AGENT_SPAWN_FAILED, AGENT_TIMEOUT, MALFORMED_OUTPUT, PRECONDITION_FAILED, EXTERNAL_SERVICE_DOWN), `ERROR_RETRY_DEFAULTS`, and `CircuitBreakerConfigSchema` (failureThreshold=3, failureWindowMs=300000).

Neither was designed as the comprehensive LangGraph error strategy. They are building blocks that this story must synthesize into a binding ADR.

Additionally, WINT-0220 (Model-per-Task Strategy) defines a 3-provider escalation chain (ollama → openrouter → anthropic) across 4 model tiers with cost ranges from $0/1M (Ollama) to $75/1M output (Claude Opus). Any LangGraph retry strategy must be budget-aware at the model tier level.

## Goal

Produce a binding Architecture Decision Record (ADR) committed to `packages/backend/orchestrator/docs/architecture/adr-langgraph-error-handling.md` that unifies the existing error handling foundations and defines the complete LangGraph error strategy as a prerequisite for WINT-9106 (checkpointer), WINT-9107 (middleware implementation), and WINT-9110 (full workflow graphs).

## Acceptance Criteria

- [ ] **AC-001** — ADR defines node-level retry policy with configurable exponential backoff (base delay, max delay, max attempts, jitter) per operation category: LLM call, file I/O, DB write, git operation. Values must align with or supersede existing `RETRY_PRESETS` in `runner/types.ts`.

- [ ] **AC-002** — ADR defines graph-level error propagation rules — which node failures are retryable vs terminal, and how errors bubble to the parent graph. Must distinguish between saga-style (compensate and continue) vs fail-fast semantics per graph type.

- [ ] **AC-003** — ADR defines circuit breaker pattern for LLM provider calls — open/half-open/closed states, failure thresholds, cooldown periods. Must explicitly decide: node-local state (current `NodeCircuitBreaker`) vs DB-shared state (`workflow.circuitBreakerState` table). If DB-shared, this constraint propagates to WINT-9107 AC-004.

- [ ] **AC-004** — ADR defines dead-letter queue (DLQ) strategy for batch workflows — failed stories written to `graph.deadLetterQueue` table with error context, retry eligibility, and manual review path.

- [ ] **AC-005** — ADR defines idempotency contracts for side-effecting nodes (KB writes, git commits, file writes, index updates) — each category must specify whether it is naturally idempotent, requires an idempotency key, or requires compensating transactions.

- [ ] **AC-006** — ADR defines timeout strategy mapping from current `max_turns` limits in agent `.md` files to LangGraph node/graph-level `timeoutMs` values. Must cover: LLM node timeout, tool node timeout, graph-level deadline.

- [ ] **AC-007** — ADR defines partial-success semantics for batch graphs — how N-of-M failures are reported, whether successful items commit independently, and what the minimum success threshold is for a batch to be considered passing.

- [ ] **AC-008** — ADR maps each existing Claude Code retry policy to its LangGraph equivalent: `dev-implement` (1 retry for type errors → node retry with error context forwarding), `elab-story` (1 retry for parse/write errors → node retry with backoff), `elab-epic` (1 retry per worker timeout, save & resume via CHECKPOINT.yaml → checkpointer + node retry), `phase-contracts` (on_failure: block transition → graph-level conditional edges).

- [ ] **AC-009** — ADR includes a budget-aware retry cap section: maximum retry spend per node invocation for each model tier (Tier 0 Claude Opus $15/$75/1M, Tier 1 Claude Sonnet $3/$15/1M, Tier 2/3 Ollama/OpenRouter $0–$0.08/1M). References WINT-0220 model-tier cost table.

- [ ] **AC-010** — ADR consolidates or explicitly partitions the two existing error-handling layers (`runner/` vs `errors/workflow-errors.ts`) — declares which layer owns which concern going forward. Recommended partition: `runner/` = node-execution primitive layer (per-invocation); `errors/workflow-errors.ts` = workflow-lifecycle layer (per-story/phase).

- [ ] **AC-011** — ADR reviewed and approved — committed to `packages/backend/orchestrator/docs/architecture/adr-langgraph-error-handling.md` with a signed-off status header (status, decision-date, deciders).

## Architecture Notes

### Layer Partition Decision (AC-010)

The recommended ADR decision partitions the two existing error systems by layer:

- **`runner/` = node-execution layer**: Handles per-invocation retry, circuit breaker, timeout, and error classification. Owns `NodeRetryConfigSchema`, `NodeCircuitBreaker`, `classifyError`, `RETRY_PRESETS`. `failureThreshold=5` applies at this layer.
- **`errors/workflow-errors.ts` = workflow-lifecycle layer**: Handles per-story/phase error logging, recovery actions, and phase-level circuit breaking. Owns `WorkflowErrorTypeSchema`, `ERROR_RETRY_DEFAULTS`, `ErrorLogSchema`. `failureThreshold=3` at this layer governs when a *phase* enters circuit-open, a different concept from when a single *node invocation* circuit-opens.

LangGraph middleware (WINT-9107) builds on the `runner/` layer for per-node execution and uses `workflow-errors.ts` types for workflow-state reporting.

### Circuit Breaker State Ownership (AC-003)

The ADR must make an explicit choice between:

1. **Node-local state** (current `NodeCircuitBreaker`): Simpler, no DB dependency. Risk: concurrent graph executions share an LLM provider but each has an independent circuit breaker — one graph's failures do not protect other graphs from the same failing provider.

2. **DB-shared state** (`workflow.circuitBreakerState` table): Protects all concurrent graphs from a shared failing LLM provider. Required if WINT-9107 AC-004 specifies cross-graph protection. Adds a DB migration to WINT-9107 scope.

The seed's recommendation from blockers: DB-shared state is needed for LLM provider calls because concurrent graph executions share provider quota. WINT-9107 must implement the migration.

### Budget-Aware Retry Caps (AC-009)

From WINT-0220-STRATEGY.md:

| Tier | Model | Input $/1M | Output $/1M | Max LLM Retries |
|------|-------|------------|-------------|-----------------|
| 0 | Claude Opus 4.6 | $15.00 | $75.00 | 2 (cost cap) |
| 1 | Claude Sonnet 4.5 | $3.00 | $15.00 | 3 |
| 2 | Ollama (local) | $0.00 | $0.00 | 5 (latency-bound) |
| 3 | OpenRouter Haiku | $0.08 | $0.08 | 5 |

Unlimited retries on Tier 0/1 are prohibited. ADR must encode these caps in the `RETRY_PRESETS.llm` extension.

### Retry Category Extensions (AC-001)

Existing `RETRY_PRESETS` covers llm, tool, validation. ADR adds:

| Category | maxAttempts | backoffMs | maxBackoffMs | timeoutMs | Idempotent? |
|----------|-------------|-----------|--------------|-----------|-------------|
| `git` | 3 | 1000 | 15000 | 30000 | Yes (git operations are idempotent by content hash) |
| `db_write` | 3 | 500 | 10000 | 15000 | Requires idempotency key |
| `file_io` | 3 | 500 | 10000 | 10000 | Requires atomic write pattern |

### Known Conflict to Resolve (from Seed)

**Pattern Consolidation Gap** (warning severity): Two overlapping error handling systems exist with different defaults:
- `runner/circuit-breaker.ts`: `failureThreshold=5`, `recoveryTimeoutMs=60000`
- `errors/workflow-errors.ts`: `failureThreshold=3`, `recoveryTimeMs=60000`, `failureWindowMs=300000`

Resolution: Layer partition (see above). Both values are correct for their respective layers. ADR must document this explicitly so WINT-9107 implementers do not unify them incorrectly.
