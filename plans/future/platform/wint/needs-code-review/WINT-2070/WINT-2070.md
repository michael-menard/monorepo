# WINT-2070: Implement Cache Warming Strategy — Create `cache-warm` Skill that Invokes All Populate Scripts

## Context

The WINT platform Phase 2 goal is 80% token reduction. Four populate scripts now exist (or are in-flight) to warm `wint.context_packs` with project conventions, domain knowledge, library patterns, and agent missions:

| Script | Location | Pack Types | Status |
|--------|----------|------------|--------|
| `populate-project-context.ts` | `packages/backend/mcp-tools/src/scripts/` | architecture, project-conventions, tech-stack-* | ready-to-work (WINT-2030) |
| `populate-domain-kb.ts` | `packages/backend/mcp-tools/src/scripts/` | lessons_learned, blockers_known | needs-code-review (WINT-2050) |
| `populate-library-cache.ts` | `packages/backend/mcp-tools/src/scripts/` | codebase/lib-* | ready-to-work (WINT-2060) |
| `agent-mission-cache-populator.ts` | `packages/backend/database-schema/src/seed/` | agent_missions | failed-code-review (WINT-2040) |

These scripts are currently invoked manually and independently. Before a high-volume agent workflow session starts, all four caches must be fresh, but there is no single entry point to trigger all warm-ups. The upcoming `context-warmer` agent (WINT-2080) needs a skill to invoke.

**Baseline gap**: Baseline predates WINT-2030/2040/2050/2060 implementation; all four populate scripts exist in the codebase now or are in-flight.

## Goal

Create `.claude/skills/cache-warm/SKILL.md` — a single entry point that:
1. Checks preconditions (env vars, DB availability)
2. Invokes each of the four populate scripts sequentially via `pnpm tsx`
3. Treats individual script failures as non-fatal (partial warming is better than no warming)
4. Reports a final summary (warm result: N/4 scripts succeeded)
5. Is executable by both human operators (`/cache-warm`) and LLM agents (context-warmer, WINT-2080)

## Non-Goals

- Do not implement a TypeScript orchestrator script — the skill invokes existing scripts via `pnpm tsx`
- Do not implement automatic cache invalidation or doc-change triggers (deferred, was WINT-2050/WINT-2070 scope per lesson)
- Do not extract `populate-utils.ts` shared module (follow-on story — flag as opportunity after delivery)
- Do not modify any populate scripts — read-only from this story's perspective
- Do not modify `wint.context_packs` table schema (protected)
- Do not add new `contextPackTypeEnum` values (protected)
- Do not implement caching scheduling or cron jobs (WINT Phase 3 scope)
- Do not implement cache retrieval for agents (WINT-2110 scope)
- Do not build a UI or HTTP endpoint
- Do not require WINT-2040 to pass code review before this story can be completed

## Architecture Notes

### Skill Structure

The `cache-warm` skill follows the established 6-phase pattern:

```
Phase 1: Precondition check
  - Verify DATABASE_URL is set and contains port 5432
  - Warn (not abort) if KB_DATABASE_URL is absent
  - Emit environment summary

Phase 2: Invoke populate-project-context.ts
  - pnpm tsx packages/backend/mcp-tools/src/scripts/populate-project-context.ts
  - Capture exit code; record pass/fail

Phase 3: Invoke populate-domain-kb.ts
  - Skip if KB_DATABASE_URL not set (warn and continue)
  - pnpm tsx packages/backend/mcp-tools/src/scripts/populate-domain-kb.ts
  - Capture exit code; record pass/fail

Phase 4: Invoke populate-library-cache.ts
  - pnpm tsx packages/backend/mcp-tools/src/scripts/populate-library-cache.ts
  - Capture exit code; record pass/fail

Phase 5: Invoke agent-mission-cache-populator.ts
  - NOTE: non-standard path at packages/backend/database-schema/src/seed/
  - Skip if --skip=agent-missions flag set
  - pnpm tsx packages/backend/database-schema/src/seed/agent-mission-cache-populator.ts
  - Capture exit code; record pass/fail

Phase 6: Summary
  - Emit structured summary block: N/4 scripts succeeded
  - List each script with pass/fail/skip status
  - Report estimated packs written
```

### Sequential vs Parallel

Per WINT-2030 and WINT-2050 lessons: sequential `pnpm tsx` invocations are the MVP approach. Parallel execution is a future optimization (out of scope for WINT-2070).

### WINT-2040 Path Discrepancy

`agent-mission-cache-populator.ts` lives at `packages/backend/database-schema/src/seed/` instead of `packages/backend/mcp-tools/src/scripts/` — a placement inconsistency from WINT-2040's failed code review. The skill must use the correct path verbatim and document this discrepancy.

### Summary Block Format

The summary block should be structured for both human readability and agent parseability:

```
Cache warm complete: 3/4 scripts succeeded.

Results:
  populate-project-context:  PASS  (5 packs written)
  populate-domain-kb:        PASS  (6 packs written)
  populate-library-cache:    PASS  (4 packs written)
  agent-mission-cache:       SKIP  (--skip=agent-missions flag set)

Total packs written: ~15
```
