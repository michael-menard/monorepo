# WINT-1160: Add Parallel Work Conflict Prevention

## Context

WINT-1130 and WINT-1140 are both in UAT with QA PASS verdicts. WINT-1130 delivered the database infrastructure (`wint.worktrees` table + 4 MCP tools: `worktree_register`, `worktree_get_by_story`, `worktree_list_active`, `worktree_mark_complete`). WINT-1140 inserted a worktree pre-flight step (Step 1.3) into `dev-implement-story` that calls `worktree_get_by_story` and presents a 3-option warning when a different-session active worktree is found.

Two gaps remain unaddressed by those stories:

1. **Status visibility gap**: The `/wt:status` skill shows only git-level worktree information — it has no visibility into which stories have active database-tracked worktrees, or whether any are in conflict states. An agent running `/wt:status` cannot determine if a story is being worked on in another session.

2. **Take-over safety gap**: The "take over" path (option b in WINT-1140 AC-4) — which marks the old worktree as abandoned and creates a new one — requires an explicit, hardened confirmation mechanism to prevent accidental work loss. The `--autonomous` flag behavior for this destructive action was left underspecified in WINT-1140.

Without this story, two sessions can still collide on the same story if agents using `/wt:status` cannot see DB-tracked worktrees, or if the take-over path's safety constraint is not formally encoded in the command.

**Note on WINT-1140 AC-4 boundary**: WINT-1140 owns the 3-option conflict prompt UX specification. WINT-1160 does NOT re-specify the prompt — it adds the take-over hardening (AC-5, AC-6) and the status enhancement (AC-1 through AC-4, AC-8, AC-9).

---

## Goal

Close the two remaining gaps in the parallel work conflict prevention system:
1. Enhance `/wt:status` to show a DB-backed worktree view (which stories have active worktrees, orphaned records, untracked git worktrees)
2. Harden the "take over" path in `dev-implement-story` Step 1.3 with an explicit confirmation requirement that overrides all autonomy levels

- [ ] **AC-4**: `/wt:status` identifies git worktrees that exist on disk but have no corresponding active DB record, and flags them with `[UNTRACKED]` in the output. Detection requires cross-referencing the `worktree_list_active` results against the list of git worktrees.

## Architecture Notes

### DB-Aware Section in wt-status

The `/wt:status` skill enhancement follows the null-check resilience pattern established in WINT-1130:

```
Call worktree_list_active →
  if null/error: render warning, show git-only view, return
  if []: render "No active database-tracked worktrees" message
  if [records]:
    for each record:
      check if worktreePath exists on disk (Bash: ls {path})
      if not on disk: flag [ORPHANED]
    compare active DB records against git worktrees
    for git worktrees with no DB record: flag [UNTRACKED]
    render tabular output
```

### Take-Over Hardening in dev-implement-story Step 1.3

The hardened take-over sequence respects the AC-6 ordering constraint:

```
User selects option (b) take-over →
  ALWAYS show confirmation prompt (regardless of autonomy level):
    "WARNING: This will mark the following worktree as abandoned:
      Story: {storyId}
      Branch: {branchName}
      Path: {worktreePath}
      Registered: {createdAt}
    This action cannot be undone. Type 'yes' to confirm take-over:"

  if user confirms:
    1. Call worktree_mark_complete({ worktreeId, status: 'abandoned' })
    2. If null/error → "Take-over aborted: failed to mark old worktree as abandoned" → STOP
    3. If success → /wt:new story/{STORY_ID} main
    4. Register new worktree → continue implementation

  if user cancels:
    Re-present options (a), (b), (c) OR show abort message
```

### Interaction with WINT-1140 AC-9 (Autonomy Auto-Selection)

WINT-1140 AC-9 specified that `moderate`/`aggressive` autonomy levels auto-select option (a) switch. WINT-1160 AC-5 adds a carve-out: option (b) is NEVER auto-selected at any autonomy level. These two rules coexist without conflict:

| Autonomy | Option (a) switch | Option (b) take-over | Option (c) abort |
|----------|-------------------|---------------------|-----------------|
| conservative | Prompt required | Prompt required | Prompt required |
| moderate | Auto-select | **ALWAYS prompt** | Prompt required |
| aggressive | Auto-select | **ALWAYS prompt** | Prompt required |
