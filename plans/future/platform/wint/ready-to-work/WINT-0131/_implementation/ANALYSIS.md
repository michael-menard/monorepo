## Issues Found

| # | Issue | Severity | Required Fix |
|---|-------|----------|--------------|
| 1 | capabilityType enum mismatch: AC-5 describes CRUD detection using `capabilityType` values (`create`, `read`, `update`, `delete`), but the live `capabilities` table schema uses `capabilityType` for domain-level types (`'business'`, `'technical'`, `'infrastructure'`). The CRUD-style values appear to be stored in `lifecycleStage` (which has same values: `'experimental'`, `'beta'`, `'stable'`, `'deprecated'`) — also not CRUD. This is the same risk explicitly flagged in the story's Risk Factors. Implementer MUST verify the actual column that stores CRUD type values before coding the Franken-feature detection query. | High | Before writing `graph-get-franken-features.ts` DB logic, query the live `wint.capabilities` table to confirm which column stores `create`/`read`/`update`/`delete` values. AC-5 and the Reuse Plan's query pattern reference `capabilities.capabilityType` — this may need to be `lifecycleStage` or a different column entirely. Update the Drizzle query accordingly. |
| 2 | Migration file naming discrepancy: Story specifies target file `0027_wint_0131_capability_feature_linkage.sql` but the actual file on disk is `0027_wint_0131_capabilities_feature_fk.sql`. The file exists and contains correct SQL. | Low | No fix required for implementation. The actual file name takes precedence. Update SCOPE.md or AC-1 target filename to match actual file if documentation accuracy is required. |
| 3 | Drizzle `_journal.json` not updated: Migration 0027 SQL file exists on disk but the Drizzle journal (`src/migrations/app/meta/_journal.json`) has not been updated to register it. | Medium | During implementation Step 3 (generate migration), the implementer must ensure the journal is updated. If the file was manually created rather than generated by `db:generate`, the journal will be out of sync and `db:migrate` will fail or skip it. Use `pnpm --filter @repo/database-schema db:generate` to regenerate, then compare with the existing file and rename to match the expected naming convention. |
