# WINT-4060: Create graph-checker Agent

## Context

The WINT Phase 4 feature cohesion subsystem has delivered its foundational infrastructure: the graph schema and query MCP tools (WINT-0130), the feature-capability linkage (WINT-0131), the feature registry (WINT-4010), the capability taxonomy (WINT-4020), and the cohesion rules registry with seeded rules (WINT-4050). The graph is being populated with existing features and epics (WINT-4030) and inferred capabilities (WINT-4040).

No agent currently queries the graph views (`franken_features`, `capability_coverage`) and applies the active cohesion rules to detect incomplete features. The pipeline has data and rules but no automated detector. This story creates the detection layer.

## Goal

Create a haiku-powered `graph-checker` worker agent that queries the feature cohesion graph and active rules registry, detects franken-features and rule violations, and emits a machine-readable report with actionable violation hints for downstream agents to consume.

## Examples

**Happy path — violations detected:**
- Given `graph.features` is populated (WINT-4030 complete) with features including `user-authentication` missing a `delete` capability, and `wint.rules` contains an active rule requiring all 4 CRUD stages, the agent invokes `graph_get_franken_features({})` which returns `user-authentication`, evaluates the active rule, and writes `graph-check-results.json` containing one violation: `{ rule_id: "RULE-001", feature_id: "feat-001", feature_name: "user-authentication", description: "Missing delete stage", severity: "warning", actionable_hint: "Add a delete capability to feature user-authentication" }`. Output: `GRAPH-CHECKER COMPLETE`.

**Happy path — no violations:**
- Given all features in `graph.features` have 4+ distinct CRUD lifecycle stages and all active rules pass, `graph_get_franken_features({})` returns an empty list, and the agent writes `graph-check-results.json` with `franken_features_found: 0`, `violations: []`, `warnings: []`, `warning_count: 0`. Output: `GRAPH-CHECKER COMPLETE`.

## Edge Cases

**Empty graph (WINT-4030 not yet stable):**
- Given `graph.features` and `graph.epics` tables are empty (WINT-4030 dependency not satisfied at runtime), `graph_get_franken_features({})` returns an empty list. The agent increments `warning_count` by 1, adds a warning entry `"graph.features empty — WINT-4030 may not have run"`, writes `graph-check-results.json` with `franken_features_found: 0`, `violations: []`. Output: `GRAPH-CHECKER COMPLETE WITH WARNINGS: 1 warnings`.

**Empty rules registry (WINT-4050 rules not yet seeded):**
- Given `rulesRegistryGet({ status: 'active' })` returns `[]` (no active rules), the agent increments `warning_count`, adds a warning entry `"No active cohesion rules found — WINT-4050 may not have run"`, and returns an empty violations list rather than erroring. Output: `GRAPH-CHECKER COMPLETE WITH WARNINGS: 1 warnings`.

- Do NOT modify `graph.features`, `graph.epics`, `capabilities`, or `wint.rules` tables
