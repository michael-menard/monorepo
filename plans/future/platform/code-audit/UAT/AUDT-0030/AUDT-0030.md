# AUDT-0030: End-to-End Integration Tests for 6 Audit Orchestration Nodes

## Context

AUDT-0020 (UAT) delivered all 9 audit lens nodes with verified unit test coverage. The audit pipeline graph (`code-audit.ts`) is fully implemented with two execution modes:

- **Pipeline**: `scan_scope → lens_parallel → synthesize → deduplicate → persist_findings → persist_trends → END`
- **Roundtable**: `scan_scope → lens_parallel → devils_advocate → roundtable → synthesize → deduplicate → persist_findings → persist_trends → END`

Existing graph-level tests in `src/graphs/__tests__/code-audit.test.ts` verify that the graph compiles and can be invoked with an empty lens list, but do not test the behavioral correctness of the 6 orchestration nodes:

| Node | File | What is Untested |
|------|------|-----------------|
| `synthesize` | `nodes/audit/synthesize.ts` | Merge dedup, severity sort, ID assignment, 100-cap |
| `deduplicate` | `nodes/audit/deduplicate.ts` | Jaccard similarity against real stories.index.md |
| `persist-findings` | `nodes/audit/persist-findings.ts` | FINDINGS YAML write + schema compliance |
| `persist-trends` | `nodes/audit/persist-trends.ts` | TRENDS YAML aggregation + empty-summaries early return |
| `devils-advocate` | `nodes/audit/devils-advocate.ts` | Low-confidence downgrade, test-file downgrade |
| `roundtable` | `nodes/audit/roundtable.ts` | False-positive removal, cross-reference detection |

**Dependency constraint**: AUDT-0020 must complete UAT before AUDT-0030 implementation begins. Elaboration (feasibility, test plan) may proceed in parallel.

## Architecture Notes

### Small Refactors Required (ST-1 — prerequisite for all filesystem tests)

Three production nodes need optional path parameters to enable hermetic tests:

```typescript

export async function deduplicate(
  state: CodeAuditState,
  plansDir = 'plans'   // <-- add this
): Promise<Partial<CodeAuditState>>

// persist-findings.ts
export async function persistFindings(
  state: CodeAuditState,
  auditDir = 'plans/audit'  // <-- add this
): Promise<Partial<CodeAuditState>>

// persist-trends.ts
export async function persistTrends(
  state: CodeAuditState,
  auditDir = 'plans/audit'  // <-- add this
): Promise<Partial<CodeAuditState>>
```

The graph node wrappers in `code-audit.ts` pass no argument (use default), preserving production behavior unchanged.

### Finding ID State Coupling

`runDevilsAdvocate` receives findings from `state.lensResults` which have `id: ''` at the time the node runs in the real pipeline — IDs are assigned by `synthesize` which runs later (after roundtable). The `challengeMap` in `runRoundtable` builds a lookup by `finding_id`. For unit tests, set explicit non-empty `id` fields on fixture findings to make the lookup work. Document this known ordering discrepancy in test file comments.
