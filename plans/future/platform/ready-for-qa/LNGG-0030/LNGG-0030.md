# Decision Callback System - Interactive User Prompts for LangGraph Workflows

## Context

LangGraph workflows currently have no mechanism for requesting user decisions during execution. Claude Code workflows use `AskUserQuestion` to present choices (e.g., "Add this gap as AC or create follow-up story?"), but LangGraph operates entirely programmatically. This blocks migration of interactive workflows like `/elab-story` and `/dev-implement-story` to LangGraph.

**Current State:**
- LangGraph workflows exist at `packages/backend/orchestrator/src/graphs/` (elaboration, story-creation)
- Decision handling protocol documented at `.claude/agents/_shared/decision-handling.md` (5-tier classification)
- Orchestrator package provides infrastructure at `packages/backend/orchestrator/src/`
- No callback system exists - workflows cannot pause for user input

**Problem:**
Without a decision callback system, LangGraph workflows cannot:
- Present choices to users for ambiguous requirements
- Collect user input for gap handling during elaboration
- Support both autonomous and interactive execution modes
- Block workflow execution while waiting for decisions
- Handle timeouts when users don't respond
- Support multiple UI modes (CLI, web, auto-decision)

**Why This Matters:**
- Blocks migration of interactive workflows from Claude Code to LangGraph
- Limits workflow flexibility (all-or-nothing autonomous execution)
- Prevents human-in-the-loop patterns for complex decisions
- No path to gradual autonomy increase with safety gates

## Goal

Create a flexible decision callback system that enables LangGraph workflows to request user decisions during execution, supporting multiple interaction modes (CLI, auto-decision, future web UI) with proper timeout and cancellation handling.

**Success Criteria:**
1. DecisionCallback interface defined with Zod schemas (request/response types)
2. CLI implementation working with inquirer prompts
3. Auto-decision implementation with configurable rule engine
4. Registry managing callback implementations
5. LangGraph nodes can invoke callbacks and wait for responses
6. >80% test coverage with unit and integration tests
7. Elaboration graph successfully uses callbacks for gap decisions

## Architecture Notes

### Design Pattern: Strategy Pattern with Registry

Use Strategy pattern for callback implementations, managed by a Registry:

```typescript
// types.ts
import { z } from 'zod'

export const QuestionTypeSchema = z.enum(['single-choice', 'multi-select', 'text-input'])
export type QuestionType = z.infer<typeof QuestionTypeSchema>

export const DecisionOptionSchema = z.object({
  value: z.string(),
  label: z.string(),
  description: z.string().optional(),
  recommended: z.boolean().optional()
})

export const DecisionRequestSchema = z.object({
  id: z.string().uuid(),
  type: QuestionTypeSchema,
  question: z.string().min(1),
  options: z.array(DecisionOptionSchema).min(1),
  context: z.record(z.unknown()).optional(),
  timeout_ms: z.number().int().positive().default(30000)
})

export const DecisionResponseSchema = z.object({
  id: z.string().uuid(),
  answer: z.union([z.string(), z.array(z.string())]),
  cancelled: z.boolean().default(false),
  timedOut: z.boolean().default(false),
  timestamp: z.string().datetime()
})

export type DecisionRequest = z.infer<typeof DecisionRequestSchema>
export type DecisionResponse = z.infer<typeof DecisionResponseSchema>

// Base interface
export interface DecisionCallback {
  ask(request: DecisionRequest): Promise<DecisionResponse>
  cleanup?(): void
}
```

### CLIDecisionCallback Implementation

```typescript
// cli-callback.ts
import inquirer from 'inquirer'
import { logger } from '@repo/logger'
import { DecisionCallback, DecisionRequest, DecisionResponse, DecisionResponseSchema } from './types'

export class CLIDecisionCallback implements DecisionCallback {
  async ask(request: DecisionRequest): Promise<DecisionResponse> {
    const { id, question, options, timeout_ms, type } = request

    logger.info('Presenting decision to user', { id, question })

    // Setup timeout
    const timeoutPromise = new Promise<DecisionResponse>((resolve) => {
      setTimeout(() => {
        logger.warn('Decision timeout', { id })
        resolve({
          id,
          answer: options[0]?.value || '',
          cancelled: false,
          timedOut: true,
          timestamp: new Date().toISOString()
        })
      }, timeout_ms)
    })

    // Setup inquirer prompt
    const promptPromise = (async () => {
      try {
        const inquirerType = type === 'multi-select' ? 'checkbox' : type === 'text-input' ? 'input' : 'list'
        const result = await inquirer.prompt([{
          type: inquirerType,
          name: 'answer',
          message: question,
          choices: type !== 'text-input' ? options.map(o => ({
            name: `${o.label}${o.description ? ` - ${o.description}` : ''}${o.recommended ? ' (recommended)' : ''}`,
            value: o.value
          })) : undefined
        }])

        return DecisionResponseSchema.parse({
          id,
          answer: result.answer,
          cancelled: false,
          timedOut: false,
          timestamp: new Date().toISOString()
        })
      } catch (error) {
        // Ctrl+C or other interruption
        logger.info('Decision cancelled by user', { id })
        return {
          id,
          answer: '',
          cancelled: true,
          timedOut: false,
          timestamp: new Date().toISOString()
        }
      }
    })()

    // Race timeout vs user input
    return Promise.race([promptPromise, timeoutPromise])
  }

  cleanup(): void {
    // Cleanup any open prompts (inquirer handles this internally)
  }
}
```

### AutoDecisionCallback Implementation

```typescript
// auto-callback.ts
import { logger } from '@repo/logger'
import { DecisionCallback, DecisionRequest, DecisionResponse } from './types'

export interface DecisionRule {
  name: string
  condition: (context: Record<string, unknown>) => boolean
  answer: string | string[]
  rationale: string
}

export class AutoDecisionCallback implements DecisionCallback {
  constructor(private rules: DecisionRule[], private defaultAnswer?: string) {}

  async ask(request: DecisionRequest): Promise<DecisionResponse> {
    const { id, context = {}, options } = request

    // Find matching rule
    const matchedRule = this.rules.find(rule => rule.condition(context))

    const answer = matchedRule?.answer
      ?? this.defaultAnswer
      ?? options[0]?.value
      ?? ''

    const rationale = matchedRule?.rationale ?? 'Default fallback'

    logger.info('Auto-decision made', {
      id,
      answer,
      rationale,
      rule: matchedRule?.name ?? 'default'
    })

    return {
      id,
      answer,
      cancelled: false,
      timedOut: false,
      timestamp: new Date().toISOString()
    }
  }
}
```

### DecisionCallbackRegistry Implementation

```typescript
// registry.ts
import { DecisionCallback } from './types'
import { CLIDecisionCallback } from './cli-callback'
import { AutoDecisionCallback } from './auto-callback'

export class DecisionCallbackRegistry {
  private static instance: DecisionCallbackRegistry
  private callbacks = new Map<string, DecisionCallback>()

  private constructor() {
    // Register built-in callbacks
    this.register('cli', new CLIDecisionCallback())
    this.register('auto', new AutoDecisionCallback([], undefined))
    this.register('noop', new NoopDecisionCallback())
  }

  static getInstance(): DecisionCallbackRegistry {
    if (!DecisionCallbackRegistry.instance) {
      DecisionCallbackRegistry.instance = new DecisionCallbackRegistry()
    }
    return DecisionCallbackRegistry.instance
  }

  register(name: string, callback: DecisionCallback): void {
    this.callbacks.set(name, callback)
  }

  get(name: string): DecisionCallback {
    const callback = this.callbacks.get(name)
    if (!callback) {
      throw new Error(`Decision callback '${name}' not found`)
    }
    return callback
  }

  getDefault(): DecisionCallback {
    return this.get('cli')
  }
}

class NoopDecisionCallback implements DecisionCallback {
  async ask(request: DecisionRequest): Promise<DecisionResponse> {
    return {
      id: request.id,
      answer: request.options[0]?.value || '',
      cancelled: false,
      timedOut: false,
      timestamp: new Date().toISOString()
    }
  }
}
```

### LangGraph Integration Pattern

```typescript
// In graph configuration
import { DecisionCallbackRegistry } from './adapters/decision-callbacks'

const graphConfig = {
  decisionCallback: DecisionCallbackRegistry.getInstance().get('cli'),
  // ... other config

// In node implementation
async function gapHandlerNode(state: State, config: GraphConfig): Promise<State> {
  const { decisionCallback } = config

  const response = await decisionCallback.ask({
    id: crypto.randomUUID(),
    type: 'single-choice',
    question: 'How should this gap be handled?',
    options: [
      { value: 'add_ac', label: 'Add as Acceptance Criterion' },
      { value: 'follow_up', label: 'Create Follow-up Story' },
      { value: 'skip', label: 'Skip This Gap' }
    ],
    context: { gap: state.currentGap },
    timeout_ms: 30000
  })

  if (response.cancelled || response.timedOut) {
    // Handle gracefully
    return { ...state, gapDecision: 'skip' }
  }

  return { ...state, gapDecision: response.answer }
}
```

### Timeout & Cancellation Handling

**Timeout Pattern:**
```typescript
Promise.race([
  userPromise,
  new Promise((resolve) => setTimeout(() => resolve(timeoutResponse), timeout_ms))
])
```

**Cancellation Cleanup:**
- Inquirer handles Ctrl+C internally via event listeners
- Return `{ cancelled: true }` on interrupt
- No need for manual signal handler registration
- Callbacks must be stateless to avoid cleanup issues

### Error Handling Strategy

1. **Schema Validation Errors**: Throw immediately (developer error)
2. **Timeout**: Return `{ timedOut: true }` with default answer
3. **Cancellation**: Return `{ cancelled: true }` with empty answer
4. **Unknown Errors**: Log and return cancelled response
