# KFMB-5020 — Migrate _implementation/ Reader Agents to kb_read_artifact

## Context

The KB-First Migration plan (KFMB plan) is eliminating `_implementation/` filesystem artifact directories in favor of KB-native storage via `kb_write_artifact` / `kb_read_artifact`. KFMB-5010 migrates writer agents so all artifacts are written to the KB. This story (KFMB-5020) is the complementary reader migration: every agent that currently reads from `_implementation/CHECKPOINT.yaml`, `_implementation/EVIDENCE.yaml`, `_implementation/ELAB.yaml`, `_implementation/SCOPE.yaml`, `_implementation/PLAN.yaml`, or `_implementation/REVIEW.yaml` from the filesystem must be updated to read from the KB via `kb_read_artifact`.

Approximately 19 reader agents reference `_implementation/` filesystem paths. Once KFMB-5010 lands, these filesystem files will no longer be reliably present. Any agent that still reads from disk will receive missing-file errors or stale data.

**Baseline date**: 2026-02-13. This story post-dates the baseline (generated 2026-02-26).

**Key precedents**: `dev-execute-leader` and `qa-verify-verification-leader` are already fully KB-native readers and serve as gold standards.

## Goal

Update every reader agent that references `_implementation/` artifact paths to use `kb_read_artifact` exclusively, following the patterns established by `dev-execute-leader` and `qa-verify-verification-leader`. After this story, no reader agent should read artifact files from the filesystem.

## Non-Goals

- Do NOT migrate writer agents — that is KFMB-5010.
- Do NOT modify any source code in `packages/` or `apps/` — only `.claude/agents/` markdown files.
- Do NOT migrate `_pm/` artifacts — that is KFMB-5040 and KFMB-5050.
- Do NOT remove `_implementation/` directories from in-progress stories — that is KFMB-6020.
- Do NOT implement KFMB-5030 (command orchestrator migration).
- Do NOT change behavior of already-migrated agents (dev-execute-leader, dev-plan-leader, dev-setup-leader, qa-verify-verification-leader, knowledge-context-loader).
- Do NOT update `_shared/kb-integration.md` — it already documents correct patterns.

**Protected features** (do not modify):
- Orchestrator artifact schemas in `packages/backend/orchestrator/src/artifacts/`
- `@repo/db` client API surface

## Architecture Notes

**Migration pattern** (apply to every agent):

1. Add `kb_read_artifact` to frontmatter `kb_tools` array.
2. Replace `Read(path/_implementation/FOO.yaml)` with `kb_read_artifact({ story_id: "{STORY_ID}", artifact_type: "foo" })` and access `.content`.
3. Use `Promise.all([...])` for parallel reads when multiple artifact types are needed.
4. For precondition existence checks: `const artifact = await kb_read_artifact(...); if (!artifact || !artifact.content) → BLOCKED`.
5. Remove deprecated file path references from Inputs sections and precondition blocks.
6. If an agent previously only needed `Read` for artifact loading, assess whether `Read` can be dropped from the tools list.

**Artifact type mapping (authoritative — from `_shared/kb-integration.md`)**:

| File (deprecated) | artifact_type |
|---|---|
| CHECKPOINT.yaml | `checkpoint` |
| SCOPE.yaml | `scope` |
| PLAN.yaml | `plan` |
| KNOWLEDGE-CONTEXT.yaml | `context` |
| EVIDENCE.yaml | `evidence` |
| REVIEW.yaml | `review` |
| ELAB.yaml | `analysis` |
| QA-VERIFY.yaml / VERIFICATION.yaml | `verification` |

**Read/Write boundary**: This story covers only READ calls. Agents that also WRITE to `_implementation/` (e.g., `dev-verification-leader` in fix mode) have their write-side migration covered by KFMB-5010. Coordinate to avoid double-counting.
