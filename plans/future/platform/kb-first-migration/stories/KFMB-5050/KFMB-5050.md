# KFMB-5050 — Migrate _pm/ Reader Agents and Remove pm_artifacts Embedding

## Context

The KB-First Migration plan is eliminating filesystem-based artifact directories in favor of KB-native storage via `kb_write_artifact` / `kb_read_artifact`. KFMB-5040 migrates PM pipeline writer agents (`pm-story-seed-agent`, `pm-draft-test-plan`, `pm-dev-feasibility-review`, `pm-uiux-recommendations`) so PM artifacts are written to the KB instead of `_pm/` files. This story (KFMB-5050) is the complementary reader migration: every agent that currently reads from `_pm/test-plan.yaml`, `_pm/dev-feasibility.yaml`, `_pm/uiux-notes.yaml`, or `_pm/STORY-SEED.md` from disk — or from the `pm_artifacts` embedding in story.yaml — must be updated to read from the KB via `kb_read_artifact`.

Currently, PM artifact consumers use two distinct read pathways that will both break once KFMB-5040 lands:

1. **Direct `_pm/` filesystem reads**: `pm-story-generation-leader` reads `_pm/test-plan.yaml`, `_pm/dev-feasibility.yaml`, `_pm/uiux-notes.yaml`, and `_pm/STORY-SEED.md` immediately after worker agents complete.
2. **`pm_artifacts` embedding in story.yaml**: `pm-story-fix-leader` and `architect-story-review` read `pm_artifacts.test_plan`, `pm_artifacts.uiux_notes`, `pm_artifacts.dev_feasibility` from story.yaml frontmatter — where they were embedded by `pm-story-generation-leader`.

Additionally, `pm-story-risk-predictor` and `story-synthesize-agent` currently receive `story_seed_path` as a filesystem path parameter. This interface must transition to a `story_id`-based KB lookup.

**Baseline**: Loaded (2026-02-13). Orchestrator artifact schemas in `packages/backend/orchestrator/src/artifacts/` and `_shared/kb-integration.md` are confirmed existing and protected. The dev-execute-leader parallel `kb_read_artifact` pattern is the established gold standard.

## Goal

All PM artifact reads go through `kb_read_artifact` and the `pm_artifacts` embedding is retired from story.yaml frontmatter. No agent in the PM lifecycle reads `_pm/*.yaml` files directly or queries `pm_artifacts.*` from story.yaml after this story ships.

## Non-Goals

- Do NOT migrate PM writer agents — that is KFMB-5040.
- Do NOT modify any source code in `packages/` or `apps/` — only `.claude/agents/` markdown files and `_shared/kb-integration.md`.
- Do NOT migrate `_implementation/` artifacts (CHECKPOINT, SCOPE, PLAN, EVIDENCE, REVIEW, ANALYSIS) — that is KFMB-5020.
- Do NOT remove `_pm/` directories from existing stories already in-progress — that is KFMB-6020 (dead code removal).
- Do NOT implement KFMB-5030 (command orchestrator migration) as part of this story.
- Do NOT migrate gap/fanout/readiness artifacts (`_pm/ATTACK.yaml`, `_pm/FANOUT-*.yaml`, `_pm/READINESS.yaml`) unless KFMB-5040 explicitly covers their writer side. Defer to a follow-on story if out of KFMB-5040 scope.
- Do NOT change the content or schema of PM artifact types — only the transport mechanism changes (filesystem → KB).
- Do NOT change behavior of PM writer agents already using `kb_write_artifact` (handled by KFMB-5040).

## Reuse Plan

- **Parallel read pattern**: `dev-execute-leader.agent.md` — uses `Promise.all([kb_read_artifact(...), kb_read_artifact(...), ...])` for simultaneous multi-artifact reads. `pm-story-generation-leader` must adopt this exact pattern for reading `test_plan`, `dev_feasibility`, `uiux_notes`, `story_seed` in parallel.
- **Null-check precondition pattern**: `qa-verify-verification-leader.agent.md` — demonstrates `const artifact = await kb_read_artifact(...)` followed by `if (!artifact || !artifact.content) → STOP` for required artifact precondition enforcement.
- **Artifact type mapping**: `_shared/kb-integration.md` Artifact Type Reference table (lines 330–343) — authoritative mapping extended by this story with PM artifact types.
- **Analogous migration**: `plans/future/platform/kb-first-migration/elaboration/KFMB-5020/_pm/STORY-SEED.md` — identical migration pattern for `_implementation/` reader agents. Apply same approach for `_pm/` reader agents.
- **No new packages**: MCP KB tools (`kb_read_artifact`, `kb_list_artifacts`) are pre-existing. No npm additions required.

## Architecture Notes

**Migration pattern per agent:**

```
BEFORE:
  Read(_pm/test-plan.yaml) → yaml content

AFTER:
  const artifact = await kb_read_artifact({ story_id, artifact_type: "test_plan" })
  if (!artifact || !artifact.content) → WARN or STOP
  const content = artifact.content
```

**pm_artifacts removal is a two-sided change:**
- KFMB-5040 stops writing `pm_artifacts` to story.yaml
- This story stops reading `pm_artifacts` from story.yaml and removes all consumer references

**Interface change for story-synthesize-agent and pm-story-risk-predictor:**
- Old: receives `story_seed_path` (filesystem path string)
- New: receives `story_id` (story identifier string); reads `story_seed` artifact via `kb_read_artifact({ story_id, artifact_type: "story_seed" })`
- All callers must be updated; this is a cross-agent interface change, not just an internal refactor

**Parallel read pattern (required for pm-story-generation-leader):**

```javascript
const [seedArtifact, testPlanArtifact, feasibilityArtifact, uiuxArtifact] = await Promise.all([
  kb_read_artifact({ story_id, artifact_type: "story_seed" }),
  kb_read_artifact({ story_id, artifact_type: "test_plan" }),
  kb_read_artifact({ story_id, artifact_type: "dev_feasibility" }),
  kb_read_artifact({ story_id, artifact_type: "uiux_notes" }),
])
```

**Protected resources — do not modify:**
- `packages/backend/orchestrator/src/artifacts/` — orchestrator artifact schemas (Zod-validated)
- `@repo/db` client package API surface
- PM writer agents being updated by KFMB-5040

## Infrastructure Notes

No infrastructure changes. All changes are agent markdown files. No deployment required. MCP KB tools (`kb_read_artifact`) are pre-existing registered tools.
