# KFMB-3020: Eliminate stories.index.md — Command Updates

## Context

The `.claude/commands/` layer currently reads and writes `stories.index.md` as a source of story state and for collision detection. Phase 3 agent updates (KFMB-3010) will have already eliminated `stories.index.md` references from approximately 32 agent files in `.claude/agents/`. This story extends that elimination to the command layer — specifically the six command files that directly read or write `stories.index.md` as story state.

The six in-scope command files are:

| Command | Current stories.index.md Usage |
|---------|-------------------------------|
| `story-update.md` | Step 4 writes directly to `{FEATURE_DIR}/stories.index.md` (update status + progress counts). The `--no-index` flag currently skips this. DB write (Step 3a) already exists and is authoritative. |
| `pm-story.md` | Collision Detection reads `stories.index.md` to check for existing entries. Step 3 passes `Index path: {INDEX_PATH}` to leader agents. |
| `story-status.md` | Feature Only and `--depth`/`--deps-order` modes read `stories.index.md`. Single-story mode already uses DB-first routing (already migrated). |
| `code-audit.md` | `/code-audit promote` instructs `audit-promote-leader` to dedup against `stories.index.md`. |
| `elab-epic.md` | Passes `stories_index` path to `elab-epic-setup-leader` via context-init template. |
| `index-update.md` | The command itself reads/writes `stories.index.md` entirely. Candidates for deprecation once callers are updated. |

After this story, no command in `.claude/commands/` reads or writes `stories.index.md` as story state. Downstream stories KFMB-3030 (scripts) and KFMB-4010 (story-move/story-update stage transitions) handle remaining filesystem coupling.

**Dependency**: KFMB-3010 (Agent Updates) must be complete before this story begins. It establishes the KB-native patterns (idiomatic `kb_get_story` usage, KB-native collision detection, removal of `index_path` parameters from agent calls) that the command layer will adopt.

**Coordination**: KFMB-4010 also modifies `story-update.md` for stage directory elimination. Review KFMB-4010 scope before editing to ensure no conflicting changes to the same sections.

## Goal

Remove all live `stories.index.md` reads and writes from the `.claude/commands/` layer. Replace with KB MCP tool calls (`kb_list_stories`, `kb_get_story`, `kb_search`) where story state is needed. Mark `index-update.md` as deprecated.

## Non-Goals

- Do NOT update agent `.md` files in `.claude/agents/` — that is KFMB-3010's scope.
- Do NOT update shell or Python scripts in `scripts/` — that is KFMB-3030's scope.
- Do NOT delete `index-update.md` or `stories.index.md` files — deletion is KFMB-6020 scope.
- Do NOT migrate `platform.stories.index.md` Step 5 in `pm-story.md` — the platform index is structural navigation metadata, not story state, and requires separate analysis.
- Do NOT modify `story-move.md` — its stage-directory elimination is KFMB-4010's scope.
- Do NOT modify the DB shim layer (`shimUpdateStoryStatus`, `story_get_status`) — those are TypeScript source code, not command `.md` files.
- Do NOT change the psql connection string or KB infrastructure — use existing MCP tools only.
- Do NOT modify other commands not listed in scope (e.g., `dev-implement-story.md`, `qa-verify-story.md`, `elab-story.md`).

## Architecture Notes

- No new code, packages, or infrastructure. All changes are instructional markdown edits.
- The KB MCP tools (`mcp__knowledge-base__*`) run against the dedicated PostgreSQL instance at `postgresql://kbuser:TestPassword123!@localhost:5433/knowledgebase`.
- `kb_list_stories` call signature: `{ feature: string, limit?: number }` — the `feature` parameter maps to the story ID prefix (e.g., `"kb-first-migration"` or the lowercase epic slug).
- `--no-index` becomes a no-op post-migration. The flag must not cause an error — it is accepted silently with a deprecation warning emitted. This preserves backward compatibility for any callers that still pass the flag during the transition window.
- `pm-story.md` has **two** collision detection locations that must both be updated:
  1. The inline `## Collision Detection (REQUIRED)` section
  2. The logic embedded in Step 1c / Step 1c-platform
- `platform.stories.index.md` Step 5 in `pm-story.md` is **not** in scope — it is structural navigation metadata (not story state) and will be handled separately.

## Infrastructure Notes

No infrastructure changes required. The `postgres-knowledgebase` MCP server must be reachable for KB tool calls at runtime (development and validation). This is the existing infrastructure at port 5433.
