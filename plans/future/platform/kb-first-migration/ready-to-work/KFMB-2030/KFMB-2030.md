# KFMB-2030: Update /pm-bootstrap-workflow Command for KB-Native Bootstrap

## Context

The `/pm-bootstrap-workflow` command (v5.0.0) orchestrates a three-phase bootstrap pipeline for converting plans into structured story artifacts. It currently operates in two modes: KB Mode (default) and File Mode (legacy `--file` flag).

In the current implementation, KB Mode contains stale references to filesystem-based operations that will no longer be accurate after KFMB-2010 and KFMB-2020 are complete:

- **Phase 0** (`pm-bootstrap-setup-leader`): Currently checks for a `stories.index.md` file on disk as a collision guard. KFMB-2020 rewrites this to use `kb_list_stories` instead.
- **Phase 2** (`pm-bootstrap-generation-leader`): Currently writes `stories.index.md` to disk and inserts stories via raw SQL. KFMB-2010 rewrites this to call `kb_create_story` with no disk index file.
- **Done — KB Mode section**: Lists `pnpm migrate:stories` as the primary story seeding mechanism, which will be obsolete after KFMB-2010 delivers direct `kb_create_story` calls.
- **Completion Report**: Lists `stories.index.md` as a generated file in KB Mode.
- **Reference doc** (`pm-bootstrap-workflow-reference.md`): Artifacts table lists `stories.index.md` unconditionally without distinguishing KB Mode vs File Mode.

This story updates the command and reference doc to accurately reflect the KB-native bootstrap flow after KFMB-2010 and KFMB-2020 are merged.

**Dependency gate**: This story MUST NOT be elaborated or implemented until KFMB-2010 and KFMB-2020 have reached `needs-code-review` or later. Their final agent files are the authoritative input for what the command documentation needs to say.

## Goal

Update `.claude/commands/pm-bootstrap-workflow.md` and `.claude/docs/pm-bootstrap-workflow-reference.md` to accurately reflect the KB-native bootstrap flow: collision detection via `kb_list_stories`, story creation via `kb_create_story`, no `stories.index.md` output in KB Mode, and a corrected completion report. File Mode behavior is preserved unchanged.

## Non-Goals

- This story does NOT rewrite `pm-bootstrap-setup-leader` or `pm-bootstrap-generation-leader` agent logic — that is KFMB-2010 and KFMB-2020.
- This story does NOT eliminate `stories.index.md` from the repository filesystem broadly — that is KFMB-3020/3030.
- This story does NOT modify any other commands (e.g., `/elab-epic`, `/pm-story`, `/story-status`).
- This story does NOT create or modify any TypeScript/code files — documentation-only (`.claude/commands/` and `.claude/docs/` only).
- This story does NOT add new MCP tools to the KB server — those are delivered by KFMB-1020/1030.
- Stage directories (`backlog/`, `elaboration/`, etc.) in the repo are not touched by this story.
- The `protected_features` of this story: File Mode (`--file`) behavior must not be broken or altered.

## Architecture Notes

The command uses a dual-path documentation pattern where KB Mode and File Mode have separate phase prompt templates and Done sections. This split must be preserved:

- **KB Mode**: Phases communicate inline via YAML blocks. No disk-based intermediate artifacts. Story seeding via `kb_create_story` (KFMB-2010).
- **File Mode**: Agents read/write from `{FEATURE_DIR}/_bootstrap/`. Story seeding via `pnpm migrate:stories`.

The edits in this story affect only the KB Mode paths. File Mode sections must be left unchanged.

**Constraint**: Do not collapse KB Mode and File Mode into a single path. Do not reference filesystem paths (e.g., `{feature_dir}/stories.index.md`) in KB Mode documentation. Do not add new MCP tool dependencies not delivered by KFMB-2010 or KFMB-2020.
