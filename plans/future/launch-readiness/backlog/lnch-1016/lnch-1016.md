# lnch-1016: Cache Operations Runbook

## Context

Phase 3 (Before general availability).

The platform uses caching to reduce database load and improve API response times. Cache invalidation bugs are a common source of user-facing stale data issues. Engineers need to know how to inspect cache state, manually invalidate cache entries, and manage the cache during deployments and incidents. This runbook covers application-level cache operations (not CDN/CloudFront, which is covered in frontend deployment runbooks).

## Goal

Write a cache operations runbook at `docs/operations/runbooks/cache-operations.md` documenting how to inspect, invalidate, and manage the application cache layer.

## Non-Goals

- Do not document CloudFront CDN cache (covered in lnch-1011/1012).
- Do not document the cache implementation architecture in detail.
- Do not redesign the caching strategy.

## Scope

**Output file:** `docs/operations/runbooks/cache-operations.md`

**Content outline:**

1. **Overview** — cache technology used (e.g., ElastiCache/Redis or in-memory), what is cached, TTL defaults
2. **Cache key conventions** — naming patterns used in this project
3. **Inspecting cache state** — how to connect to the cache, commands to inspect a specific key
4. **Manual cache invalidation** — invalidating a specific key, invalidating by pattern, full flush (and when NOT to flush)
5. **Cache warming** — if applicable, how to pre-warm cache after a flush
6. **Deployment considerations** — what cache operations are needed before/after a deployment
7. **Troubleshooting stale data** — how to confirm stale cache is the root cause, invalidation procedure
8. **Monitoring cache health** — cache hit rate metric, memory usage, eviction rate
9. **Emergency flush procedure** — full cache flush steps with impact warning
