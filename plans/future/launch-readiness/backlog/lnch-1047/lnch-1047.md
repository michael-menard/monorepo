# lnch-1047: 404 & Error Page Design

## Context

Phase 3 (Before general availability).

The app currently has no designed 404 page or general error page — TanStack Router's default fallback is likely shown. A well-designed 404 page maintains brand trust, provides navigation back into the app, and reduces frustration. An error page (for unhandled crashes) similarly needs to be on-brand and helpful. These pages are simple to build and have a disproportionate impact on first impressions.

## Goal

Implement a custom 404 page and a global error fallback page in the main app, consistent with the LEGO-inspired design system.

## Non-Goals

- Do not implement custom error pages for every error type (that is lnch-1040).
- Do not implement server-side custom error pages (handled at CDN/API layer).
- Do not redesign the full design system.

## Scope

**Output locations:**
- `apps/web/main-app/src/pages/NotFound/` — 404 page component
- `apps/web/main-app/src/pages/ErrorPage/` — global error fallback page
- TanStack Router configuration to use custom 404 route

**Implementation requirements:**

**404 Page:**
- LEGO-inspired design (brick motif or illustration)
- Friendly headline: "Oops! This page got lost in the bricks."
- Short description: "The page you're looking for doesn't exist or has been moved."
- Navigation links: Home, Browse Gallery, Go Back
- Consistent with the design system palette (sky/teal)

**Error Page (global crash fallback):**
- Used by the root ErrorBoundary (from lnch-1040)
- Shows a safe, on-brand message
- "Something went wrong. Our team has been notified."
- Refresh button and Home link
- Does NOT show technical error details to users

**Both pages:**
- Fully accessible (keyboard nav, screen reader, proper heading hierarchy)
- Responsive across mobile and desktop

## Acceptance Criteria

- [ ] Custom 404 page renders when navigating to a non-existent route
- [ ] 404 page is on-brand and includes links back to main app sections
- [ ] Error fallback page renders when ErrorBoundary catches an unhandled error
- [ ] Neither page shows raw error messages or stack traces to users
- [ ] Both pages pass axe-core accessibility scan
- [ ] Both pages are responsive at mobile breakpoints
- [ ] Reviewed and approved by the product designer
