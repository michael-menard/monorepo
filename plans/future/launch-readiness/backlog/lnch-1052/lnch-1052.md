# lnch-1052: E2E UX Improvements Verification

## Context

Phase 2 (Before soft launch).

Once the UX improvements from this epic are implemented (error messages, empty states, loading states, session expiry, onboarding), they need E2E verification to prevent regressions. Unit tests cover component-level behavior but cannot catch issues that only appear in the full browser environment — for example, session expiry requiring an actual Cognito token lifecycle, or accessibility violations that only manifest with real DOM rendering. This story depends on lnch-1051 (the foundational E2E infrastructure) and on the UX stories being complete.

## Goal

Implement Playwright E2E tests verifying all UX improvements from the lnch epic, with a focus on edge cases and scenarios that cannot be tested at the unit level.

## Non-Goals

- Do not duplicate happy path tests from lnch-1051.
- Do not implement visual regression tests (separate effort).
- Do not test UX improvements that are trivially covered by Vitest unit tests.

## Scope

**Output file locations:** `apps/web/playwright/tests/ux/`

**UX scenarios to verify:**

| Scenario | UX Story | Test File |
|----------|----------|-----------|
| Error message display on API failure | lnch-1040 | `ux/error-messages.spec.ts` |
| Empty state renders when gallery is empty | lnch-1041 | `ux/empty-states.spec.ts` |
| Skeleton loading displays before content | lnch-1042 | `ux/loading-states.spec.ts` |
| Session expiry warning appears at 5 min | lnch-1045 | `ux/session-expiry.spec.ts` |
| Session expiry re-auth preserves URL | lnch-1045 | `ux/session-expiry.spec.ts` |
| Onboarding flow completes and persists | lnch-1039 | `ux/onboarding.spec.ts` |
| 404 page renders for invalid routes | lnch-1047 | `ux/error-pages.spec.ts` |
| Accessibility: axe-core on all main routes | lnch-1043 | `ux/accessibility.spec.ts` |
| Feedback widget submits successfully | lnch-1048 | `ux/feedback.spec.ts` |
| Coming soon items are not activatable | lnch-1049 | `ux/coming-soon.spec.ts` |
| Toast appears after MOC creation | lnch-1050 | `ux/success-feedback.spec.ts` |

**Implementation requirements:**

1. **Reuse lnch-1051 fixtures** — authentication, test user setup, Page Object Models
2. **axe-core integration** — use `@axe-core/playwright` for accessibility assertions in `accessibility.spec.ts`
3. **Network interception** — use Playwright's `page.route()` to simulate API failures for error message tests
4. **Session expiry simulation** — mock token expiry for session expiry tests (may require Playwright clock manipulation or Cognito test client)
5. **Viewport coverage** — mobile viewport tests for responsive UX scenarios

## Acceptance Criteria

- [ ] All 11 UX scenario categories have passing Playwright tests
- [ ] axe-core accessibility tests pass on all main app routes (zero Critical violations)
- [ ] Network interception is used to test error message display without real API failures
- [ ] Session expiry flow is tested end-to-end (warning + re-auth + redirect)
- [ ] All tests are isolated and do not depend on global state
- [ ] All tests pass in CI against staging environment
- [ ] Tests reuse authentication fixtures from lnch-1051
- [ ] Reviewed alongside lnch-1051 for consistency of patterns
