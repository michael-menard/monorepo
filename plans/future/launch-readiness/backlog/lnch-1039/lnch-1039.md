# lnch-1039: First-Time User Onboarding Flow

## Context

Phase 1 (Critical path — must be complete before launch).

New users arriving at the LEGO MOC instructions platform for the first time have no context about what the platform does, how to get started, or what they can create. Without an onboarding flow, user activation rates will be low — users will land on the dashboard, not know what to do, and leave. First-time user experience is one of the strongest predictors of retention.

## Goal

Implement a first-time user onboarding flow in the main app that guides new users from account creation to their first meaningful action (viewing a MOC or creating their first MOC).

## Non-Goals

- Do not build a product tour library from scratch — use an existing approach (progressive disclosure, inline tips).
- Do not implement email onboarding sequences (separate marketing effort).
- Do not gate the application behind onboarding — it should be skippable.

## Scope

**Output locations:**
- `apps/web/main-app/src/components/Onboarding/` — onboarding flow components
- `apps/web/main-app/src/components/Onboarding/__tests__/` — component tests
- `apps/web/main-app/src/components/Onboarding/__types__/` — Zod schemas

**Content outline / implementation requirements:**

1. **Welcome screen** — shown once on first login post-registration, brief value proposition, "Get started" CTA
2. **Profile setup step** — display name, optional avatar (skippable)
3. **Explore step** — surface 3-5 featured MOCs from the gallery as inspiration
4. **Create step** — "Create your first MOC" CTA with a brief explanation of what that means
5. **Completion state** — acknowledge completion, dismiss and go to dashboard
6. **Skip functionality** — skip button at every step, remembers skip in user preferences
7. **Return entry point** — "Finish setup" prompt in dashboard for users who skipped
8. **State persistence** — onboarding state stored in user profile (not localStorage alone), survives device switch
9. **Accessibility** — keyboard navigable, screen reader compatible, focus managed between steps

**Component directory structure:**
```
Onboarding/
  index.tsx
  WelcomeStep/index.tsx
  ProfileStep/index.tsx
  ExploreStep/index.tsx
  CreateStep/index.tsx
  __tests__/

  __types__/
    index.ts
```

## Acceptance Criteria

- [ ] Onboarding flow is shown to new users exactly once after first login
- [ ] Every step has a working "Skip" button
- [ ] Onboarding state persists across devices (stored server-side)
- [ ] All steps are keyboard navigable and pass axe-core accessibility scan
- [ ] "Finish setup" prompt appears in dashboard for users who skipped
- [ ] Vitest coverage meets project minimum (45%)
- [ ] Playwright E2E test covers the full happy path (linked to lnch-1051)
- [ ] Reviewed and approved by the product designer
