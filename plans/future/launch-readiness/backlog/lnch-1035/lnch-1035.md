# lnch-1035: Security Incident Response

## Context

Phase 2 (Before soft launch).

Security incidents (data breaches, credential compromise, unauthorized access, DDoS) require a different response playbook from service incidents. They involve legal, compliance, and regulatory considerations in addition to technical remediation. This document depends on lnch-1033 (Secret Rotation Runbook) because emergency secret rotation is a common first response to a security incident.

## Goal

Write a security incident response guide at `docs/operations/security/incident-response.md` that covers the full lifecycle of a security incident from detection to resolution and disclosure.

## Non-Goals

- Do not replace the general on-call playbook (lnch-1024) — this is security-specific.
- Do not document vulnerability management / patch management.
- Do not document penetration testing procedures.

## Scope

**Output file:** `docs/operations/security/incident-response.md`

**Content outline:**

1. **Overview** — what constitutes a security incident, how this differs from service incidents
2. **Security incident types** — credential compromise, data breach, unauthorized access, DDoS, malware
3. **Severity classification for security incidents** — mapping to SEV1-4 with security-specific criteria
4. **Immediate response steps (first 30 minutes)**
   - Contain: isolate affected systems, revoke compromised credentials (link to lnch-1033 emergency rotation)
   - Assess: scope of compromise, data affected, access obtained
   - Preserve: CloudTrail logs, VPC flow logs, relevant Lambda logs — do NOT delete evidence
5. **Evidence collection procedure** — what to collect, how to preserve, chain of custody
6. **Escalation** — security lead, legal, executive team, when each is required
7. **Customer notification** — when legally required to notify users, process, template
8. **Regulatory reporting** — relevant regulations (GDPR, etc.), reporting timelines
9. **Remediation** — patching access vectors, credential rotation, WAF rules
10. **Post-incident** — security-focused post-mortem template additions
11. **Contact list** — security lead, legal counsel, AWS security support
