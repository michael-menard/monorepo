# lnch-1022: Cognito User Management Runbook

## Context

Phase 3 (Before general availability).

Amazon Cognito manages user authentication and authorization. Support teams and engineers will need to perform user management operations: resetting passwords, disabling compromised accounts, looking up users, and managing user groups. Without a runbook, these operations require guessing through the AWS console, creating risk of accidentally modifying the wrong user pool or user.

## Goal

Write a Cognito user management runbook at `docs/operations/runbooks/cognito-management.md` covering user lifecycle operations from a support and operations perspective.

## Non-Goals

- Do not document Cognito configuration or User Pool setup.
- Do not document the authentication flow in the application.
- Do not document Cognito triggers/Lambda authorizer implementation.

## Scope

**Output file:** `docs/operations/runbooks/cognito-management.md`

**Content outline:**

1. **Overview** — User Pool IDs per environment, how to identify the correct pool
2. **Finding a user** — search by email, username, or sub (user ID)
3. **Resetting a user's password** — admin reset procedure, what the user experiences
4. **Disabling a compromised account** — how to disable, impact on active sessions
5. **Re-enabling an account** — procedure and verification
6. **User group management** — listing groups, adding/removing a user from a group
7. **Deleting a user** — confirmation steps, data retention considerations (link to lnch-1036)
8. **MFA management** — checking MFA status, resetting MFA for a user
9. **Bulk operations** — importing users, bulk disable (with warning)
10. **Audit log** — CloudTrail events for Cognito, how to find actions for a specific user
11. **Throttling** — Cognito API limits, how to recognize throttling errors

## Acceptance Criteria

- [ ] Runbook exists at `docs/operations/runbooks/cognito-management.md`
- [ ] User Pool IDs for all environments are referenced (or pointer to where they are stored securely)
- [ ] Account disable/enable procedure is tested
- [ ] Data retention consideration is linked to lnch-1036
- [ ] Reviewed by the security or support lead
