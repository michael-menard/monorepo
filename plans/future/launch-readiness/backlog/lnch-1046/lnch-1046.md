# lnch-1046: Browser Compatibility Testing

## Context

Phase 3 (Before general availability).

The main app uses React 19 and modern web APIs. Not all browsers support the latest features equally, and the Module Federation architecture adds additional compatibility requirements. Browser compatibility has not been systematically verified. LEGO enthusiasts may use a wide range of browsers including Safari on iOS/macOS where compatibility differences are most common.

## Goal

Test the main app against the target browser matrix, document any issues found, and remediate critical failures.

## Non-Goals

- Do not support IE11 or other legacy browsers.
- Do not implement polyfills for features not in the browser support matrix.
- Do not audit the dashboard app.

## Scope

**Output locations:**
- `apps/web/main-app/` — browser-specific fixes
- Documentation of browser support matrix and any known limitations

**Browser support matrix (minimum):**

| Browser | Version | Priority |
|---------|---------|---------|
| Chrome | Latest 2 | Critical |
| Firefox | Latest 2 | High |
| Safari (macOS) | Latest 2 | Critical |
| Safari (iOS) | Latest 2 | Critical |
| Edge | Latest 2 | High |
| Chrome (Android) | Latest | High |

**Testing areas:**
1. All critical user flows (signup, login, create MOC, browse gallery)
2. CSS features (Tailwind, animations, grid/flexbox behavior)
3. Module Federation loading
4. File upload functionality
5. Authentication (Cognito flows)
6. Form behavior (especially on iOS Safari)

## Acceptance Criteria

- [ ] All critical flows work in Chrome, Firefox, Safari (macOS), Safari (iOS), and Edge
- [ ] No critical CSS or JavaScript errors in the browser support matrix
- [ ] File upload works on iOS Safari (known compatibility pain point)
- [ ] Authentication flow works on all target browsers
- [ ] Browser support matrix is documented
- [ ] Known issues below critical threshold are documented with workarounds where possible
- [ ] Tested on physical iOS device (not just emulator)
