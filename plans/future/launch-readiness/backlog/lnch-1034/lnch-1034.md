# lnch-1034: IAM Audit Process

## Context

Phase 2 (Before soft launch).

IAM over-permissioning is one of the most common AWS security risks. Lambda functions, developer accounts, and CI/CD roles accumulate permissions over time. A documented, repeatable audit process ensures the principle of least privilege is maintained and excessive permissions are identified and revoked before they can be exploited. This process should be run before launch and periodically afterward.

## Goal

Write an IAM audit process document at `docs/operations/security/iam-audit.md` that provides a repeatable checklist for auditing IAM roles and policies across the platform.

## Non-Goals

- Do not redesign the IAM architecture.
- Do not document IAM for Cognito (that is lnch-1022).
- Do not automate the audit (separate effort) — document the manual process.

## Scope

**Output file:** `docs/operations/security/iam-audit.md`

**Content outline:**

1. **Overview** — audit frequency (quarterly minimum), who is responsible, audit log location
2. **IAM roles inventory** — table of all IAM roles, their purpose, attached policies, last used date
3. **Audit checklist**
   - Check each Lambda execution role: are all permissions actually used?
   - Check for wildcard resource ARNs (`*`) in policies — document any that are intentional
   - Check CI/CD deployment roles: scope to minimum required actions
   - Check developer IAM users/SSO permissions: verify role assignments match job function
   - Check for unused roles (last used > 90 days)
   - Verify MFA is enforced for all human IAM users
   - Check for hardcoded credentials in code (grep patterns)
4. **Tools for audit** — AWS IAM Access Analyzer, AWS Trusted Advisor, CloudTrail for last-used analysis
5. **Remediation procedure** — how to remove or restrict excessive permissions safely
6. **Documenting exceptions** — when a broad permission is intentional, how to document the justification
7. **Audit report template** — what to record after each audit
8. **Pre-launch specific audit** — prioritized checklist items for the initial launch audit

## Acceptance Criteria

- [ ] Document exists at `docs/operations/security/iam-audit.md`
- [ ] IAM roles inventory table is populated with all production roles
- [ ] Audit checklist covers Lambda roles, CI/CD roles, and human access
- [ ] Pre-launch audit is completed using this document before launch
- [ ] Audit report is filed after pre-launch audit
- [ ] Reviewed and approved by the security lead
