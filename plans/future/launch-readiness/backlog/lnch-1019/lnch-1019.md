# lnch-1019: VPC Troubleshooting Runbook

## Context

Phase 3 (Before general availability).

Lambda functions run inside a VPC to access Aurora PostgreSQL and other private resources. VPC misconfigurations are a common source of Lambda timeouts that look like application bugs. Engineers troubleshooting Lambda connectivity issues (lnch-1013) need to know how to validate VPC configuration and diagnose network-level connectivity problems without needing deep AWS networking expertise.

## Goal

Write a VPC troubleshooting runbook at `docs/operations/runbooks/vpc-troubleshooting.md` that helps engineers diagnose Lambda-to-resource connectivity issues and common VPC configuration problems.

## Non-Goals

- Do not document VPC provisioning or architecture design.
- Do not document general Lambda troubleshooting (lnch-1013).
- Do not document Security Group management as a routine operation.

## Scope

**Output file:** `docs/operations/runbooks/vpc-troubleshooting.md`

**Content outline:**

1. **Overview** — VPC topology for this project (subnets, Lambda placement, Aurora placement)
2. **Common symptoms of VPC issues** — Lambda timeout with no logged error, "Network error" from Lambda to RDS
3. **Diagnosing Lambda-to-RDS connectivity**
   - Verify Lambda VPC config (subnets, security groups) in console
   - Verify Aurora security group allows inbound from Lambda security group
   - Check route tables for private subnets
4. **Diagnosing Lambda-to-OpenSearch connectivity** — similar steps for OpenSearch
5. **Diagnosing Lambda-to-external connectivity** — NAT Gateway configuration, verifying outbound internet access
6. **VPC Flow Logs analysis** — how to enable and read VPC Flow Logs for a specific Lambda ENI
7. **Security Group rules reference** — table of expected SG rules for each resource
8. **When to engage AWS Support** — symptoms that indicate a platform-level issue
9. **Escalation** — infra team contacts
