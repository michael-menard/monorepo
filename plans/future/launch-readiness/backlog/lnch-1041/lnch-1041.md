# lnch-1041: Empty States Audit & Implementation

## Context

Phase 2 (Before soft launch).

Empty states occur when a user has no data to display — an empty gallery, no MOCs created, no search results. Currently, most empty states either show a blank area, a generic loading spinner that never resolves, or raw "No data" text. Good empty states are a conversion opportunity: they explain why the space is empty and provide a clear call to action to fill it. This is particularly critical for new users who will frequently encounter empty states.

## Goal

Audit all empty state occurrences in the main app and implement a consistent, helpful empty state component used everywhere.

## Non-Goals

- Do not redesign page layouts around empty states.
- Do not implement empty states for the dashboard app (out of scope).
- Do not create empty state illustrations from scratch — use existing design system assets or simple icons.

## Scope

**Output locations:**
- `apps/web/main-app/src/components/EmptyState/` — reusable empty state component
- Updates to all pages/components that have empty states

**Implementation requirements:**

1. **EmptyState component** — accepts: icon/illustration, heading, description, optional CTA button props
2. **Empty state inventory** — audit and document all places in the app where empty states occur
3. **Gallery empty state** — "No MOCs yet. Be the first to share your build!" + Browse or Create CTA
4. **My MOCs empty state** — "You haven't created any MOCs yet." + Create your first MOC CTA
5. **Search results empty state** — "No results for '{query}'. Try different keywords." + Clear search link
6. **Notifications empty state** — "You're all caught up!" with friendly icon
7. **Consistent visual style** — neutral icon, short heading, 1-2 sentence description, optional action

**Component structure:**
```
EmptyState/
  index.tsx
  __tests__/
    EmptyState.test.tsx
  __types__/
    index.ts
```

## Acceptance Criteria

- [ ] `EmptyState` component exists and is used at all identified empty state locations
- [ ] At least 4 specific empty states (gallery, my MOCs, search, notifications) are implemented
- [ ] Empty states include a relevant CTA where appropriate
- [ ] Empty states pass axe-core accessibility scan
- [ ] Vitest tests cover the EmptyState component with all prop variants
- [ ] Empty state inventory document is created listing all locations
- [ ] Reviewed and approved by the product designer
