# lnch-1011: Frontend Deployment Runbook

## Context

Phase 1 (Critical path — must be complete before launch).

The frontend consists of multiple apps under `apps/web/` (main-app, app-dashboard) using Module Federation. Deploying the frontend involves building with Turborepo, uploading static assets, and managing cache invalidation. The Module Federation architecture adds complexity — remote entry points must be deployed in the correct order. This runbook ensures deployments are repeatable and do not leave users with broken remote module references.

## Goal

Write a step-by-step frontend deployment runbook at `docs/operations/runbooks/frontend-deployment.md` covering the full deployment sequence for all frontend apps.

## Non-Goals

- Do not document API deployment (lnch-1009).
- Do not document frontend rollback (lnch-1012).
- Do not document Module Federation debugging (lnch-1018) — link to it instead.

## Scope

**Output file:** `docs/operations/runbooks/frontend-deployment.md`

**Content outline:**

1. **Overview** — apps involved (main-app, app-dashboard), hosting platform (S3/CloudFront or other), Module Federation deployment order requirements
2. **Prerequisites** — AWS access, correct bucket/distribution IDs, environment variables, Node/pnpm versions
3. **Pre-deployment checklist** — Playwright E2E passing, Vitest passing, accessibility audit passing (lnch-1043), no open SEV1/2 incidents
4. **Build steps** — `pnpm build` commands, Turborepo cache considerations, verifying build output
5. **Deployment sequence** — why order matters with Module Federation, which app deploys first
6. **Cache invalidation** — CloudFront invalidation commands after deploy
7. **Smoke tests** — URLs to verify, expected page loads, critical flows to check manually
8. **Post-deployment verification** — check browser console for Module Federation errors, check CSP headers
9. **Environment-specific notes** — staging vs production differences
10. **Communicating the deployment** — notification template
11. **When to abort** — smoke test failure criteria
12. **Rollback trigger** — link to lnch-1012
