# lnch-1044: Mobile Responsive Audit

## Context

Phase 2 (Before soft launch).

The main app is built with Tailwind CSS which supports responsive design, but responsiveness has not been systematically audited. LEGO enthusiasts browse on mobile devices, and a poor mobile experience will hurt activation and retention. Common issues include: overflow causing horizontal scroll, tap targets too small (minimum 44x44px per WCAG), modals not fitting viewport, and navigation not adapting to small screens.

## Goal

Audit the main app at common mobile breakpoints and remediate all critical responsive layout and usability issues.

## Non-Goals

- Do not build a native mobile app.
- Do not redesign the information architecture for mobile.
- Do not audit the dashboard app.

## Scope

**Output locations:**
- `apps/web/main-app/src/` — CSS/component fixes across the app
- Breakpoints to test: 375px (iPhone SE), 390px (iPhone 14), 768px (tablet portrait)

**Audit areas:**

1. **Navigation** — hamburger menu on mobile, all routes accessible on small screen
2. **Gallery grid** — appropriate column count at each breakpoint, image sizing
3. **MOC detail page** — image viewer, description layout, action buttons
4. **Forms** — input sizing, keyboard handling on mobile, submit button accessibility
5. **Modals and dialogs** — fit within viewport, scrollable content, close button accessible
6. **Tap targets** — all interactive elements at least 44x44px
7. **Typography** — minimum 16px base font size (prevents iOS auto-zoom on inputs)
8. **Horizontal overflow** — no unintentional horizontal scroll at any breakpoint
9. **Images** — no oversized images causing layout shift

**Testing approach:**
- Chrome DevTools device emulation for all breakpoints
- Physical device test on at least iOS and Android
- Playwright visual regression tests at mobile viewport (if time allows, links to lnch-1051)

## Acceptance Criteria

- [ ] No horizontal scroll at 375px, 390px, or 768px viewport widths
- [ ] Navigation is fully accessible on mobile (all routes reachable)
- [ ] All tap targets are at least 44x44px
- [ ] Base font size is at least 16px (iOS auto-zoom prevention)
- [ ] Gallery, MOC detail, and forms are usable on physical mobile device
- [ ] Modals fit within viewport and are scrollable if content overflows
- [ ] Tested on at least one physical iOS device and one Android device
- [ ] Reviewed by the product designer for mobile UX quality
