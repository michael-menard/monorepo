# lnch-1038: Access Control Matrix

## Context

Phase 3 (Before general availability).

An access control matrix documents which roles can perform which actions on which resources. Without it, new features are built with inconsistent access control, support engineers request the wrong permissions, and security audits take longer. The matrix serves as the authoritative reference for the authorization model and feeds into the IAM audit process (lnch-1034).

## Goal

Write an access control matrix at `docs/operations/security/access-control-matrix.md` documenting all roles, resources, and permitted actions across the platform.

## Non-Goals

- Do not implement new authorization checks in code.
- Do not document Cognito group configuration (implementation detail).
- Do not document AWS IAM permissions (that is lnch-1034).

## Scope

**Output file:** `docs/operations/security/access-control-matrix.md`

**Content outline:**

1. **Overview** — purpose, relationship to IAM audit (lnch-1034), Cognito group mapping
2. **Role definitions** — describe each role: anonymous user, authenticated user, MOC author, admin
3. **Application-level access control matrix**
   | Resource | Action | Anonymous | User | Author | Admin |
   |----------|--------|-----------|------|--------|-------|
   | Gallery | View | Yes | Yes | Yes | Yes |
   | MOC | Create | No | Yes | Yes | Yes |
   | MOC | Edit own | No | No | Yes | Yes |
   | MOC | Delete own | No | No | Yes | Yes |
   | MOC | Delete any | No | No | No | Yes |
   | User account | View own | No | Yes | Yes | Yes |
   | User account | Delete | No | Yes | Yes | Yes |
   | Admin panel | Access | No | No | No | Yes |
4. **API endpoint authorization reference** — which Cognito group or JWT claim is checked for each endpoint category
5. **Cognito group to role mapping** — group names and their corresponding application role
6. **Authorization failure behavior** — 401 vs 403, redirect behavior
7. **How to add a new role** — process and required code changes
8. **Review schedule** — matrix must be updated when new features are added

## Acceptance Criteria

- [ ] Matrix exists at `docs/operations/security/access-control-matrix.md`
- [ ] Matrix covers all application resources and all roles
- [ ] Matrix has been verified against actual authorization code
- [ ] Cognito group mapping is accurate
- [ ] Process for adding new roles is documented
- [ ] Reviewed by the security lead and product manager
