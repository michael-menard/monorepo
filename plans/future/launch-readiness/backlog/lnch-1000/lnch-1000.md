# lnch-1000: README for @repo/ui (app-component-library)

## Context

Phase 1 (Critical path — must be complete before launch).

The `@repo/app-component-library` package (aliased as `@repo/ui`) is the most heavily used shared package in the monorepo. Every frontend app imports from it. Despite this, it currently has no README, making onboarding slow and increasing the risk of misuse. New developers and contributors need clear guidance on the package's architecture, import conventions, and extension patterns before making changes.

The library has two layers: `_primitives/` (thin wrappers around shadcn/Radix components wired to our Tailwind theme) and feature folders (`buttons/`, `cards/`, `selects/`, etc.) containing app-level composed components.

## Goal

Write a comprehensive README at `packages/core/app-component-library/README.md` that documents the package's architecture, usage patterns, and contribution guidelines so that any developer can onboard to the library in under 15 minutes.

## Non-Goals

- Do not document every individual component (that belongs in component-level JSDoc/Storybook).
- Do not restructure the package or change any code.
- Do not create a Storybook instance (separate effort).

## Scope

**Output file:** `packages/core/app-component-library/README.md`

**Content outline:**

1. **Package overview** — purpose, the `@repo/ui` alias, who uses it
2. **Architecture** — two-layer model: `_primitives/` vs feature folders, rationale
3. **Installation / workspace setup** — `workspace:*` dependency, no npm publish
4. **Import conventions** — always import from `@repo/ui`, never from sub-paths; `NO BARREL FILES` rule
5. **Available component categories** — bulleted list of feature folders with one-line descriptions
6. **Adding a new primitive** — checklist: source from shadcn, wire to Tailwind theme, export via `_primitives/`
7. **Adding a new app-level component** — checklist: directory structure (`index.tsx`, `__tests__/`, `__types__/`, `utils/`), Zod schema for props
8. **Styling rules** — Tailwind only, no hardcoded colors, use `cn` utility
9. **Testing** — Vitest + React Testing Library, minimum coverage expectations
10. **Related packages** — links to `@repo/design-system`, `@repo/accessibility`
