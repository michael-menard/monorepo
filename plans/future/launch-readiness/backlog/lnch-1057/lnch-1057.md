# lnch-1057: app-wishlist-gallery Operational Runbook

## Context

Phase 3 (Launch readiness).

`apps/web/app-wishlist-gallery/` is a private, authenticated micro-frontend where users manage their personal list of desired LEGO sets and MOC instructions. Unlike other galleries that serve public content, the wishlist is 100% user-specific: every request must be authenticated and all data is scoped to the authenticated user's identity. This changes its failure mode profile significantly — auth token propagation issues, per-user data isolation bugs, and write-path failures (add/remove wishlist items) are the primary concerns rather than CDN caching or content API availability.

It is built with React 19, TanStack Router, and Module Federation, deployed to its own CloudFront distribution and S3 bucket.

## Goal

A verified runbook at `docs/operations/runbooks/app-wishlist-gallery.md` covering deployment, rollback, health checks, environment variable inventory, and failure modes specific to the authenticated wishlist micro-frontend — with particular attention to auth and per-user data path issues.

## Non-Goals

- Cognito user pool provisioning and token configuration (auth runbook).
- Wishlist API schema and data model (backend runbook).
- CloudFront distribution setup (infra runbook).

## Scope

**Output file:** `docs/operations/runbooks/app-wishlist-gallery.md`

**Runbook sections to write:**

### 1. App Overview
- Purpose: authenticated personal wishlist management for LEGO sets and MOC instructions
- Architecture: Module Federation remote; shell loads it at `/wishlist`, requires auth
- CloudFront distribution ID and S3 bucket per environment
- Upstream dependencies: wishlist API (read + write), Cognito auth, sets/instructions content API (for item metadata)

### 2. Environment Variables
| Variable | Purpose | Example |
|---|---|---|
| `VITE_API_BASE_URL` | Wishlist and content API | `https://api.example.com` |
| `VITE_COGNITO_USER_POOL_ID` | Auth (required — unauthenticated access not supported) | `us-east-1_XXXXXX` |
| `VITE_COGNITO_CLIENT_ID` | Cognito app client | `abc123` |
| `VITE_IMAGE_CDN_BASE_URL` | Set/MOC thumbnail CDN | `https://images.example.com` |
| `VITE_MF_REMOTE_URL` | Module Federation entry URL | `https://wishlist.cdn.example.com/remoteEntry.js` |
| `VITE_ENVIRONMENT` | Environment tag | `prod` |

### 3. Deployment Procedure
1. Pre-deploy checklist: verify wishlist API health, confirm Cognito integration is healthy, no active user-facing incidents on related services
2. Build: `pnpm turbo run build --filter=app-wishlist-gallery`
3. S3 sync with correct cache-control headers (no-cache for `remoteEntry.js`; long TTL for hashed chunks)
4. CloudFront invalidation for `remoteEntry.js` and `index.html`
5. Post-deploy smoke test:
   - Attempt to load `/wishlist` as an unauthenticated user — confirm redirect to login (not a blank screen or error)
   - Log in as a test user with an existing wishlist — confirm wishlist items load
   - Add an item to the wishlist — confirm it persists on reload
   - Remove an item — confirm removal persists

### 4. Rollback Procedure
1. Retrieve previous S3 object versions
2. Restore `remoteEntry.js` and JS chunks via S3 versioning
3. Invalidate CloudFront for affected paths
4. Smoke test with a test user account (add and remove an item)
5. Log rollback event in ops channel

### 5. Health Check
- URL: `https://{wishlist-cdn-domain}/remoteEntry.js` — HTTP 200
- Auth redirect: unauthenticated visit to `/wishlist` must redirect to login, not return 4xx or blank
- Authenticated read: test user can load their wishlist items
- Write path: test user can add and remove an item without error
- API: `GET /api/wishlist` with a valid test token returns HTTP 200

### 6. Common Errors and Remediation

| Error | Likely Cause | Remediation |
|---|---|---|
| Wishlist loads but shows empty despite saved items | Auth token not forwarded by shell to the MF remote; user ID mismatch | Verify shell passes Cognito token in MF context; check API user scope query |
| Adding item returns 400 | Wishlist API validation rejecting the payload structure | Check API Lambda logs; verify request payload schema matches API expectation |
| Unauthenticated users see a blank white screen instead of login redirect | Auth guard in TanStack Router not applied | Check router auth guard config; confirm redirect is set up for unauthenticated access |
| Wishlist saves succeed but items disappear on reload | Race condition in optimistic update or DB write not persisting | Check API response for write confirmation; look for DB transaction errors in Lambda logs |
| `remoteEntry.js` 404 | S3 sync incomplete | Re-sync; re-invalidate CloudFront |
| Module Federation fails to load | Shell MF config references wrong remote URL | Verify `VITE_MF_REMOTE_URL` in env and shell's MF config |

- Auth / Cognito issues: identity team
