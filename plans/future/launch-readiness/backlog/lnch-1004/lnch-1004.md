# lnch-1004: README for upload-types

## Context

Phase 2 (Before soft launch).

`upload-types` in `packages/backend/` provides the Zod schemas and inferred TypeScript types shared between the frontend upload client and the backend Lambda upload handler. Type mismatches between these two sides are a common source of subtle bugs. Documenting the schemas makes it easy to understand the contract and know what must change on both sides if requirements evolve.

## Goal

Write a README at `packages/backend/upload-types/README.md` documenting all exported Zod schemas, their fields, validation rules, and the cross-boundary contract they enforce.

## Non-Goals

- Do not add new schemas or change validation rules.
- Do not document upload business logic (belongs in upload-client or API docs).

## Scope

**Output file:** `packages/backend/upload-types/README.md`

**Content outline:**

1. **Package overview** — purpose: shared type contract between frontend and Lambda
2. **Installation** — workspace dependency, used by both `upload-client` and Lambda handler
3. **Exported schemas** — for each Zod schema: name, fields table (name, type, validation, required/optional), example valid value
4. **Inferred types** — show `z.infer<typeof SchemaName>` pattern
5. **Validation rules** — file size limits, allowed MIME types, naming constraints
6. **Cross-boundary usage** — diagram or explanation of how schemas are used on both sides
7. **Extending schemas** — how to add a field and what both sides need to update
8. **Related packages** — `upload-client` (lnch-1003), `s3-client` (lnch-1008)

## Acceptance Criteria

- [ ] README exists at `packages/backend/upload-types/README.md`
