# lnch-1007: README for mock-data package

## Context

Phase 4 (Nice to have — complete when time allows).

The `mock-data` package in `packages/core/` provides shared test fixtures and factory functions used in Vitest unit/integration tests and MSW handlers. It is a developer-experience package with no runtime impact. Its documentation is low priority relative to operational readiness stories but will reduce confusion when writing new tests.

## Goal

Write a README at `packages/core/mock-data/README.md` documenting the available mock factories, fixture data, and how to add new mocks when new domain entities are introduced.

## Non-Goals

- Do not document MSW setup itself (that belongs in the testing overview docs).
- Do not create new mock factories as part of this story.
- Do not document Playwright fixtures (separate concern).

## Scope

**Output file:** `packages/core/mock-data/README.md`

**Content outline:**

1. **Package overview** — purpose, test-only package, not bundled in production
2. **Installation** — workspace dependency, devDependency pattern
3. **Available factories** — for each factory function: name, what entity it creates, parameters, example output
4. **Using with Vitest** — import pattern, example test using a factory
5. **Using with MSW handlers** — how mock data feeds into MSW response handlers
6. **Seeding patterns** — how to create related/nested entities
7. **Adding a new factory** — step-by-step guide, Zod schema requirement
8. **Avoiding flaky tests** — guidance on deterministic IDs, dates, and randomness

## Acceptance Criteria

- [ ] README exists at `packages/core/mock-data/README.md`
- [ ] All factory functions are listed and described
- [ ] Vitest and MSW usage examples are present
- [ ] "Adding a new factory" guide is included
- [ ] README renders correctly in GitHub markdown preview
