# lnch-1051: E2E Happy Path Journeys

## Context

Phase 2 (Before soft launch).

The platform has unit and integration tests but no E2E test coverage of critical user journeys. Before soft launch, E2E tests are essential to catch regressions that only appear when the full stack is running. The Playwright setup exists in `apps/web/playwright/` but test coverage of the main app flows is missing. This story creates the foundational E2E test suite.

## Goal

Implement Playwright E2E tests for all critical happy path user journeys in the main app, covering signup through the primary value actions.

## Non-Goals

- Do not cover unhappy paths here (that is lnch-1052).
- Do not test the dashboard app in this story.
- Do not implement visual regression tests (separate effort).

## Scope

**Output file locations:** `apps/web/playwright/tests/`

**User journeys to cover:**

| Journey | Steps | Test File |
|---------|-------|-----------|
| Signup to dashboard | Email signup, verify, first login, onboarding | `auth/signup.spec.ts` |
| Login to dashboard | Login with credentials, reach dashboard | `auth/login.spec.ts` |
| Browse gallery | Navigate to gallery, browse MOCs, view detail | `gallery/browse.spec.ts` |
| Create MOC | Navigate to create, fill form, upload image, publish | `moc/create.spec.ts` |
| Edit MOC | Find own MOC, edit details, save | `moc/edit.spec.ts` |
| Delete MOC | Find own MOC, delete, confirm deletion | `moc/delete.spec.ts` |
| Update profile settings | Navigate to settings, update display name, save | `settings/profile.spec.ts` |

**Implementation requirements:**

1. **Test users** — use Playwright fixtures to create isolated test users per test run (not shared state)
2. **Authentication helper** — reusable `loginAs(user)` fixture that handles Cognito auth flow
3. **Test data isolation** — each test creates its own data and does not depend on global state
4. **Retry strategy** — configure Playwright retries (2) for CI flakiness tolerance
5. **Screenshot on failure** — configured for CI artifact uploads
6. **Page Object pattern** — use Page Object Model for maintainability
7. **Environment configuration** — tests run against staging environment, configured via env vars
8. **CI integration** — Playwright tests run in CI on PR merge to main

**Playwright configuration:**
- Browsers: Chromium (required), Firefox (required), WebKit (required)
- Viewport: Desktop (1280x720) and Mobile (375x667)
- Base URL: from `PLAYWRIGHT_BASE_URL` env var

## Acceptance Criteria

- [ ] All 7 journeys have passing Playwright tests
- [ ] Tests run in Chromium, Firefox, and WebKit
- [ ] Tests run at both desktop and mobile viewports
- [ ] No shared state between tests — each test is isolated
- [ ] Authentication helper fixture is reusable across test files
- [ ] All tests pass in CI (GitHub Actions or equivalent) against staging
- [ ] Flaky tests addressed before merging (retry count documented if needed)
- [ ] Page Object Model is used consistently
