# lnch-1002: README for accessibility package

## Context

Phase 2 (Before soft launch).

The `@repo/accessibility` package provides shared a11y utilities used across `apps/web`. This package is tightly coupled to lnch-1043 (WCAG 2.1 AA Accessibility Audit), which is a Phase 1 critical story. Having the README in place before the audit makes the auditor's job easier and ensures utilities are used correctly during remediation work.

## Goal

Write a README at `packages/core/accessibility/README.md` documenting all exported utilities, their intended use, and how they map to WCAG 2.1 AA requirements.

## Non-Goals

- Do not perform accessibility remediation (that is lnch-1043).
- Do not add new utilities unless they are needed for existing documented functionality.
- Do not document axe-core directly — link to its docs instead.

## Scope

**Output file:** `packages/core/accessibility/README.md`

**Content outline:**

1. **Package overview** — purpose, relationship to WCAG 2.1 AA compliance goal
2. **Installation** — workspace dependency
3. **Exported utilities** — document each exported function/hook with signature, description, and example
4. **ARIA helpers** — any helpers for managing ARIA attributes or live regions
5. **Focus management utilities** — focus trap, focus restore, skip links
6. **Keyboard navigation helpers** — key event utilities
7. **WCAG mapping** — table linking utilities to the WCAG criteria they address
8. **Testing with axe-core** — how to use the package in Vitest tests, example test
9. **What to avoid** — common a11y antipatterns this package helps prevent
10. **Related docs** — link to lnch-1043 audit story, WCAG 2.1 AA spec

## Acceptance Criteria

- [ ] README exists at `packages/core/accessibility/README.md`
- [ ] Every exported utility is documented with usage example
- [ ] WCAG criteria mapping table is present
- [ ] axe-core integration example is included
- [ ] README renders correctly in GitHub markdown preview
- [ ] Reviewed by the developer completing lnch-1043
