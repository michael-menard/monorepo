# lnch-1023: S3 Lifecycle Management Runbook

## Context

Phase 4 (Nice to have — complete when time allows).

S3 buckets store MOC instruction files and other assets. Without lifecycle policies and operational procedures, buckets will accumulate stale data, increase storage costs, and potentially expose data that should have been deleted per the data retention policy (lnch-1036). This runbook covers the operational procedures for managing S3 bucket contents and lifecycle policies.

## Goal

Write an S3 lifecycle management runbook at `docs/operations/runbooks/s3-lifecycle.md` documenting how to manage S3 bucket contents, lifecycle policies, and storage costs.

## Non-Goals

- Do not document the s3-client package (lnch-1008).
- Do not document upload operations (lnch-1003).
- Do not design data retention policy (lnch-1036) — reference it.

## Scope

**Output file:** `docs/operations/runbooks/s3-lifecycle.md`

**Content outline:**

1. **Overview** — bucket names per environment, what each bucket stores, versioning status
2. **Bucket inventory** — how to list bucket contents, size per prefix
3. **Lifecycle policies** — current policies per bucket, what transitions/expirations are configured
4. **Adding or modifying lifecycle policies** — steps, testing in non-production first
5. **Deleting objects** — individual object delete, bulk delete by prefix, delete with version markers
6. **Storage class transitions** — when to use Glacier/IA, how to trigger manual transitions
7. **Cross-region replication** — if configured, how to verify replication is working
8. **Storage cost monitoring** — CloudWatch Storage Lens, cost allocation tags
9. **Data retention compliance** — aligning lifecycle rules with lnch-1036 policy
10. **Recovering accidentally deleted objects** — S3 versioning, restoring a deleted version

## Acceptance Criteria

- [ ] Runbook exists at `docs/operations/runbooks/s3-lifecycle.md`
- [ ] Bucket names and current lifecycle policies are documented
- [ ] Bulk delete procedure includes a confirmation/dry-run step
- [ ] Data retention policy (lnch-1036) is referenced
- [ ] Reviewed by the engineer who manages S3 configuration
