# lnch-1027: Communication Templates

## Context

Phase 3 (Before general availability).

During incidents, engineers under stress often struggle with external communication — what to say, how to say it, and when to say it. Pre-written templates for Slack announcements, status page updates, and user-facing messages remove cognitive load during high-pressure situations and ensure consistent, professional communication. This is a quick win (Size S) that benefits both the team and users.

## Goal

Write a set of communication templates at `docs/operations/incidents/communication-templates.md` covering internal (Slack) and external (status page, email) communications for all severity levels.

## Non-Goals

- Do not set up a status page tool or service (separate infrastructure effort).
- Do not define escalation contacts (lnch-1028).
- Do not create actual Slack channel configurations.

## Scope

**Output file:** `docs/operations/incidents/communication-templates.md`

**Content outline:**

1. **Overview** — when to use each template, communication principles (honest, timely, brief)
2. **Internal Slack templates**
   - Incident start announcement (SEV1/2)
   - Status update (every 30 min during SEV1, hourly during SEV2)
   - Incident resolved announcement
   - Escalation request message
3. **Status page templates** — short public-facing messages for each incident phase (investigating, identified, monitoring, resolved)
4. **User-facing email templates** — for major outages requiring user notification
5. **Stakeholder update template** — brief summary for management/non-technical stakeholders
6. **Template variables guide** — `{INCIDENT_ID}`, `{AFFECTED_SERVICE}`, `{ESTIMATED_RESOLUTION}` — how to fill them in
7. **Communication timeline guide** — when to send each type of message relative to incident start

## Acceptance Criteria

- [ ] Templates file exists at `docs/operations/incidents/communication-templates.md`
- [ ] At least 5 distinct templates are provided
- [ ] All templates have placeholder variables marked consistently
- [ ] Communication timeline guidance is included
- [ ] Reviewed by the engineering lead and product/customer success representative
