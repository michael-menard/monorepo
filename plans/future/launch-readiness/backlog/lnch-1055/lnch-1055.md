# lnch-1055: app-instructions-gallery Operational Runbook

## Context

Phase 3 (Launch readiness).

`apps/web/app-instructions-gallery/` is the core product micro-frontend — it is the primary surface where users discover, preview, and download LEGO MOC building instructions. It is the highest-value page on the platform in terms of user intent and conversion. It is built with React 19, TanStack Router, and Module Federation, deployed to its own CloudFront distribution and S3 bucket.

This app has the most complex upstream dependencies of any micro-frontend: the instructions content API, a PDF/asset delivery pipeline, a search index (for filtering), and optional AI-assisted recommendations. A runbook is essential for the on-call team to triage incidents without needing deep familiarity with all these integrations.

## Goal

A verified runbook at `docs/operations/runbooks/app-instructions-gallery.md` covering deployment, rollback, health checks, environment variable inventory, and the failure modes most commonly encountered in the instructions gallery — especially those involving asset delivery and search.

## Non-Goals

- PDF generation pipeline and S3 asset storage management (media/assets runbook).
- Search index provisioning and re-indexing (search infrastructure runbook).
- AI recommendation engine configuration (AI runbook).
- CloudFront distribution setup (infra runbook).

## Scope

**Output file:** `docs/operations/runbooks/app-instructions-gallery.md`

**Runbook sections to write:**

### 1. App Overview
- Purpose: primary gallery for discovering and downloading MOC building instructions
- Architecture: Module Federation remote; shell loads it at `/instructions`
- CloudFront distribution ID and S3 bucket per environment
- Upstream dependencies: instructions API, PDF asset CDN, search service, (optional) recommendations API

### 2. Environment Variables
| Variable | Purpose | Example |
|---|---|---|
| `VITE_API_BASE_URL` | Instructions content API | `https://api.example.com` |
| `VITE_ASSET_CDN_BASE_URL` | PDF and asset delivery CDN | `https://assets.example.com` |
| `VITE_SEARCH_API_URL` | Search service endpoint | `https://search.example.com` |
| `VITE_RECOMMENDATIONS_API_URL` | AI recommendations endpoint (optional) | `https://ai.example.com` |
| `VITE_COGNITO_USER_POOL_ID` | Auth for downloads and saves | `us-east-1_XXXXXX` |
| `VITE_COGNITO_CLIENT_ID` | Cognito app client | `abc123` |
| `VITE_MF_REMOTE_URL` | Module Federation entry URL | `https://instructions.cdn.example.com/remoteEntry.js` |
| `VITE_ENVIRONMENT` | Environment tag | `prod` |

Document SSM paths and which are required vs optional (recommendations API is optional).

### 3. Deployment Procedure
1. Pre-deploy checklist: verify instructions API health, confirm search service is indexing, confirm asset CDN origin is accessible
2. Build: `pnpm turbo run build --filter=app-instructions-gallery`
3. S3 sync with cache-control headers (long TTL hashed chunks, no-cache for `remoteEntry.js` and `index.html`)
4. CloudFront invalidation for `remoteEntry.js` and `index.html`
5. Post-deploy smoke test:
   - Load gallery as anonymous user — confirm instruction cards render
   - Use search filter — confirm filtered results load
   - Attempt download as authenticated user — confirm PDF link resolves
   - Check browser console for no Module Federation errors

### 4. Rollback Procedure
1. Identify last known-good S3 versions for `remoteEntry.js` and JS chunks
2. Restore via S3 versioning (CLI commands documented)
3. Invalidate CloudFront for affected paths
4. Smoke test: repeat post-deploy checks above
5. Log rollback with timestamp and reason in ops channel

### 5. Health Check
- URL: `https://{instructions-cdn-domain}/remoteEntry.js` — HTTP 200
- Gallery render: at least one instruction card visible without login
- Search: filter by a known tag returns results within 2 s
- Asset delivery: a known PDF asset URL returns HTTP 200 from asset CDN
- Auth-gated download: authenticated user can reach a download link (not 403)

### 6. Common Errors and Remediation

| Error | Likely Cause | Remediation |
|---|---|---|
| Gallery renders but search returns no results | Search index stale or service down | Check search service health; trigger re-index if index is stale |
| PDF download link returns 403 | Asset CDN signed URL generation failing or Cognito token not passed | Verify auth token is forwarded to asset API; check signed URL TTL |
| Instruction cards render without thumbnails | Asset CDN origin misconfigured or `VITE_ASSET_CDN_BASE_URL` wrong | Verify env var; check CDN origin S3 bucket policy |
| Recommendations panel missing | Recommendations API unreachable | Verify `VITE_RECOMMENDATIONS_API_URL`; this is non-blocking — gallery should degrade gracefully |
| `remoteEntry.js` 404 | S3 sync incomplete | Re-run sync; re-invalidate |
| Module Federation version mismatch with shell | Shell and instructions-gallery built against different MF versions | Redeploy both in the same pipeline run |
| Slow initial gallery load (>3 s) | Lambda cold start on instructions API | Check provisioned concurrency; verify p99 in CloudWatch |

### 7. Contacts and Escalation

- Instructions content API: backend team
- Asset delivery pipeline: media team
- Search service: search infrastructure team
