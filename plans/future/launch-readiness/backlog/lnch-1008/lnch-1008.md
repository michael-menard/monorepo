# lnch-1008: README for s3-client
 
## Context

Phase 4 (Nice to have — complete when time allows).

The `s3-client` package in `packages/backend/` wraps the AWS SDK S3 client with project-specific configuration and helper functions. It is used by Lambda handlers for object storage operations. Documentation priority is low because operational procedures for S3 (lnch-1023) and upload flows (lnch-1003/1004) are higher priority, but this README should exist before GA to prevent incorrect direct SDK usage.

## Goal

Write a README at `packages/backend/s3-client/README.md` documenting the s3-client API, configuration, error handling, and usage patterns for Lambda handlers.

## Non-Goals

- Do not document S3 bucket policy or lifecycle rules (that is lnch-1023).
- Do not add new S3 operations unless documenting existing ones.
- Do not document the upload-client (that is lnch-1003).

## Scope

**Output file:** `packages/backend/s3-client/README.md`

**Content outline:**

1. **Package overview** — purpose, wraps AWS SDK S3, Lambda-only usage
2. **Installation** — workspace dependency
3. **Configuration** — required env vars (bucket names, region), IAM permissions needed
4. **API reference** — all exported functions with parameters, return types, and examples
5. **Object operations** — get, put, delete, list, presigned URLs
6. **Error handling** — AWS error codes, how errors are surfaced
7. **Testing** — how to mock s3-client in Lambda handler tests
8. **IAM permissions reference** — minimum required S3 actions per operation
9. **Related packages** — `upload-client` (lnch-1003), `upload-types` (lnch-1004)
10. **Related runbooks** — S3 lifecycle management (lnch-1023)
