# lnch-1037: Data Privacy Documentation

## Context

Phase 3 (Before general availability).

Data privacy documentation serves both internal and external audiences: engineers need to understand what data is collected and how it must be handled; users need a privacy policy; legal needs documentation of data flows for compliance. This document is distinct from the data retention policy (lnch-1036) — it covers what data is collected and how it flows, not how long it is kept.

## Goal

Write data privacy documentation at `docs/operations/security/data-privacy.md` covering data collection inventory, data flows, and the basis for processing each data category.

## Non-Goals

- Do not write the user-facing privacy policy (legal/product responsibility).
- Do not document data retention periods (lnch-1036).
- Do not implement privacy controls in code.

## Scope

**Output file:** `docs/operations/security/data-privacy.md`

**Content outline:**

1. **Overview** — purpose, audience (internal), relationship to user-facing privacy policy
2. **Personal data inventory** — what PII/personal data the platform collects
   - Account data: email, username, profile information
   - Usage data: MOC views, interactions, session data
   - Technical data: IP addresses, device information
3. **Data flow diagram** — where data enters the system, how it flows between services, where it exits
4. **Third-party data sharing** — which external services receive data (Cognito, analytics, etc.)
5. **Data processing basis** — for each data category, the legal basis (consent, contractual, legitimate interest)
6. **Sensitive data handling** — what counts as sensitive in our context, how it is protected
7. **Developer guidelines** — rules for engineers: no logging of PII, no storing passwords, encryption requirements
8. **Data subject rights** — internal procedures for handling access requests, deletion requests
9. **Privacy by design checklist** — what engineers must consider when building new features

## Acceptance Criteria

- [ ] Document exists at `docs/operations/security/data-privacy.md`
- [ ] Personal data inventory is comprehensive and reviewed by legal
- [ ] Data flow diagram accurately represents the system
- [ ] Developer guidelines section is actionable and linked from CLAUDE.md or onboarding docs
- [ ] Reviewed and approved by legal before GA launch
