# lnch-1003: README for upload-client

## Context

Phase 2 (Before soft launch).

The `upload-client` package in `packages/backend/` provides the client-side interface for file uploads (MOC instruction images/files) to S3. MOC creation is a critical user journey (covered in lnch-1051 E2E tests). Without documentation, upload error handling and retry logic are a black box to frontend developers and makes debugging upload failures in production harder.

## Goal

Write a README at `packages/backend/upload-client/README.md` that fully documents the upload client API, configuration, error handling, and integration patterns.

## Non-Goals

- Do not document S3 bucket configuration (that is lnch-1008 / lnch-1023).
- Do not add new upload features.
- Do not document the Lambda upload handler (belongs in API docs).

## Scope

**Output file:** `packages/backend/upload-client/README.md`

**Content outline:**

1. **Package overview** — purpose, what it uploads, S3 as the target
2. **Installation** — workspace dependency
3. **Configuration** — required environment variables, presigned URL endpoint
4. **API reference** — all exported functions with TypeScript signatures (Zod schemas), parameters, return types
5. **Basic upload example** — single file upload end-to-end
6. **Multi-file upload** — if supported, how to handle batches
7. **Error handling** — error types, retry behavior, how to surface errors to the user
8. **Progress tracking** — upload progress callbacks/events if available
9. **File type and size validation** — what is validated client-side vs server-side
10. **Testing** — how to mock the upload client in Vitest tests with MSW
11. **Related packages** — `upload-types` (lnch-1004), `s3-client` (lnch-1008)
