# lnch-1054: app-inspiration-gallery Operational Runbook

## Context

Phase 3 (Launch readiness).

`apps/web/app-inspiration-gallery/` is a public-facing micro-frontend that surfaces curated LEGO MOC inspiration content — images, creator spotlights, and themed collections. It is one of the high-traffic entry points for new users discovering the platform. The app is built with React 19 and TanStack Router, deployed as a Module Federation remote to its own S3 bucket and CloudFront distribution.

Because it is public-facing and image-heavy, its failure modes differ from other micro-frontends: CDN caching behaviour for large image assets, image optimisation pipeline dependencies, and content API latency all have a direct effect on user experience. A runbook specific to this app is required to enable the on-call team to respond to issues without needing to understand the full monorepo.

## Goal

A verified runbook at `docs/operations/runbooks/app-inspiration-gallery.md` covering deployment, rollback, health checks, environment variable inventory, and the failure modes most likely to occur in a high-traffic, image-heavy public gallery micro-frontend.

## Non-Goals

- CloudFront distribution provisioning and cache policy configuration (infra runbook).
- Image upload pipeline and S3 media bucket management (media runbook).
- Content moderation workflows for inspiration items (dashboard runbook).
- Shared Module Federation shell wiring (main-app runbook).

## Scope

**Output file:** `docs/operations/runbooks/app-inspiration-gallery.md`

**Runbook sections to write:**

### 1. App Overview
- Purpose: public inspiration gallery, unauthenticated read access, authenticated saves/likes
- Architecture: Module Federation remote, served from dedicated CloudFront distribution
- CloudFront distribution ID and S3 bucket name per environment
- Upstream dependencies: content API (inspiration endpoints), image CDN origin

### 2. Environment Variables
| Variable | Purpose | Example |
|---|---|---|
| `VITE_API_BASE_URL` | Content API endpoint | `https://api.example.com` |
| `VITE_IMAGE_CDN_BASE_URL` | CDN prefix for MOC images | `https://images.example.com` |
| `VITE_COGNITO_USER_POOL_ID` | Optional auth for saves/likes | `us-east-1_XXXXXX` |
| `VITE_COGNITO_CLIENT_ID` | Cognito app client | `abc123` |
| `VITE_MF_REMOTE_URL` | Module Federation entry URL | `https://inspiration.cdn.example.com/remoteEntry.js` |
| `VITE_ENVIRONMENT` | Environment tag | `prod` |

Document SSM Parameter Store paths and build-time injection method.

### 3. Deployment Procedure
1. Pre-deploy checklist: confirm content API is healthy, image CDN is accessible, no active traffic spike (check CloudWatch)
2. Build: `pnpm turbo run build --filter=app-inspiration-gallery`
3. S3 sync with correct cache-control headers (long TTL for hashed assets, short TTL for `remoteEntry.js`)
4. CloudFront invalidation scoped to `/remoteEntry.js` and `/index.html` (avoid full `/*` invalidation where possible to control costs)
5. Post-deploy smoke test: load gallery as anonymous user, verify images load, verify like button appears when authenticated

### 4. Rollback Procedure
1. Retrieve previous build's S3 object versions for `remoteEntry.js` and all hashed JS chunks
2. Restore via S3 versioning (document the CLI command)
3. Invalidate CloudFront for affected paths
4. Confirm restoration by checking the build hash in the browser `remoteEntry.js` response
5. Log rollback event in ops channel

### 5. Health Check
- URL: `https://{inspiration-cdn-domain}/remoteEntry.js` — HTTP 200, `Content-Type: application/javascript`
- Gallery load check: open gallery in browser, confirm at least one inspiration card renders with image
- API dependency check: `GET /api/inspiration?limit=1` returns HTTP 200 with at least one item
- Image CDN check: any image URL from the API response loads with HTTP 200

### 6. Common Errors and Remediation

| Error | Likely Cause | Remediation |
|---|---|---|
| Images fail to load (broken image icons) | `VITE_IMAGE_CDN_BASE_URL` misconfigured or CDN origin down | Check env var value; verify image CDN origin S3 bucket policy |
| Gallery shows empty state despite content existing | Content API returning 500 or pagination bug | Check content API Lambda logs; verify DB connection pool |
| `remoteEntry.js` 404 | S3 sync did not complete before invalidation | Re-run S3 sync; re-invalidate CloudFront |
| Like button missing for authenticated users | Cognito env vars incorrect or token not propagated to MF | Verify Cognito vars; check shell passes auth context to remote |
| High TTFB on gallery load | Lambda cold start on content API | Check provisioned concurrency config; monitor p99 latency in CloudWatch |
| Stale content after editorial update | CloudFront TTL too long on API responses | Manually invalidate `/api/inspiration*` or reduce TTL for API origin |

### 7. Contacts and Escalation
- Platform on-call: `#ops-escalation`
- Content API owner for backend issues
- Image CDN / media pipeline team for asset delivery issues
