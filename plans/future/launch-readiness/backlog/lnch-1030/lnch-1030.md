# lnch-1030: Alarm Response Runbooks

## Context

Phase 2 (Before soft launch).

Every CloudWatch alarm should have a corresponding response procedure. When an alarm fires, the on-call engineer should not have to improvise — they should follow a documented playbook for that specific alarm. This story creates alarm-specific runbooks for all production alarms. It depends on lnch-1029 (CloudWatch Dashboard Guide) because alarm runbooks reference specific dashboard panels for context.

## Goal

Write alarm response runbooks at `docs/operations/monitoring/alarm-response.md` that provide step-by-step response procedures for every production CloudWatch alarm.

## Non-Goals

- Do not create or modify CloudWatch alarms (infrastructure work).
- Do not document CloudWatch dashboard navigation in depth (lnch-1029).
- Do not document all possible failure modes — focus on the specific alarm trigger.

## Scope

**Output file:** `docs/operations/monitoring/alarm-response.md`

**Content outline:**

For each alarm below, document: alarm name, what it means, initial severity, diagnosis steps, resolution steps, and escalation criteria.

1. **API Error Rate High** — Lambda 5xx rate above threshold
2. **API Latency P99 High** — p99 latency exceeds SLO
3. **Lambda Throttling** — invocations being throttled
4. **Lambda Error Rate** — per-function error rate alarms
5. **Aurora CPU High** — database CPU above threshold
6. **Aurora Connection Count High** — connections approaching limit
7. **Aurora Free Storage Low** — storage dropping below threshold
8. **Aurora Replica Lag High** — read replica lagging behind primary
9. **OpenSearch Cluster Status Red/Yellow** — cluster health degraded
10. **CloudFront Error Rate High** — CDN serving elevated errors
11. **API Gateway 4xx Rate High** — client errors elevated (could indicate auth issues)
12. **Lambda Duration P99 Approaching Timeout** — functions running close to timeout limit

Each alarm entry includes:
- Alarm name (exact AWS name)
- Severity classification guide (usually SEV2 or SEV3, with escalation to SEV1 criteria)
- Relevant dashboard panel (link to lnch-1029)
- Diagnosis steps
- Resolution steps
- Links to detailed runbooks (lnch-1013, lnch-1015, etc.)
