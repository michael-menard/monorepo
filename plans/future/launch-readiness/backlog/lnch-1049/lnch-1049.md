# lnch-1049: Navigation for Incomplete Features

## Context

Phase 2 (Before soft launch).

Some features in the main app are partially built or planned but not yet ready for users. These features may appear in the navigation or be discoverable through the UI. Exposing incomplete features at launch erodes user trust and creates confusing dead ends. This is a quick win (Size S) — incomplete features should be hidden from the nav or clearly marked as "Coming soon" with a consistent pattern.

## Goal

Audit all navigation items and feature entry points, identify incomplete features, and apply a consistent pattern for hiding or labeling them.

## Non-Goals

- Do not complete the unfinished features themselves.
- Do not remove code for incomplete features — only hide entry points.
- Do not create a public roadmap (separate product decision).

## Scope

**Output locations:**
- `apps/web/main-app/src/components/Navigation/` — navigation component updates
- Any component that has a link or entry point to an incomplete feature

**Implementation requirements:**

1. **Incomplete feature inventory** — list all features visible in the UI that are not production-ready
2. **Decision per feature** — hide completely, OR show with "Coming soon" badge
3. **"Coming soon" component** — small badge/chip that can be applied to navigation items and buttons
4. **Disabled state** — incomplete features shown as "Coming soon" must be non-clickable (not just visually disabled — must prevent keyboard activation and inform screen readers)
5. **Routing guard** — if a user navigates directly to a coming-soon URL, redirect to a "coming soon" page or home
6. **Consistent copy** — all labels use "Coming soon" (not "Beta", "In progress", etc. — one term)

**Component structure:**
```
ComingSoon/
  index.tsx          # Coming soon badge component
  __tests__/
  __types__/
```

## Acceptance Criteria

- [ ] Incomplete feature inventory is documented
- [ ] All incomplete features are either hidden from navigation or display a "Coming soon" badge
- [ ] "Coming soon" items are not clickable via mouse or keyboard
- [ ] Screen readers are informed about "Coming soon" state (aria-disabled, aria-label)
- [ ] Direct URL navigation to incomplete feature routes redirects appropriately
- [ ] All labels use consistent "Coming soon" terminology
- [ ] Vitest tests cover the ComingSoon badge component and routing guard
- [ ] Reviewed by the product manager to confirm the feature inventory
