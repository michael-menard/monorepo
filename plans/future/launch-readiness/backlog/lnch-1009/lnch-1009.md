# lnch-1009: API Deployment Runbook

## Context

Phase 1 (Critical path — must be complete before launch).

The API (`apps/api/`) is a serverless application deployed to AWS Lambda via API Gateway. Deploying it correctly requires a specific sequence of steps including pre-deployment checks, CDK deploy commands, smoke tests, and post-deploy verification. Without a written runbook, deployments are fragile and dependent on tribal knowledge. This runbook is a prerequisite for the on-call playbook (lnch-1024) and is needed before any production traffic runs.

## Goal

Write a step-by-step deployment runbook at `docs/operations/runbooks/api-deployment.md` that any engineer with AWS access can follow to deploy the API safely.

## Non-Goals

- Do not document infrastructure provisioning (VPC, Aurora setup) — that belongs in infra runbooks.
- Do not document rollback — that is lnch-1010.
- Do not create CI/CD pipelines (separate effort).

## Scope

**Output file:** `docs/operations/runbooks/api-deployment.md`

**Content outline:**

1. **Overview** — what this deploys (Lambda functions, API Gateway), estimated duration
2. **Prerequisites** — AWS CLI configured, correct IAM role assumed, environment variables set, pnpm build passing
3. **Pre-deployment checklist** — verify tests pass, check for open SEV1/SEV2 incidents, confirm deployment window
4. **Step-by-step deployment**
   a. Build the API: `pnpm build` from repo root or `apps/api/`
   b. Run type checks: `pnpm check-types`
   c. Run tests: `pnpm test`
   d. Deploy via CDK (or SAM/Serverless — actual command TBD by implementer after reading the codebase)
   e. Verify Lambda function versions updated in AWS console
5. **Smoke tests** — specific API endpoints to hit after deploy, expected responses
6. **Post-deployment verification** — CloudWatch logs check, error rate baseline, latency check
7. **Deployment to each environment** — dev, staging, production differences
8. **Communicating the deployment** — who to notify, what to post in which Slack channel
9. **When to abort** — criteria for stopping mid-deployment
10. **Rollback trigger** — if smoke tests fail, link to lnch-1010

## Acceptance Criteria

- [ ] Runbook exists at `docs/operations/runbooks/api-deployment.md`
- [ ] All steps are numbered and unambiguous
- [ ] Prerequisites section lists exact tools and permissions needed
- [ ] Smoke test commands are specific (actual endpoint URLs/paths, not placeholders)
- [ ] Rollback link to lnch-1010 runbook is present
- [ ] A non-original author follows the runbook on staging and confirms it works end-to-end
- [ ] Reviewed and approved by the engineer who owns API deployment
