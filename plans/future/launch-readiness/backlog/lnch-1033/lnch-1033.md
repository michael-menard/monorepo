# lnch-1033: Secret Rotation Runbook

## Context

Phase 2 (Before soft launch).

Secrets (database credentials, API keys, Cognito secrets, third-party service credentials) must be rotated regularly and immediately when compromised. Without a documented rotation procedure, secret rotation is either skipped or done incorrectly, leaving the platform vulnerable. This runbook is referenced by lnch-1035 (Security Incident Response), which covers the emergency rotation path.

## Goal

Write a secret rotation runbook at `docs/operations/security/secret-rotation.md` documenting the rotation procedure for every secret type used in the platform.

## Non-Goals

- Do not document secret storage architecture (AWS Secrets Manager configuration).
- Do not document IAM key rotation (that is lnch-1034).
- Do not handle the rotation automation code — document the manual procedure.

## Scope

**Output file:** `docs/operations/security/secret-rotation.md`

**Content outline:**

1. **Overview** — secrets inventory, storage location (AWS Secrets Manager), rotation schedule
2. **Secrets inventory table** — secret name, type, rotation frequency, last rotated, owner
3. **Routine rotation procedure**
   - Pre-rotation: notify team, schedule maintenance window if needed
   - Generate new secret value
   - Update in AWS Secrets Manager
   - Verify Lambda functions pick up the new value (Lambda environment variable refresh or Secrets Manager SDK)
   - Verify application health after rotation
   - Update documentation
4. **Database credential rotation** — specific steps for Aurora PostgreSQL credentials, zero-downtime rotation strategy
5. **API key rotation** — third-party service key rotation (generic procedure)
6. **Cognito client secret rotation** — app client secret update procedure
7. **Emergency rotation** — immediate rotation when a secret is suspected compromised (link from lnch-1035)
8. **Rotation verification** — how to confirm the new secret is being used
9. **Rollback** — restoring previous secret value if rotation causes an outage
10. **Audit log** — CloudTrail events for secret access, rotation logging

## Acceptance Criteria
