# lnch-1005: README for gallery package

## Context

Phase 3 (Before general availability).

The `gallery` package in `packages/core/` provides gallery display components used for browsing MOC instructions. It is a user-facing feature package but its priority is medium because the gallery view is not gated on critical onboarding flows. Phase 1 and 2 stories need to land first.

## Goal

Write a README at `packages/core/gallery/README.md` documenting the gallery package's components, props, data requirements, and customization options.

## Non-Goals

- Do not redesign gallery components.
- Do not add new gallery features.
- Do not document backend gallery API endpoints (belongs in API docs).

## Scope

**Output file:** `packages/core/gallery/README.md`

**Content outline:**

1. **Package overview** — purpose, which app(s) consume it
2. **Installation** — workspace dependency
3. **Exported components** — for each component: description, props table (name, type, required, description), usage example
4. **Data shape** — what data structure the gallery expects (Zod schema reference)
5. **Filtering and sorting** — how filter/sort props work if applicable
6. **Pagination / infinite scroll** — behavior and configuration
7. **Empty state** — what renders when there are no items (links to lnch-1041)
8. **Loading state** — skeleton behavior (links to lnch-1042)
9. **Accessibility** — keyboard navigation, ARIA roles used
10. **Customization** — Tailwind class overrides, theming
11. **Testing** — example test with mock data

## Acceptance Criteria

- [ ] README exists at `packages/core/gallery/README.md`
- [ ] All exported components are documented with props tables
- [ ] Data shape / Zod schema is documented
- [ ] Empty and loading state behaviors are described
- [ ] Usage example compiles without type errors
- [ ] README renders correctly in GitHub markdown preview
