# lnch-1006: README for charts package

## Context

Phase 4 (Nice to have — complete when time allows).

The `charts` package in `packages/core/` provides chart/visualization components used in the dashboard app (`apps/web/app-dashboard/`). Charts are not on the critical user journey for the main MOC instructions app and are lower priority than documentation needed for operational readiness. This story is deferred to Phase 4.

## Goal

Write a README at `packages/core/charts/README.md` that documents the available chart components, their data requirements, and customization options.

## Non-Goals

- Do not add new chart types.
- Do not migrate chart library versions.
- Do not document the dashboard app itself.

## Scope

**Output file:** `packages/core/charts/README.md`

**Content outline:**

1. **Package overview** — purpose, which app uses it, underlying chart library
2. **Installation** — workspace dependency
3. **Exported components** — for each chart type: description, props table, data format, example
4. **Theming** — color palette integration with LEGO design system
5. **Responsive behavior** — how charts adapt to container size
6. **Accessibility** — chart accessibility considerations, ARIA labels, alternative data tables
7. **Performance** — lazy loading, large dataset handling
8. **Testing** — how to test chart components with mock data
9. **Adding a new chart type** — contribution guide

## Acceptance Criteria

- [ ] README exists at `packages/core/charts/README.md`
- [ ] All exported chart components are documented
- [ ] Data format for each chart type is clearly specified
- [ ] Accessibility section addresses chart-specific a11y requirements
- [ ] README renders correctly in GitHub markdown preview
