# lnch-1056: app-sets-gallery Operational Runbook

## Context

Phase 3 (Launch readiness).

`apps/web/app-sets-gallery/` is a public-facing micro-frontend that lets users browse official LEGO sets — used as a reference layer when viewing MOC instructions or adding items to their wishlist. It is data-heavy and read-mostly: the sets catalog is large (tens of thousands of entries) and is populated from an upstream LEGO sets data source. It is built with React 19, TanStack Router, and Module Federation, deployed to its own CloudFront distribution and S3 bucket.

Because the sets catalog data has a distinct sync/refresh lifecycle (periodically refreshed from an upstream source rather than user-generated), the failure modes here include data freshness issues that do not exist in other galleries. This runbook covers deployment and also the data refresh procedure.

## Goal

A verified runbook at `docs/operations/runbooks/app-sets-gallery.md` covering deployment, rollback, health checks, environment variable inventory, data refresh procedure, and failure modes specific to the sets gallery.

## Non-Goals

- Upstream LEGO sets data ingestion pipeline provisioning (data platform runbook).
- Search index setup for sets catalog (search infrastructure runbook).
- CloudFront distribution provisioning (infra runbook).

### 1. App Overview
- Purpose: browse official LEGO sets catalog; supports filtering by theme, year, piece count
- Architecture: Module Federation remote; shell loads it at `/sets`
- CloudFront distribution ID and S3 bucket per environment
- Upstream dependencies: sets catalog API, sets data sync job, image CDN for set box art

### 3. Deployment Procedure
1. Pre-deploy checklist: verify sets API health, confirm last data sync completed successfully (check sync job logs in CloudWatch)
2. Build: `pnpm turbo run build --filter=app-sets-gallery`
3. S3 sync with correct cache-control headers
4. CloudFront invalidation for `remoteEntry.js` and `index.html`
5. Post-deploy smoke test:
   - Load gallery anonymously — confirm set cards render with box art
   - Apply a theme filter — confirm filtered results
   - Check browser console for no Module Federation errors

### 4. Sets Data Refresh Procedure
The sets catalog is refreshed on a schedule. If data appears stale or a manual refresh is needed:
1. Identify the data sync Lambda function name (document name per environment)
2. Trigger a manual invocation via AWS Console or CLI: `aws lambda invoke --function-name sets-sync-{env} /tmp/out.json`
3. Monitor sync job logs in CloudWatch for completion (expected duration: document in runbook)
4. After sync, verify new set data appears in the API: `GET /api/sets?sort=created_at&order=desc&limit=5`
5. If the search index is stale after a sync, trigger a re-index (document the command)

### 5. Rollback Procedure
1. Identify previous S3 versions for `remoteEntry.js` and JS chunks
2. Restore via S3 versioning
3. Invalidate CloudFront
4. Smoke test gallery load
5. Note: data rollback (reverting a bad sync) is a separate procedure — document the sync Lambda rollback mechanism (idempotent re-run vs snapshot restore)

### 6. Health Check
- URL: `https://{sets-cdn-domain}/remoteEntry.js` — HTTP 200
- Gallery render: set cards visible with box art images
- Filter: theme filter returns scoped results
- API freshness: `GET /api/sets/sync-status` returns timestamp of last successful sync within expected window

### 7. Common Errors and Remediation

| Error | Likely Cause | Remediation |
|---|---|---|
| Set cards render without box art | Image CDN origin misconfigured or box art missing for new sets | Check `VITE_IMAGE_CDN_BASE_URL`; verify box art assets were included in last sync |
| Gallery shows stale sets (missing new releases) | Data sync job failed silently | Check CloudWatch sync job logs; trigger manual re-sync |
| Filter returns no results for valid theme | Search index not updated after sync | Re-trigger search index build for sets catalog |
| Gallery renders empty state despite synced data | Sets API returning empty response due to query bug or DB issue | Check API Lambda logs; verify DB row count in sets table |
| `remoteEntry.js` 404 | S3 sync incomplete | Re-run sync; re-invalidate CloudFront |
| High latency on large set list pages | Pagination not applied or N+1 query in sets API | Check API query patterns; verify pagination params are passed |
