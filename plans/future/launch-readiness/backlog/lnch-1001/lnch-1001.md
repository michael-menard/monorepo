# lnch-1001: README for @repo/logger

## Context

Phase 1 (Critical path — must be complete before launch).

`@repo/logger` is a required import throughout the codebase. CLAUDE.md explicitly forbids `console.log` and mandates use of `logger` from this package. Despite being a hard project rule, the package has no README explaining its API, log levels, or configuration. This creates a constant friction point — developers either forget the rule, use it incorrectly, or have to dig into source code to understand available methods.

This is a quick win (Size S) and should be completed early in Phase 1.

## Goal

Write a concise README at `packages/core/logger/README.md` that fully documents the logger API, log levels, environment configuration, and usage patterns so that any developer knows exactly how to use it without reading source code.

## Non-Goals

- Do not add new features or log levels to the logger.
- Do not change any logger source code.
- Do not document internal implementation details beyond what's needed for usage.

## Scope

**Output file:** `packages/core/logger/README.md`

**Content outline:**

1. **Package overview** — purpose, why `console.log` is forbidden, the `@repo/logger` alias
2. **Installation** — workspace dependency setup
3. **Basic usage** — import pattern, `logger.info`, `logger.warn`, `logger.error`, `logger.debug`
4. **Log levels** — table of levels, when to use each
5. **Structured logging** — how to pass context objects, example with request IDs
6. **Environment configuration** — LOG_LEVEL env var, defaults per environment
7. **Usage in Lambda handlers** — any special considerations for serverless context
8. **Usage in React components** — client-side logging, avoiding sensitive data
9. **What NOT to do** — `console.log`, `console.error`, logging PII
10. **Related packages** — monitoring docs pointer (`docs/operations/monitoring/`)

## Acceptance Criteria

- [ ] README exists at `packages/core/logger/README.md`

- [ ] Log levels table is present and accurate
- [ ] At least one Lambda and one React usage example are included
