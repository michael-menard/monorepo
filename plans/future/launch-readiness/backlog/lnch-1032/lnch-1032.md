# lnch-1032: Log Analysis Guide

## Context

Phase 3 (Before general availability).

CloudWatch Logs and structured logs from `@repo/logger` are the primary debugging tool for incidents and investigations. Without a guide, engineers spend significant time discovering log group names, learning Logs Insights query syntax, and constructing useful queries from scratch during incidents. A guide with pre-built queries dramatically reduces investigation time.

## Goal

Write a log analysis guide at `docs/operations/monitoring/log-analysis.md` that documents log group names, log format, and a library of useful CloudWatch Logs Insights queries for common investigation scenarios.

## Non-Goals

- Do not document the logger package implementation (lnch-1001).
- Do not document X-Ray tracing (covered in lnch-1013).
- Do not build a log aggregation pipeline.

## Scope

**Output file:** `docs/operations/monitoring/log-analysis.md`

**Content outline:**

1. **Overview** — where logs live, retention policy per log group
2. **Log group reference** — table of all log groups, what they contain, retention period
3. **Log format** — JSON log structure from `@repo/logger`, standard fields (timestamp, level, message, requestId, userId, etc.)
4. **CloudWatch Logs Insights basics** — accessing Logs Insights, time range selection, query structure
5. **Pre-built query library**
   - All errors in the last hour
   - Errors for a specific request ID
   - Errors for a specific user
   - Slow requests (duration > threshold)
   - Specific Lambda function errors
   - Aurora connection errors
   - Auth failures
   - Requests by status code breakdown
6. **Log correlation** — how to correlate Lambda logs with API Gateway access logs using request ID
7. **Exporting logs** — how to export a log stream for offline analysis
8. **Log-based alerting** — how to create a metric filter from a log pattern

## Acceptance Criteria

- [ ] Guide exists at `docs/operations/monitoring/log-analysis.md`
- [ ] All log groups are listed in the reference table
- [ ] At least 8 pre-built Logs Insights queries are provided and tested
- [ ] Queries are verified against real log data (not hypothetical)
- [ ] Log correlation procedure is documented
- [ ] Reviewed by an engineer who regularly does log-based debugging
