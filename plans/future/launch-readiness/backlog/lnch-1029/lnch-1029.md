# lnch-1029: CloudWatch Dashboard Guide

## Context

Phase 2 (Before soft launch).

CloudWatch dashboards are the primary operational visibility tool. Without documentation, engineers waste time during incidents clicking through unfamiliar dashboards to find the metrics that matter. This guide must be created before lnch-1030 (Alarm Response Runbooks), which references specific dashboard panels. It is a prerequisite for effective on-call response.

## Goal

Write a CloudWatch dashboard guide at `docs/operations/monitoring/cloudwatch-dashboard.md` that gives any engineer a quick orientation to the dashboards and explains what each panel shows and why it matters.

## Non-Goals

- Do not create new CloudWatch dashboards as part of this story.
- Do not document every possible CloudWatch metric — focus on the dashboards that exist.
- Do not document X-Ray traces in depth (covered in lnch-1013).

## Scope

**Output file:** `docs/operations/monitoring/cloudwatch-dashboard.md`

**Content outline:**

1. **Overview** — how to access CloudWatch dashboards in the AWS console, dashboard naming convention
2. **Dashboard inventory** — table of dashboard names, their purpose, and which environment they cover
3. **Main API dashboard** — panel-by-panel description
   - Lambda invocation count, error rate, duration
   - API Gateway request count, 4xx rate, 5xx rate, latency (p50, p95, p99)
   - Aurora connection count, CPU, read/write IOPS
4. **Frontend / CDN dashboard** — CloudFront request count, error rate, cache hit ratio
5. **Infrastructure dashboard** — VPC flow logs summary, ECS/Lambda concurrency
6. **Alarm state panel** — how to read the alarm state widget, what each color means
7. **Time range selection** — best practices for incident investigation (start with 1h, zoom out)
8. **Creating saved time ranges** — how to bookmark a view
9. **Cross-linking to runbooks** — for each dashboard panel, which runbook to consult if a metric is bad
10. **Dashboard maintenance** — who owns each dashboard, how to propose changes

## Acceptance Criteria

- [ ] Guide exists at `docs/operations/monitoring/cloudwatch-dashboard.md`
- [ ] All existing dashboards are listed in the inventory table
- [ ] Every panel in the main API dashboard is documented with a description and normal ranges
- [ ] Each metric links to the relevant runbook for response
- [ ] An engineer unfamiliar with the dashboards can navigate to any metric in under 2 minutes using the guide
- [ ] Reviewed and approved by the engineer who built the dashboards
