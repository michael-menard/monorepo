# ORCH-3010 — Integration Tests: Elaboration Delta Cluster

## Context

The elaboration graph (`FLOW-043`) orchestrates incremental story refinement through a
multi-node pipeline. The delta cluster — `delta_detect`, `delta_review`, and `escape_hatch_eval`
— forms the core of this pipeline and operates entirely as pure computation over
`SynthesizedStory` objects, with no LLM calls.

Predecessor stories:

- **ORCH-1010** (completed): Delivered shared test factories — `createMockLLMProvider` and
  `createMockGraphState`. Established the `vi.hoisted()` + `vi.mock()` compliance pattern.
- **ORCH-2010** (in_progress): Unit tests for conditional edge routing functions in
  `elaboration.ts` (`afterDeltaDetect`, `afterEscapeHatch`). Adds export keywords to
  private routing functions. ORCH-3010 must not depend on these exports; it calls
  `runElaboration` at the top level.

Current gap: `elaboration.test.ts` tests individual node functions in isolation but does
NOT verify the full `delta_detect → delta_review → escape_hatch_eval` pipeline as a
connected unit. ORCH-3010 fills that gap.

---

## Goal

Add a Vitest integration test that runs the elaboration delta cluster pipeline end-to-end
via `runElaboration()`, verifying that `delta_detect`, `delta_review`, and `escape_hatch_eval`
nodes compose correctly and produce a coherent, type-safe `ElaborationResult` across the
happy path, the no-change path, and the escape-hatch-triggered path.

---

## Examples

**Example 1 — Happy path (one AC added):**
Given a `previousStory` with 2 ACs and a `currentStory` with 3 ACs (one added), calling
`runElaboration(currentStory, previousStory, config)` returns:

```
{ success: true, deltaDetectionResult: { detected: true, stats: { totalChanges: 1 } },
  deltaReviewResult: { reviewed: true }, escapeHatchResult: { evaluated: true, triggered: false },
  aggregatedFindings: { passed: true } }
```

**Example 2 — Escape hatch triggered (3+ sections changed):**
Given a `previousStory` with 2 ACs / 1 testHint / 1 knownUnknown and a `currentStory`
that adds 2 ACs + 2 testHints + 2 knownUnknowns, with config `escapeHatchConfig.triggerThreshold: 0.1`,
calling `runElaboration` returns:

```
{ success: true, escapeHatchResult: { triggered: true },
  aggregatedFindings: { escapeHatchTriggered: true } }
```

---

## Edge Cases

**Edge Case 1 — No-change path:**
Given identical `previousStory` and `currentStory`, `runElaboration` returns
`deltaDetectionResult.stats.totalChanges: 0`. Delta review is skipped (no sections changed).
`aggregatedFindings.passed: true`. No crash on zero-delta input.

**Edge Case 2 — Null previousStory (first elaboration):**
Given `previousStory: null`, all items in `currentStory` are treated as "added" by
`delta_detect`. Pipeline completes successfully without error. `result.success: true`.

**Edge Case 3 — High triggerThreshold suppresses escape hatch:**
Even with a large cross-cutting diff spanning 3+ sections, setting
`escapeHatchConfig.triggerThreshold: 0.99` results in `escapeHatchResult.triggered: false`.
The threshold is a configurable guard, not a hard-coded trigger.

**Edge Case 4 — Empty story (no ACs, no sections):**
Given `currentStory` with all sections empty and `previousStory: null`, `runElaboration`
returns `success: true` with `totalChanges: 0`. No crash on empty input.

- Do NOT export new functions from `elaboration.ts` to make integration tests work.
  Test exclusively via the public `runElaboration` API.

## Architecture Notes

- **Pure computation cluster**: `delta_detect`, `delta_review`, and `escape_hatch_eval` make
  zero LLM calls. No AI mocking is required. Tests run deterministically.
- **Test isolation**: `recalculateReadiness: false` skips the `update_readiness` node (LLM-dependent).
  `persistToDb: false` skips `load_from_db` and `save_to_db` nodes (DB-dependent).
- **File placement**: Integration test goes in `src/__tests__/integration/` (not `src/graphs/__tests__/`)
  to follow the established integration vs. unit separation established by `change-loop.integration.test.ts`.
- **No barrel file imports**: Import directly from source modules (e.g., `from '../graphs/elaboration.js'`).
- **vi.mock hoisting**: The `vi.mock('@repo/logger', ...)` call MUST appear at top level of the
  test file, not inside `describe` blocks, to be hoisted correctly by Vitest.
- **Escape hatch calibration**: Before writing AC-5, read `escape-hatch.ts` to understand
  confidence scoring for `cross_cutting`. Set `triggerThreshold: 0.1` to ensure even
  low-confidence cross-cutting detections fire the trigger.
- **ORCH-2010 coordination**: ORCH-3010 does not import routing functions from `elaboration.ts`,
  so there is no write conflict with ORCH-2010. Recommend implementing ORCH-3010 after
  ORCH-2010 merges to avoid any transient type drift.
