## Architecture Notes

**Migration pattern (established by WINT-7070):**
1. Replace `{FEATURE_DIR}/stories/{STORY_ID}/` path construction with `kb_get_story({ story_id })`
2. Replace filesystem artifact reads with `kb_read_artifact({ story_id, artifact_type, artifact_name })`
3. Replace filesystem artifact writes with `kb_write_artifact({ story_id, artifact_type, artifact_name, content })`
4. Replace state file writes with `kb_update_story_status`
5. After updating call bodies, scan Output section prose for remaining filesystem path references
6. Deprecate agents superseded by LangGraph/orchestrate-story-flow.sh rather than migrating complex state-management logic

**Scrum-master STATE.md decision rationale (KB OPP-002):**
`_workflow/STATE.md` has no direct KB equivalent. The `orchestrate-story-flow.sh` script and LangGraph pipeline handle story state management in the MVP architecture. If no active invocations of `/scrum-master` exist in `.claude/` (excluding the agent files themselves), deprecation is faster and lower risk than a full KB state-machine rewrite.

**ADR-005 preservation:**
uat-orchestrator and uat-precondition-check enforce the ADR-005 testing constraint (real services, no mocks in UAT). This behavioral constraint must be preserved after migration — only filesystem path references are changed, not the behavioral logic or constraint language.
