# WINT-5020: Create Classification Agent

## Context

The WINT workflow platform has a fully deployed ML pipeline infrastructure:
`workflow.ml_models`, `workflow.model_predictions`, and `workflow.training_data` tables with 7
operational MCP tools (WINT-0140). The HiTL Interview Sidecar (WINT-5010) produces labeled
`hitl_interview` training rows as upstream data for the classification layer. The Classification
Agent is the next layer in this pipeline: it applies classification logic to workflow entities
(stories, decisions) and records structured predictions via `mlPredictionRecord`.

This story delivers a `.claude/agents/classification.agent.md` agent-prompt file following the
established `weekly-analyst.agent.md` and `workflow-retro.agent.md` patterns. No new sidecar,
HTTP server, or DB schema is required. An optional TypeScript compute helper may accompany the
agent if classification logic exceeds what the agent prompt can orchestrate directly via KB tool
calls.

---

## Goal

Enable the WINT pipeline to classify workflow entities (stories) by quality, risk, or effort using
registered ML models or a heuristic fallback, recording structured predictions in
`workflow.model_predictions` for downstream consumption.

---

## Examples

**Example 1 — Risk classification with active model:**

Given: `storyId: "WINT-9010"`, `classificationTarget: "risk"`, and an active `risk_classifier`
model registered in `workflow.ml_models`

The agent calls `mlModelGetActive({ modelType: "risk_classifier" })`, receives the active model's
UUID, computes features (e.g., acCount, complexity signals, agentPhase), calls
`mlPredictionRecord({ modelId, predictionType: "risk", entityType: "story", entityId: "WINT-9010", features, prediction })`,
and emits: `CLASSIFICATION COMPLETE: WINT-9010 — risk score recorded`

**Example 2 — Quality classification with no active model (heuristic fallback):**

Given: `storyId: "WINT-9020"`, `classificationTarget: "quality"`, and no active `quality_predictor`
model in `workflow.ml_models`

The agent calls `mlModelGetActive({ modelType: "quality_predictor" })`, receives an empty array,
logs a warning via `@repo/logger`, computes a heuristic quality score from AC count and scope
description, and emits: `CLASSIFICATION COMPLETE: WINT-9020 — heuristic quality score, no active model`

---

## Edge Cases

**Edge Case 1 — Missing required storyId input:**

Agent invoked without `storyId`. Agent emits an error signal, logs via `@repo/logger`, and does
NOT call `mlPredictionRecord` with a null/undefined `entityId` (which would violate FK constraints
on `workflow.model_predictions`).

**Edge Case 2 — Unknown classificationTarget value:**

Agent receives a `classificationTarget` value not in the enum (`quality | risk | effort`). Agent
emits an error signal or defaults to heuristic mode. It does NOT pass an unknown model type to
`mlModelGetActive` (which could silently return an empty array with a confusing log message).

## Architecture Notes

### ARCH-001: MCP Tool Direct Call (from WINT-5010)

MCP tool wrappers call compute functions directly. No HTTP fetch to localhost. If a TypeScript
compute helper is needed, it must be invoked synchronously without self-calling the MCP server.

### ARCH-002: No Cross-Sidecar HTTP (from WINT-5010)

Sidecars must not issue HTTP calls to other sidecars. Shared compute must go through `@repo/db`
or shared packages.

### Fallback-First Design

The agent must handle the cold-start scenario where `mlModelGetActive` returns an empty array
(WINT-5010 not yet deployed, no validated training data, no registered model). In this case:

1. Log warning via `@repo/logger`
2. Compute heuristic score from available inputs (AC count, scope description, storyId signals)
3. Return deferred/null prediction OR record heuristic prediction with a sentinel `modelId` if supported
4. Emit warning signal to caller — never fail silently

### classificationTarget → modelType Mapping

| classificationTarget | modelType (MlModelTypeSchema) |
|---------------------|-------------------------------|
| `quality` | `quality_predictor` |
| `risk` | `risk_classifier` |
| `effort` | `effort_estimator` |

### Feature Schema for Classification Input

Recommended features derived from `hitl_interview` training data and story metadata:

```yaml
features:
  acCount: integer           # number of acceptance criteria
  storyComplexityScore: float # from hitl_interview rows (if available)
  agentPhase: string         # current workflow phase
  decisionType: string       # decision type if applicable
  confidence: float          # hitl confidence signal (0.0–1.0)
  scopeKeywords: string[]    # keywords extracted from scope description
```
