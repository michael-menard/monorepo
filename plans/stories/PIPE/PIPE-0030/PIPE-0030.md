# PIPE-0030: Add Integration Tests and Observability for Artifact Gate Enforcement in `kb_update_story_status`

## Context

The `kb_update_story_status` MCP tool already enforces artifact preconditions before allowing certain state transitions. The `ARTIFACT_GATES` map in `apps/api/knowledge-base/src/crud-operations/story-crud-operations.ts` (lines 589-621) gates four transitions:

- `elab ‚Üí ready` requires `elaboration` artifact
- `in_progress ‚Üí needs_code_review` requires `proof` artifact
- `needs_code_review ‚Üí ready_for_qa` requires `review` artifact
- `in_qa ‚Üí completed` requires `qa_gate` artifact

**Problem**: No integration tests exist for this gate logic. The existing `story-crud-operations.test.ts` covers `kb_create_story`, `kb_get_story`, and `kb_update_story` but has zero tests for `kb_update_story_status` ‚Äî including no coverage of the gate rejection path. A silent regression that removes or changes the gate map would go undetected.

Additionally, the original story description referenced `qa_verify` as the artifact type for the completed gate, but the code uses `qa_gate`. This naming discrepancy needs to be reconciled in a code comment to prevent future confusion.

**Scope**: The implementation is complete. This story is purely about test coverage and a naming clarification comment. No new migrations, no schema changes, no tool handler changes.

---

## Goal

Provide integration test coverage for all four artifact gate transitions in `kb_update_story_status` ‚Äî both happy path (artifact present, transition allowed) and rejection path (artifact absent, transition blocked) ‚Äî so regressions in the gate enforcement logic are caught immediately.

---

## Examples

**Example 1 ‚Äî Gate rejected (no artifact):**
Given a story in state `in_progress` with no `proof` artifact in `story_artifacts`, calling `kb_update_story_status({ story_id, state: "needs_code_review" })` returns `{ updated: false, message: "...required artifact 'proof' not found in KB" }`. The story state remains `in_progress`.

**Example 2 ‚Äî Gate passed (artifact present):**
Given a story in state `in_qa` with a `qa_gate` artifact written to `story_artifacts`, calling `kb_update_story_status({ story_id, state: "completed" })` returns `{ updated: true, story: { state: "completed" } }`.

---

## Edge Cases

**Edge Case 1 ‚Äî Ungated transition is never blocked:**
Given a story in state `backlog`, calling `kb_update_story_status({ story_id, state: "created" })` succeeds with `updated: true` regardless of whether any artifacts exist. No artifact check is performed for non-gated transitions.

**Edge Case 2 ‚Äî Rejection leaves state unchanged:**
When a gate rejects a transition (e.g., `in_progress ‚Üí needs_code_review` with no `proof`), a subsequent `kb_get_story` call must confirm the story state is still `in_progress`. The gate must be side-effect free.

**Edge Case 3 ‚Äî Wrong direction from needs_code_review is not gated:**
`needs_code_review ‚Üí failed_code_review` has no gate. A story can move to `failed_code_review` without any artifact, even though `needs_code_review ‚Üí ready_for_qa` requires a `review` artifact.

## Architecture Notes

The gate implementation in `story-crud-operations.ts` queries `storyArtifacts` with:

```typescript
WHERE storyId = ? AND artifactType = ?  LIMIT 1
```

The return shape on rejection is:

```typescript
{ story: existingStory, updated: false, message: "Cannot transition ... required artifact '${type}' not found in KB" }
```

This is a `{ updated: false }` return, NOT a thrown exception. Tests must assert on the return value, not catch errors.

The `ARTIFACT_GATES` key format is `"${fromState}‚Üí${toState}"` using a Unicode arrow (‚Üí U+2192), not a hyphen-minus. Test setup must use the same canonical state strings from `StoryStateSchema`.

**Naming reconciliation (AC-12)**: The artifact type is `qa_gate` in both `ARTIFACT_GATES` and the `GATED_ARTIFACT_TYPES` set. Historical story descriptions used `qa_verify`. A clarifying comment in the `ARTIFACT_GATES` map will resolve ambiguity for future readers.
