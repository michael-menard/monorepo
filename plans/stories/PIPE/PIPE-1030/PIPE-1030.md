## Edge Cases

**Idempotent re-run**: If one of the 4 target stories was previously partially elaborated and
already advanced to `elab` state by another process, skip elaboration for that story (do not
overwrite existing ELABORATION artifact) and log an idempotent-skip notice. Continue with
remaining stories.

**Pre-existing test suite failures**: Running `pnpm test --filter @repo/orchestrator` without
scoping will show 13 pre-existing failures in `health-gate.integration.test.ts` (these require a
live DB). These failures are NOT introduced by PIPE-1030. Always use scoped test commands
(e.g., `pnpm test --filter @repo/orchestrator -- validateGraphState`) for story-specific
verification.

**ORCH-4010/ORCH-4020 dependency inversion**: ORCH-4010 (full graph tests) is technically a
prerequisite of ORCH-4020 (E2E tests), but ORCH-4020 is already completed. When elaborating
ORCH-4010, use ORCH-4020's completed patterns as canonical references — the completed E2E test
story provides full-graph mock patterns that make ORCH-4010 elaboration easier, not harder.

**Ghost state usage**: If an agent attempts to use ghost states (`ready_to_work`, `in_review`,
`uat`) in KB tool calls, Zod validation will fail after PIPE-0010. Always use canonical states:
`backlog`, `created`, `elab`, `ready`, `in_progress`, `needs_code_review`, `ready_for_qa`,
`in_qa`, `completed`.
