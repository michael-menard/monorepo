# PIPE-1040 — Confirm APIP-1031 and APIP-1032 Code Is Merged to Main and Tests Pass

## Context

PIPE-1040 was created to verify that APIP-1031 (implementation graph skeleton) and APIP-1032 (change-loop node) — developed in worktrees and advanced to UAT state — have been properly merged to the main branch and are functional.

**Current Reality (2026-03-18)**: Both stories are confirmed merged to main:

- `feat(APIP-1031)` — commit `2164bd74` — all 5 implementation source files present on main
- `feat(APIP-1032)` — commit `c525a8b5` — `change-loop.ts` and `change-spec-schema.ts` present on main

All 89 APIP-1031/1032-specific tests pass across 9 test files. The orchestrator suite has 37 pre-existing failures in unrelated files (index-adapter, story-file-adapter, health-gate integration, dep-audit DB integration) that are not attributable to these stories.

**Gap Identified**: `runImplementation()` from `packages/backend/orchestrator/src/graphs/implementation.ts` is confirmed **not exported** from `packages/backend/orchestrator/src/graphs/index.ts`. This means downstream stories (PIPE-2020, PIPE-3010) cannot consume the implementation graph via the package's public API surface. This is documented here as a finding for PIPE-3010 to resolve — it is out of scope for PIPE-1040.

**Active Dependents**:

- PIPE-2020 (Dispatch Router) is `in_progress` and needs `runImplementation()` availability confirmed
- PIPE-3020 (Worktree Isolation) is `in_progress` and uses `create-worktree.ts` as the implementation pattern
