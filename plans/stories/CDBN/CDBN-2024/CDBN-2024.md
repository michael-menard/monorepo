# CDBN-2024: Cleanup — Drop/Deprecate `public.stories`, Redirect `story-similarity.ts` to `workflow.stories`, Update Drizzle ERD

## Context

The four-story CDBN migration sequence (CDBN-2021 through CDBN-2024) consolidates the legacy `public.stories` table into the canonical `workflow.stories` Drizzle-managed table:

- **CDBN-2021**: Data migration from `public.stories` to `workflow.stories` (in_progress)
- **CDBN-2022**: Migrate related tables and FK references to workflow schema (backlog)
- **CDBN-2023**: Verify data integrity and update all FK references (backlog)
- **CDBN-2024**: Final cleanup — drop `public.stories`, fix live code references, update Drizzle schema (this story)

CDBN-2024 is the **terminal story** in this chain. It cannot begin until CDBN-2023 is complete and verified.

The current canonical stories table is `workflow.stories`, defined in `apps/api/knowledge-base/src/db/schema/workflow.ts`. The `999_full_schema_baseline.sql` does not include `public.stories`, suggesting it was already dropped in the live DB. However, one live code reference remains broken: `apps/api/knowledge-base/src/search/story-similarity.ts` still queries `FROM public.stories` — a table that no longer exists in the canonical schema. This is the primary technical problem to solve.

---

## Goal

Remove the legacy `public.stories` table (via an idempotent migration), redirect `findSimilarStories` to `workflow.stories`, and clean up stale Drizzle schema definitions so the knowledge base story similarity feature functions correctly against the canonical data model.

---

**Example 1 — Migration idempotency:**
Given a database where `public.stories` does not exist (already dropped by CDBN-2021), when the CDBN-2024 cleanup migration is applied, then the migration completes without error (`DROP TABLE IF EXISTS` is a no-op) and subsequent pgtap assertions confirm `hasnt_table('public', 'stories', ...)` passes.

**Example 3 — Zod type correctness:**

## Edge Cases

**Edge Case 1 — embedding column absent on workflow.stories:**
If `workflow.stories` does not yet have the `embedding vector(1536)` column (it is explicitly absent from the current Drizzle schema in `workflow.ts` line 53), redirecting `story-similarity.ts` will produce a runtime SQL error. AC-3 requires this column to be confirmed present or added in this migration before the redirect is deployed.

**Edge Case 2 — storyDetails imported by active code:**
If any active code imports `storyDetails` from `legacy.ts`, removing the Drizzle definition will cause a runtime error (Drizzle table object undefined). The implementation must run `grep -rn "storyDetails" apps/api/knowledge-base/src/ --include="*.ts"` to verify no active imports before removing the definition.

**Edge Case 3 — audit_story_changes() function orphaned after CASCADE drop:**
`DROP TABLE public.stories CASCADE` will silently drop `story_audit_trigger`. The `audit_story_changes()` PostgreSQL function may then be orphaned (no longer referenced). The migration must explicitly document and DROP orphaned functions rather than leaving them silently broken.
