# LEGO MOC Design System

Welcome to the **LEGO MOC Instructions Platform Design System** - a comprehensive component library and documentation hub for building consistent, accessible, and delightful user experiences.

## 🎯 **What's Inside**

This design system contains:

### 🧱 **Core Components** (`@repo/ui`)
- **Buttons** - Primary, secondary, destructive, and icon variants
- **Forms** - Inputs, selects, checkboxes, and validation
- **Layout** - Containers, grids, and spacing utilities
- **Navigation** - Headers, sidebars, and breadcrumbs
- **Feedback** - Toasts, alerts, and loading states
- **Data Display** - Tables, cards, and lists

### 🔐 **Authentication** (`@repo/auth`)
- Login and registration forms
- Password reset flows
- Email verification components
- User profile management

### 📁 **File Management** (`@repo/upload`)
- Drag-and-drop file uploads
- Image preview and cropping
- Progress indicators
- File validation and error handling

### 🖼️ **Gallery** (`@repo/gallery`)
- Image galleries with filtering
- Lightbox and zoom functionality
- Responsive grid layouts
- Image optimization

### 📋 **MOC Instructions** (`@repo/moc-instructions`)
- Step-by-step instruction builders
- Part lists and inventories
- Difficulty ratings
- Progress tracking

### ♿ **Accessibility** (`@repo/accessibility`)
- Screen reader utilities
- Keyboard navigation helpers
- Focus management
- ARIA components

## 🎨 **Design Principles**

### **Consistency**
All components follow the same design patterns, spacing, and interaction behaviors.

### **Accessibility First**
Every component is built with accessibility in mind, following WCAG 2.1 AA guidelines.

### **Performance**
Components are optimized for bundle size and runtime performance.

### **Developer Experience**
TypeScript-first with comprehensive documentation and examples.

## 🚀 **Getting Started**

### **For Developers**
```bash
# Start the design system
pnpm --filter @repo/design-system dev

# Build for production
pnpm --filter @repo/design-system build
```

### **For Designers**
Browse the components in the sidebar to see:
- **Visual examples** of each component
- **Interactive controls** to test different props
- **Code examples** for implementation
- **Accessibility notes** and best practices
