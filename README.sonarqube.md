# SonarQube Quick Start Guide

Quick reference for running SonarQube code quality analysis on this monorepo.

## TL;DR - First Time Setup

```bash
# 1. Start SonarQube (requires Docker)
pnpm sonar:up

# 2. Open http://localhost:9001
# Login: admin / admin (you'll be prompted to change password)

# 3. Generate a token: Profile → My Account → Security → Generate Token

# 4. Create .env file in project root
echo "SONAR_TOKEN=your_token_here" > .env

# 5. Run analysis
pnpm sonar:test    # Generate coverage
pnpm sonar:scan    # Analyze code
```

## Daily Usage

```bash
# Start SonarQube
pnpm sonar:up

# Run tests and analysis
pnpm sonar:test && pnpm sonar:scan

# View results at http://localhost:9001

# Stop SonarQube
pnpm sonar:down
```

## Available Commands

| Command | Description |
|---------|-------------|
| `pnpm sonar:up` | Start SonarQube server (Docker) |
| `pnpm sonar:down` | Stop SonarQube server |
| `pnpm sonar:reset` | Stop and delete all data/volumes |
| `pnpm sonar:test` | Run tests with coverage report |
| `pnpm sonar:scan` | Run SonarQube code analysis |

## Access SonarQube

- **URL**: http://localhost:9001
- **Default Login**: `admin` / `admin` (change on first login)

> **Note**: SonarQube runs on port 9001 (not 9000) to avoid conflicts

## Configuration Files

- `docker-compose.sonarqube.yml` - Docker setup
- `sonar-project.properties` - Analysis configuration
- `.env` - Authentication token (create from `.env.example`)

## Troubleshooting

### SonarQube won't start

```bash
# Make sure Docker is running
docker info

# Check logs
docker compose -f docker-compose.sonarqube.yml logs sonarqube

# Reset everything
pnpm sonar:reset
pnpm sonar:up
```

### "Not authorized" error during scan

Make sure you've:
1. Generated a token in SonarQube UI
2. Added it to `.env` file as `SONAR_TOKEN=your_token`

### No coverage data shown

Run tests before scanning:

```bash
pnpm sonar:test
pnpm sonar:scan
```

### Port already in use

The setup uses port 9001. If that's taken, edit `docker-compose.sonarqube.yml`:

```yaml
ports:
  - '9002:9000'  # Change 9001 to any available port
```

Also update `sonar-project.properties`:

```properties
sonar.host.url=http://localhost:9002
```

## What Gets Analyzed?

- **Source Code**: All TypeScript/JavaScript in `apps/` and `packages/`
- **Tests**: All `.test.ts`, `.test.tsx`, `.spec.ts`, `.spec.tsx` files
- **Coverage**: Generated by Vitest from `pnpm sonar:test`

**Excluded**:
- `node_modules`, build outputs, `.next`, coverage reports, test files (from duplication)

## Stopping SonarQube

```bash
# Stop containers (data persists)
pnpm sonar:down

# Stop and remove all data
pnpm sonar:reset
```

Data persists in Docker volumes between restarts unless you use `sonar:reset`.
